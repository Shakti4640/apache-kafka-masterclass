📋 PHASE 4: HARD — ADVANCED PATTERNS

🔴 HYPOTHETICAL PROBLEM

You run a high-traffic online marketplace with these demands:

WRITE LOAD:
  → 5,000 orders placed per second (peak)
  → Each order requires: inventory check, payment validation,
    fraud scoring, loyalty points calculation
  → Complex business rules, heavy validation
  → Must be CONSISTENT — cannot oversell inventory

READ LOAD:
  → 200,000 product page views per second (peak)
  → Each view needs: product details, current price, stock level,
    average rating, seller info, delivery estimate
  → Must be FAST — < 50ms response time
  → Can tolerate slightly stale data (1-2 seconds)

THE CRISIS WITH SINGLE DATABASE:
─────────────────────────────────
  ┌─────────────────────────────────────────────────────┐
  │ SINGLE PostgreSQL DATABASE                           │
  │                                                       │
  │ Writes: 5,000/sec                                    │
  │   → Row locks during inventory update                │
  │   → Transaction isolation → blocking reads           │
  │   → Complex validation queries → slow writes         │
  │                                                       │
  │ Reads: 200,000/sec                                   │
  │   → Competing for same connection pool                │
  │   → Read queries blocked by write locks              │
  │   → Complex JOINs across 15 tables per page view    │
  │   → Indexes optimized for writes hurt read perf      │
  │   → Indexes optimized for reads hurt write perf      │
  │                                                       │
  │ RESULT:                                               │
  │   → Write latency: 500ms (should be 50ms)            │
  │   → Read latency: 2000ms (should be 10ms)            │
  │   → Both sides suffer because of shared resource      │
  │   → Cannot scale reads without also scaling writes    │
  │   → Cannot optimize schema for both workloads         │
  └─────────────────────────────────────────────────────┘

THE ROOT CAUSE:
───────────────
  → SAME data model serves BOTH reads and writes
  → Write model needs: normalized tables, constraints, transactions
  → Read model needs: denormalized views, pre-computed aggregates
  → These are FUNDAMENTALLY DIFFERENT requirements
  → Trying to serve both with ONE model = compromise on BOTH

WHAT PROJECT 34 REVEALED:
─────────────────────────
  → Event sourcing naturally split into:
      WRITE: Command handler → event store
      READ: Projection consumer → materialized view
  → But both lived in the SAME application
  → Same process, same scaling, same deployment

WHAT WE NEED:
──────────────
  → SEPARATE the write side from the read side
  → Scale them INDEPENDENTLY
  → Optimize each for its specific workload
  → Connect them via Kafka (event bus)
  → THIS IS CQRS

Step 3: UNDERSTAND EVENTUAL CONSISTENCY
  → Write side produces event at T=0
  → Kafka delivers event at T=2ms
  → Query service processes at T=5ms
  → Read model updated at T=6ms
  
  → Between T=0 and T=6ms: READ returns STALE data
  → This is EVENTUAL CONSISTENCY
  → For most queries: 6ms staleness is ACCEPTABLE
  → For critical queries: read from write side directly

🟡 WHAT COULD BE DONE

Option A: Database Read Replicas
  → Primary DB for writes, replicas for reads
  → Simple to implement
  → Problem: Same schema for both — cannot optimize separately
  → Problem: Replication lag = eventual consistency anyway
  → Problem: Complex JOINs still slow on replicas

Option B: CQRS with Kafka (✓ CORRECT APPROACH)
  → Separate write and read services
  → Connected via Kafka event topics
  → Each read model optimized for specific query patterns
  → Independent scaling, independent deployment
  → Event sourcing (Project 34) on write side
  → Multiple materialized views on read side

Option C: CQRS with Direct Database Sync
  → Write service updates write DB
  → Change Data Capture syncs to read DB (Project 37 Debezium)
  → Read DB has different schema optimized for queries
  → Less event-driven, more database-centric

Option D: GraphQL Federation
  → Single GraphQL endpoint
  → Resolvers route to different microservices
  → Each service owns its data
  → Not true CQRS but achieves similar separation
  → Better for API layer, doesn't solve storage optimization

✅ WHAT SHOULD DO

1. DEFINE CLEAR BOUNDARY BETWEEN COMMAND AND QUERY
   → Commands: POST, PUT, DELETE — modify state
   → Queries: GET — read state
   → NEVER modify state during a query
   → NEVER query the write store for display (use read store)

2. DESIGN READ MODELS FOR SPECIFIC QUERY PATTERNS
   → Product page needs: name, price, stock, rating, seller
   → Pre-compute and store as single denormalized document
   → ONE read per query — no JOINs, no aggregations at query time
   → Different read models for different query patterns

3. USE KAFKA AS THE EVENT BUS BETWEEN SIDES
   → Write side produces domain events to Kafka
   → Read side consumes events and updates projections
   → Kafka provides: ordering, durability, replay capability
   → Multiple read services from same event stream

4. HANDLE EVENTUAL CONSISTENCY EXPLICITLY
   → Inform users: "Order placed. Tracking updates shortly."
   → Use correlation IDs to track command → event → view update
   → Provide "freshness" metadata in query responses
   → For critical reads: query write side directly (escape hatch)

5. IMPLEMENT PROJECTION REBUILDING
   → Read model corrupted or needs schema change?
   → Reset consumer offset to beginning → replay all events
   → Rebuild read model from scratch
   → Blue-green deployment: build new projection, switch traffic

6. VERSION YOUR EVENTS (connects to Project 38)
   → Events are the CONTRACT between write and read sides
   → Changing event schema affects ALL read services
   → Use schema evolution: backward + forward compatible
   → Include schema_version in every event

7. MONITOR BOTH SIDES INDEPENDENTLY
   → Write side: command latency, rejection rate, event production rate
   → Read side: query latency, consumer lag, projection freshness
   → Alert: if consumer lag exceeds SLA threshold

❌ WHAT SHOULD NOT DO

1. DO NOT QUERY THE WRITE STORE FOR READ OPERATIONS
   → Write store is optimized for writes (normalized, event log)
   → Querying it defeats the purpose of CQRS
   → Exception: "read-your-writes" for critical consistency

2. DO NOT UPDATE READ STORE DIRECTLY (BYPASS EVENTS)
   → All state changes MUST flow through events
   → Direct updates create inconsistency between stores
   → Read store is DERIVED — events are source of truth

3. DO NOT USE SYNCHRONOUS COMMUNICATION BETWEEN SIDES
   → Command service should NOT call query service
   → Query service should NOT call command service
   → Both communicate ONLY through Kafka events
   → Synchronous coupling defeats independent scaling

4. DO NOT BUILD ONE GIANT READ MODEL
   → "Universal read model with everything" = back to single DB
   → Build SPECIFIC read models for SPECIFIC query patterns
   → Product catalog view ≠ order history view ≠ analytics view
   → Each optimized differently

5. DO NOT IGNORE CONSUMER LAG MONITORING
   → Consumer lag = staleness of read model
   → If lag grows → queries return increasingly stale data
   → Must monitor and alert on lag thresholds
   → Scale read-side consumers when lag increases

6. DO NOT FORGET IDEMPOTENT PROJECTIONS
   → Consumer may process same event twice (rebalance, restart)
   → Projection update MUST be idempotent
   → Processing same event twice = same result as processing once
   → Use event sequence numbers to detect duplicates

7. DO NOT COUPLE COMMAND AND QUERY DEPLOYMENT
   → Deploy write side independently from read side
   → Scaling write side should not require read side changes
   → Read side bug should not bring down write side
   → Separate repos, separate CI/CD, separate monitoring

🚫 WHEN NOT TO USE

1. WHEN APPLICATION IS SIMPLE CRUD
   → Blog posts, user profiles, settings pages
   → Read and write patterns are similar
   → CQRS adds complexity with no benefit
   → Use standard MVC/repository pattern

2. WHEN TEAM IS SMALL AND UNFAMILIAR
   → CQRS requires understanding eventual consistency
   → Debugging across two systems is harder
   → Start with single model, evolve to CQRS when pain emerges
   → Premature CQRS = premature complexity

3. WHEN STRONG CONSISTENCY IS NON-NEGOTIABLE
   → Banking regulations requiring immediate consistency
   → Real-time trading systems (sub-millisecond)
   → CQRS eventual consistency may violate requirements
   → Use distributed transactions instead (slower but consistent)

4. WHEN DATA MODEL IS IDENTICAL FOR READS AND WRITES
   → If read model = write model, separation adds no value
   → Only split when models diverge significantly
   → Different indexes don't justify CQRS — use read replicas

5. WHEN EVENT VOLUME IS LOW
   → < 100 events/sec → single database handles both easily
   → CQRS infrastructure cost > performance benefit
   → Consider CQRS only when scaling becomes an issue

          EVENTUAL CONSISTENCY — TIMING DIAGRAM
═══════════════════════════════════════════════════════════════

  T=0ms:    Command received by write service
  T=10ms:   Validation complete, event produced to Kafka
  T=12ms:   Kafka acknowledges (acks=all, replication done)
  T=15ms:   Response sent to client: 202 Accepted
  T=18ms:   Product catalog consumer receives event
  T=20ms:   Product catalog read model updated
  T=22ms:   Order history consumer receives event
  T=25ms:   Order history read model updated

  CONSISTENCY WINDOW:
  T=15ms → T=20ms (5ms): Client got 202, but product page
                          still shows old stock (50 instead of 48)
  T=20ms onwards:         Product page shows correct stock (48)

  FOR MOST APPLICATIONS: 5ms staleness is INVISIBLE to users
  
  PROBLEM SCENARIO:
  T=15ms: Client places order (gets 202 Accepted)
  T=16ms: Client immediately refreshes order history page
  T=16ms: Order history consumer hasn't processed event yet
  T=16ms: Order history shows NO new order → client confused!

  SOLUTIONS:
  1. Optimistic UI: Show order immediately on client side
  2. Polling with correlation_id: "Wait until COR-abc123 appears"
  3. Write-side read: GET /commands/COR-abc123/status
  4. WebSocket: Push update when projection processes event

from confluent_kafka import Producer

class CommandResult:
    """Result of command processing"""
    def __init__(self, success: bool, correlation_id: str,
                 message: str, data: dict = None):
        self.success = success
        self.correlation_id = correlation_id
        self.message = message
        self.data = data or {}

    def to_dict(self) -> dict:
        return {
            "success": self.success,
            "correlation_id": self.correlation_id,
            "message": self.message,
            "data": self.data,
            "timestamp": int(time.time() * 1000),
        }


class WriteService:
    """
    CQRS Command Service (Write Side).
    
    Responsibilities:
    1. Receive commands from API
    2. Validate against current aggregate state
    3. Produce domain events to Kafka topics
    4. Return command acknowledgment
    
    Does NOT:
    - Serve read queries (that's the query service)
    - Maintain denormalized views
    - Respond with full entity details
    """

    def __init__(self, bootstrap_servers: str = "localhost:9092"):
        self.bootstrap = bootstrap_servers

        # ── Event Producer ──
        self.producer = Producer({
            "bootstrap.servers": bootstrap_servers,
            "acks": "all",
            "enable.idempotence": True,
            "retries": 5,
            "linger.ms": 5,
        })

        # ── Write-side state (for command validation) ──
        # In production: event-sourced aggregates (Project 34)
        # Here: in-memory state for demonstration
        self.inventory = {}      # product_id → {stock, name, price, ...}
        self.orders = {}         # order_id → {status, items, ...}
        self.order_sequences = defaultdict(int)
        self._lock = threading.RLock()

        # ── Metrics ──
        self.commands_processed = 0
        self.commands_rejected = 0
        self.events_produced = 0

    def _produce_event(self, topic: str, event: CQRSEvent):
        """Produce event to Kafka topic"""
        self.producer.produce(
            topic=topic,
            key=event.aggregate_id.encode("utf-8"),
            value=event.to_json().encode("utf-8"),
            headers=[
                ("event_type", event.event_type.encode("utf-8")),
                ("correlation_id", event.correlation_id.encode("utf-8")),
            ],
        )
        self.producer.flush()
        self.events_produced += 1

    # ═══════════════════════════════════════════════════════════
    #  COMMAND: Add Product (Seed inventory)
    # ═══════════════════════════════════════════════════════════

    def handle_add_product(self, product_id: str, name: str,
                           category: str, price: float,
                           stock: int, seller: str) -> CommandResult:
        correlation_id = f"COR-{uuid.uuid4().hex[:12]}"

        with self._lock:
            if product_id in self.inventory:
                self.commands_rejected += 1
                return CommandResult(
                    False, correlation_id,
                    f"Product {product_id} already exists"
                )

            # Create event
            event = product_added(product_id, name, category, price, stock, seller)
            event.correlation_id = correlation_id

            # Produce to Kafka
            self._produce_event("marketplace-inventory-events", event)

            # Update write-side state
            self.inventory[product_id] = {
                "product_id": product_id,
                "name": name,
                "category": category,
                "price": price,
                "stock": stock,
                "seller": seller,
            }

        self.commands_processed += 1
        return CommandResult(
            True, correlation_id,
            f"Product {product_id} added",
            {"product_id": product_id}
        )

    # ═══════════════════════════════════════════════════════════
    #  COMMAND: Place Order
    # ═══════════════════════════════════════════════════════════

    def handle_place_order(self, user_id: str,
                           items: list) -> CommandResult:
        """
        Command: PlaceOrder
        items: [{"product_id": "P-001", "quantity": 2}, ...]
        
        Validation:
        1. All products must exist
        2. Sufficient stock for each item
        3. Calculate total
        
        Events produced:
        1. ORDER_PLACED → order-events topic
        2. INVENTORY_RESERVED (per item) → inventory-events topic
        """
        correlation_id = f"COR-{uuid.uuid4().hex[:12]}"
        order_id = f"ORD-{uuid.uuid4().hex[:8].upper()}"

        with self._lock:
            # ── Validate all items ──
            total = 0.0
            enriched_items = []

            for item in items:
                product_id = item["product_id"]
                quantity = item["quantity"]

                product = self.inventory.get(product_id)
                if product is None:
                    self.commands_rejected += 1
                    return CommandResult(
                        False, correlation_id,
                        f"Product {product_id} not found"
                    )

                if product["stock"] < quantity:
                    self.commands_rejected += 1
                    return CommandResult(
                        False, correlation_id,
                        f"Insufficient stock for {product_id}: "
                        f"available={product['stock']}, requested={quantity}"
                    )

                item_total = product["price"] * quantity
                total += item_total
                enriched_items.append({
                    "product_id": product_id,
                    "product_name": product["name"],
                    "quantity": quantity,
                    "unit_price": product["price"],
                    "item_total": round(item_total, 2),
                })

            total = round(total, 2)

            # ── All validations passed → produce events ──

            # Event 1: Order Placed
            order_event = order_placed(
                order_id=order_id,
                user_id=user_id,
                items=enriched_items,
                total=total,
                correlation_id=correlation_id,
            )
            self._produce_event("marketplace-order-events", order_event)

            # Update write-side order state
            self.orders[order_id] = {
                "order_id": order_id,
                "user_id": user_id,
                "items": enriched_items,
                "total": total,
                "status": "PLACED",
                "sequence": 1,
            }

            # Event 2: Inventory Reserved (per item)
            for item in enriched_items:
                product_id = item["product_id"]
                quantity = item["quantity"]

                # Reserve stock
                self.inventory[product_id]["stock"] -= quantity
                remaining = self.inventory[product_id]["stock"]

                inv_event = inventory_reserved(
                    product_id=product_id,
                    quantity=quantity,
                    order_id=order_id,
                    remaining=remaining,
                    correlation_id=correlation_id,
                )
                self._produce_event("marketplace-inventory-events", inv_event)

        self.commands_processed += 1

        # Return 202 Accepted — read model will be updated asynchronously
        return CommandResult(
            True, correlation_id,
            "Order placed successfully",
            {
                "order_id": order_id,
                "total": total,
                "item_count": len(items),
                "status": "PLACED",
                "note": "Order details will be available shortly via query API",
            }
        )

    #  COMMAND: Cancel Order

    def handle_cancel_order(self, order_id: str,
                            reason: str) -> CommandResult:
        correlation_id = f"COR-{uuid.uuid4().hex[:12]}"

        with self._lock:
            order = self.orders.get(order_id)
            if order is None:
                self.commands_rejected += 1
                return CommandResult(
                    False, correlation_id,
                    f"Order {order_id} not found"
                )

            if order["status"] not in ("PLACED", "CONFIRMED"):
                self.commands_rejected += 1
                return CommandResult(
                    False, correlation_id,
                    f"Cannot cancel order in status: {order['status']}"
                )

            # Event 1: Order Cancelled
            order["sequence"] += 1
            cancel_event = order_cancelled(
                order_id=order_id,
                reason=reason,
                sequence=order["sequence"],
                correlation_id=correlation_id,
            )
            self._produce_event("marketplace-order-events", cancel_event)
            order["status"] = "CANCELLED"

            # Event 2: Release inventory (per item)
            for item in order["items"]:
                product_id = item["product_id"]
                quantity = item["quantity"]

                self.inventory[product_id]["stock"] += quantity
                remaining = self.inventory[product_id]["stock"]

                release_event = inventory_released(
                    product_id=product_id,
                    quantity=quantity,
                    order_id=order_id,
                    remaining=remaining,
                    correlation_id=correlation_id,
                )
                self._produce_event("marketplace-inventory-events", release_event)

        self.commands_processed += 1
        return CommandResult(
            True, correlation_id,
            f"Order {order_id} cancelled",
            {"order_id": order_id, "status": "CANCELLED"}
        )

    # ═══════════════════════════════════════════════════════════

    # ═══════════════════════════════════════════════════════════

            f"Order {order_id} shipped",
            {"order_id": order_id, "tracking": tracking_number, "carrier": carrier}

class ProductCatalogProjection:
    """
    Read model optimized for product page queries.
    
    Query patterns:
    - GET /products/{id} → single product with all details
    - GET /products?category=X → filter by category
    - GET /products/search?q=phone → text search
    - GET /products/low-stock → stock alerts
    
    Data is DENORMALIZED — single lookup, no JOINs.
    Updated asynchronously from inventory-events topic.
    """

    def __init__(self):
        self.products = {}
        self._lock = threading.RLock()
        self.events_processed = 0
        self.last_event_time = 0

    def handle_event(self, event_data: dict):
        event_type = event_data.get("event_type", "")
        payload = event_data.get("payload", {})
        timestamp = event_data.get("timestamp", 0)

        with self._lock:
            if event_type == "PRODUCT_ADDED":
                product_id = payload["product_id"]
                self.products[product_id] = {
                    "product_id": product_id,
                    "name": payload.get("name", ""),
                    "category": payload.get("category", ""),
                    "price": payload.get("price", 0),
                    "stock": payload.get("stock", 0),
                    "seller": payload.get("seller", ""),
                    "availability": "IN_STOCK" if payload.get("stock", 0) > 0 else "OUT_OF_STOCK",
                    "last_updated": timestamp,
                    "total_reserved": 0,
                    "total_released": 0,
                }

            elif event_type == "INVENTORY_RESERVED":
                product_id = payload["product_id"]
                if product_id in self.products:
                    product = self.products[product_id]
                    product["stock"] = payload.get("remaining_stock", product["stock"])
                    product["total_reserved"] += payload.get("quantity_reserved", 0)
                    product["availability"] = "IN_STOCK" if product["stock"] > 0 else "OUT_OF_STOCK"
                    product["last_updated"] = timestamp

            elif event_type == "INVENTORY_RELEASED":
                product_id = payload["product_id"]
                if product_id in self.products:
                    product = self.products[product_id]
                    product["stock"] = payload.get("remaining_stock", product["stock"])
                    product["total_released"] += payload.get("quantity_released", 0)
                    product["availability"] = "IN_STOCK" if product["stock"] > 0 else "OUT_OF_STOCK"
                    product["last_updated"] = timestamp

            self.events_processed += 1
            self.last_event_time = timestamp

    # ── Query Methods ──

    def get_product(self, product_id: str) -> dict:
        with self._lock:
            product = self.products.get(product_id)
            if product is None:
                return None
            return {
                **product,
                "projection_freshness_ms": int(time.time() * 1000) - self.last_event_time,
                "projection_name": "product-catalog",
            }

    def search_products(self, query: str) -> list:
        query_lower = query.lower()
        with self._lock:
            return [
                p for p in self.products.values()
                if query_lower in p["name"].lower() or
                   query_lower in p["category"].lower()
            ]

    def get_by_category(self, category: str) -> list:
        with self._lock:
            return [
                p for p in self.products.values()
                if p["category"].lower() == category.lower()
            ]

    def get_low_stock(self, threshold: int = 10) -> list:
        with self._lock:
            return sorted(
                [p for p in self.products.values() if p["stock"] < threshold],
                key=lambda x: x["stock"]
            )

    def get_all_products(self, page: int = 1, page_size: int = 20) -> dict:
        with self._lock:
            all_products = sorted(
                self.products.values(),
                key=lambda x: x["product_id"]
            )
            start = (page - 1) * page_size
            end = start + page_size
            return {
                "products": all_products[start:end],
                "total": len(all_products),
                "page": page,
                "page_size": page_size,
                "total_pages": (len(all_products) + page_size - 1) // page_size,
            }

    def get_metadata(self) -> dict:
        with self._lock:
            return {
                "projection_name": "product-catalog",
                "total_products": len(self.products),
                "events_processed": self.events_processed,
                "last_event_time": self.last_event_time,
                "freshness_ms": int(time.time() * 1000) - self.last_event_time
                                if self.last_event_time > 0 else None,
            }

    def handle_event(self, event_data: dict):

        payload = event_data.get("payload", {})

        correlation_id = event_data.get("correlation_id", "")

        with self._lock:
            if event_type == "ORDER_PLACED":
                order_id = payload["order_id"]
                user_id = payload["user_id"]

                self.orders[order_id] = {
                    "order_id": order_id,
                    "user_id": user_id,
                    "items": payload.get("items", []),
                    "total": payload.get("total", 0),
                    "currency": payload.get("currency", "USD"),
                    "status": "PLACED",
                    "correlation_id": correlation_id,
                    "created_at": timestamp,
                    "last_updated": timestamp,
                    "timeline": [

                            "status": "PLACED",
                            "timestamp": timestamp,
                            "timestamp_human": time.strftime(

                                time.localtime(timestamp / 1000)
                            ),
                        }
                    ],
                    "tracking_number": None,
                    "carrier": None,
                }

                if order_id not in self.user_orders[user_id]:
                    self.user_orders[user_id].append(order_id)

            elif event_type == "ORDER_CONFIRMED":
                order_id = payload["order_id"]
                if order_id in self.orders:
                    self.orders[order_id]["status"] = "CONFIRMED"
                    self.orders[order_id]["last_updated"] = timestamp
                    self.orders[order_id]["timeline"].append({

                        "timestamp": timestamp,
                        "timestamp_human": time.strftime(

                        ),
                    })

            elif event_type == "ORDER_SHIPPED":
                order_id = payload["order_id"]
                if order_id in self.orders:
                    self.orders[order_id]["status"] = "SHIPPED"
                    self.orders[order_id]["tracking_number"] = payload.get("tracking_number")
                    self.orders[order_id]["carrier"] = payload.get("carrier")
                    self.orders[order_id]["last_updated"] = timestamp
                    self.orders[order_id]["timeline"].append({
                        "status": "SHIPPED",
                        "timestamp": timestamp,
                        "timestamp_human": time.strftime(
                            "%Y-%m-%d %H:%M:%S",
                            time.localtime(timestamp / 1000)
                        ),
                        "tracking": payload.get("tracking_number"),
                        "carrier": payload.get("carrier"),

            elif event_type == "ORDER_DELIVERED":
                order_id = payload["order_id"]
                if order_id in self.orders:
                    self.orders[order_id]["status"] = "DELIVERED"
                    self.orders[order_id]["last_updated"] = timestamp
                    self.orders[order_id]["timeline"].append({
                        "status": "DELIVERED",
                        "timestamp": timestamp,
                        "timestamp_human": time.strftime(
                            "%Y-%m-%d %H:%M:%S",
                            time.localtime(timestamp / 1000)
                        ),
                    })

            elif event_type == "ORDER_CANCELLED":
                order_id = payload["order_id"]
                if order_id in self.orders:
                    self.orders[order_id]["status"] = "CANCELLED"
                    self.orders[order_id]["last_updated"] = timestamp

                        "status": "CANCELLED",
                        "timestamp": timestamp,
                        "timestamp_human": time.strftime(
                            "%Y-%m-%d %H:%M:%S",
                            time.localtime(timestamp / 1000)

                        "reason": payload.get("reason", ""),
                    })

            self.events_processed += 1
            self.last_event_time = timestamp

    # ── Query Methods ──

    def get_order(self, order_id: str) -> dict:
        with self._lock:
            order = self.orders.get(order_id)
            if order is None:
                return None
            return {
                **order,
                "projection_freshness_ms": int(time.time() * 1000) - self.last_event_time,
                "projection_name": "order-history",
            }

    def get_user_orders(self, user_id: str, status: str = None) -> list:
        with self._lock:
            order_ids = self.user_orders.get(user_id, [])
            orders = [self.orders[oid] for oid in order_ids if oid in self.orders]

            if status:
                orders = [o for o in orders if o["status"] == status.upper()]

            return sorted(orders, key=lambda x: x["created_at"], reverse=True)

    def get_orders_by_status(self, status: str) -> list:

    def get_metadata(self) -> dict:
        with self._lock:

            for order in self.orders.values():
                status_counts[order["status"]] += 1

            return {
                "projection_name": "order-history",
                "total_orders": len(self.orders),
                "total_users": len(self.user_orders),
                "events_processed": self.events_processed,
                "status_breakdown": dict(status_counts),
                "freshness_ms": int(time.time() * 1000) - self.last_event_time
                                if self.last_event_time > 0 else None,
            }

                    self.orders_by_category[category] += 1

                    self.product_sales[product_id]["revenue"] += item_total
                    self.product_sales[product_id]["name"] = item.get("product_name", product_id)

            elif event_type == "ORDER_CANCELLED":
                self.cancelled_orders += 1

            elif event_type == "ORDER_SHIPPED":
                self.shipped_orders += 1

class ReadService:
    """
    CQRS Query Service (Read Side).
    
    Manages multiple projections, each consuming from Kafka topics
    and building optimized read models.
    
    Responsibilities:
    1. Consume events from Kafka topics
    2. Route events to appropriate projections
    3. Serve queries from projections
    4. Track projection health (freshness, lag)
    """

    def __init__(self, bootstrap_servers: str = "localhost:9092"):
        self.bootstrap = bootstrap_servers
        self.running = False

        # ── Projections ──
        self.product_catalog = ProductCatalogProjection()
        self.order_history = OrderHistoryProjection()
        self.analytics = AnalyticsDashboardProjection()

        # ── Consumer threads ──
        self.consumer_threads = []

    def start(self):
        """Start all projection consumers"""
        self.running = True
        print("  🚀 Starting Read Service (Query Side)...")

        # Consumer for inventory events → product catalog
        t1 = threading.Thread(
            target=self._consume_events,
            args=(
                "marketplace-inventory-events",
                "product-catalog-projection-v1",
                [self.product_catalog],
            ),
            daemon=True,
            name="inventory-consumer",
        )

        # Consumer for order events → order history + analytics
        t2 = threading.Thread(
            target=self._consume_events,
            args=(
                "marketplace-order-events",
                "order-projections-v1",
                [self.order_history, self.analytics],
            ),
            daemon=True,
            name="order-consumer",
        )

        self.consumer_threads = [t1, t2]
        for t in self.consumer_threads:
            t.start()

        # Wait for initial population
        time.sleep(3)
        print("  ✅ Read Service RUNNING")
        self._print_projection_status()

    def _consume_events(self, topic: str, group_id: str,
                        projections: list):
        """Background consumer that feeds events to projections"""
        consumer = Consumer({
            "bootstrap.servers": self.bootstrap,
            "group.id": group_id,
            "auto.offset.reset": "earliest",
            "enable.auto.commit": True,
            "auto.commit.interval.ms": 1000,
        })
        consumer.subscribe([topic])

        while self.running:
            msg = consumer.poll(timeout=1.0)
            if msg is None:
                continue
            if msg.error():
                continue

            try:
                event_data = json.loads(msg.value().decode("utf-8"))

                # Route event to all projections for this topic
                for projection in projections:
                    try:
                        projection.handle_event(event_data)
                    except Exception as e:
                        print(f"    ⚠️ Projection error: {e}")

            except (json.JSONDecodeError, UnicodeDecodeError):
                continue

        consumer.close()

    def _print_projection_status(self):
        print(f"\n  {'─' * 50}")
        print(f"  PROJECTION STATUS")
        print(f"  {'─' * 50}")

        for proj_name, proj in [
            ("Product Catalog", self.product_catalog),
            ("Order History", self.order_history),
            ("Analytics", self.analytics),
        ]:
            meta = proj.get_metadata()
            freshness = meta.get("freshness_ms", "N/A")
            print(f"    {proj_name}: {meta['events_processed']} events, "
                  f"freshness={freshness}ms")
        print()

    def get_all_metadata(self) -> dict:
        return {
            "product_catalog": self.product_catalog.get_metadata(),
            "order_history": self.order_history.get_metadata(),
            "analytics": self.analytics.get_metadata(),
        }

    def stop(self):
        self.running = False
        for t in self.consumer_threads:
            t.join(timeout=5)
        print("  [READ SERVICE] Stopped")
