PROJECT 29: THE ENCRYPTED CHANNEL — SSL/TLS FOR KAFKA

🏗️ HYPOTHETICAL PROBLEM

SCENARIO: FinTech Payment Processing Platform

→ A FinTech company processes credit card transactions via Kafka:
  - 3-broker Kafka cluster handling payment events
  - Topics: card_transactions, fraud_alerts, settlement_batch
  - PCI-DSS compliance required (Payment Card Industry Data Security Standard)
  - SOC 2 Type II audit scheduled in 60 days
  - HIPAA compliance for health-savings account transactions

→ CURRENT STATE: Everything runs over PLAINTEXT

  Producer ──PLAINTEXT──▶ Broker ──PLAINTEXT──▶ Consumer
                            │
                      PLAINTEXT
                            │
                      Broker ◄──▶ Broker
                      (inter-broker replication)

→ THE AUDIT FINDINGS:

  FINDING 1 — "Network Sniffing Attack Surface" (CRITICAL)

  → Security team ran Wireshark on the Kafka network segment

  → PCI-DSS Requirement 4.1: "Encrypt transmission of

  ──────────────────────────────────────────────

  → No verification of client identity

    - Read ALL payment transactions
    - Produce fake transactions

  FINDING 3 — "Inter-Broker Communication Unencrypted" (HIGH)
  ───────────────────────────────────────────────────────────
  → Replication traffic between brokers is plaintext
  → If attacker gains network access between broker nodes

  ────────────────────────────────────────────────────────────────
  → From Project 27: MM2 replicates across datacenters

  - 30 days: Encrypt all client-broker communication

4. WHAT IS SSL/TLS IN KAFKA CONTEXT?

   SSL (Secure Sockets Layer) / TLS (Transport Layer Security):
   → Encrypts data in transit between two endpoints
   → Provides: ENCRYPTION + AUTHENTICATION + INTEGRITY
   
   ENCRYPTION: Data cannot be read by eavesdroppers
   AUTHENTICATION: Proves identity of server (and optionally client)
   INTEGRITY: Data cannot be tampered with in transit

🛠️ HOW CAN WE SOLVE IT

SOLUTION: PHASED SSL/TLS ROLLOUT WITH ZERO-DOWNTIME MIGRATION

PHASE A: Generate Certificates (Day 1)
═══════════════════════════════════════
  1. Create private Certificate Authority (CA)
  2. Generate broker keystores + certificates
  3. Generate client keystores + certificates
  4. Sign all certificates with CA
  5. Create truststores with CA certificate

PHASE B: Enable SSL Listener alongside PLAINTEXT (Day 2-5)
══════════════════════════════════════════════════════════
  Broker config BEFORE:
    listeners=PLAINTEXT://0.0.0.0:9092

  Broker config AFTER:
    listeners=PLAINTEXT://0.0.0.0:9092,SSL://0.0.0.0:9093
    
  → Both PLAINTEXT and SSL available simultaneously
  → Existing clients continue on PLAINTEXT (no disruption)
  → New clients can start using SSL
  → Rolling restart: one broker at a time

PHASE C: Migrate Clients to SSL (Day 5-25)
═══════════════════════════════════════════
  → Producers switch to SSL bootstrap servers
  → Consumers switch to SSL bootstrap servers
  → Kafka Connect workers switch to SSL
  → MirrorMaker 2.0 switches to SSL
  → Monitoring verified at each step (Project 28)

PHASE D: Enable Inter-Broker SSL (Day 25-30)
═════════════════════════════════════════════
  Broker config:
    security.inter.broker.protocol=SSL
    
  → Replication traffic now encrypted
  → Rolling restart: one broker at a time

PHASE E: Remove PLAINTEXT Listener (Day 30-35)
═══════════════════════════════════════════════
  Broker config FINAL:
    listeners=SSL://0.0.0.0:9093
    
  → PLAINTEXT no longer available
  → Any unpatched client will fail to connect (intentional)
  → Forces complete migration

PHASE F: Enable Mutual TLS (Optional, Day 35-45)
═════════════════════════════════════════════════
  Broker config:
    ssl.client.auth=required
    
  → Clients MUST present certificate
  → Broker verifies client identity
  → Strongest authentication level

✅ WHAT SHOULD DO

1. USE A PRIVATE CERTIFICATE AUTHORITY (CA)
   → Don't use self-signed certificates per broker
   → Create ONE CA → sign ALL broker and client certificates
   → Truststores only need the CA cert (not every individual cert)
   → Adding new brokers/clients = sign new cert with same CA
   → Much simpler certificate management

2. USE UNIQUE CERTIFICATES PER BROKER
   → Each broker gets its own keystore with unique certificate
   → Certificate CN (Common Name) = broker hostname
   → Or use SAN (Subject Alternative Name) for multiple hostnames
   → If one broker's key is compromised, only that cert is revoked

3. ENABLE SSL ALONGSIDE PLAINTEXT FIRST
   → Never switch directly from PLAINTEXT-only to SSL-only
   → Dual-listener period allows gradual client migration
   → If SSL misconfigured, PLAINTEXT is fallback
   → Remove PLAINTEXT only after ALL clients migrated

4. ROLLING RESTART FOR ZERO DOWNTIME
   → Change one broker at a time
   → Wait for ISR to fully sync before restarting next
   → Monitor UnderReplicatedPartitions (Project 28) during restart
   → If issues: stop rollout, fix, continue

5. TEST SSL CONFIGURATION IN STAGING FIRST
   → SSL misconfigurations are hard to debug
   → Common errors: wrong password, wrong CN, expired cert
   → Test full producer-broker-consumer flow in staging
   → Verify with openssl s_client command

6. SET CERTIFICATE VALIDITY TO 1-2 YEARS
   → Too short (30 days): constant rotation overhead
   → Too long (10 years): security risk if compromised
   → 1-2 years with automated renewal process
   → CA validity: 5-10 years (longer than individual certs)

7. STORE PASSWORDS SECURELY
   → Keystore and truststore passwords are in broker config
   → server.properties is readable by anyone with server access
   → Use environment variables or external secret management
   → HashiCorp Vault, AWS Secrets Manager, K8s Secrets

8. DOCUMENT THE CERTIFICATE CHAIN
   → Who is the CA? Where is the CA private key stored?
   → What is the expiration date of each certificate?
   → What is the renewal procedure?
   → Who is authorized to sign new certificates?

❌ WHAT SHOULD NOT DO

1. DO NOT USE THE SAME KEYSTORE FOR ALL BROKERS
   → If one broker compromised, attacker has ALL broker identities
   → Cannot revoke one without revoking all
   → Unique keystore per broker = proper isolation

2. DO NOT USE SELF-SIGNED CERTIFICATES IN PRODUCTION
   → Each self-signed cert must be individually trusted
   → 3 brokers + 10 clients = 13 certificates in every truststore
   → Adding a new client = update ALL truststores
   → Use a CA: truststores only need the ONE CA cert

3. DO NOT STORE CA PRIVATE KEY ON BROKER MACHINES
   → CA key signs new certificates
   → If stolen, attacker can create valid certificates for anything
   → Store CA key OFFLINE or in HSM (Hardware Security Module)
   → Only bring online when signing new certificates

4. DO NOT DISABLE HOSTNAME VERIFICATION IN PRODUCTION
   → ssl.endpoint.identification.algorithm="" disables verification
   → Without verification: man-in-the-middle attacks possible
   → Attacker presents ANY valid certificate → accepted
   → Keep ssl.endpoint.identification.algorithm=https

5. DO NOT USE WEAK CIPHER SUITES
   → Kafka defaults are generally secure
   → But don't override with weak ciphers for "compatibility"
   → Avoid: RC4, DES, MD5, SSLv3
   → Prefer: TLS 1.2+, AES-256-GCM, SHA-256+

6. DO NOT FORGET TO ENCRYPT INTER-BROKER TRAFFIC
   → Client-broker SSL is visible
   → Inter-broker is invisible (happens internally)
   → Easy to forget → replication traffic stays plaintext
   → security.inter.broker.protocol=SSL is separate config

7. DO NOT LET CERTIFICATES EXPIRE UNNOTICED
   → Expired certificate = ALL connections fail
   → No warning before expiration unless monitored
   → Set up certificate expiration alerting
   → Alert 30 days before expiration
   → Automate renewal where possible

1. WHEN PERFORMANCE OVERHEAD IS UNACCEPTABLE
   → SSL adds 5-20% throughput reduction
   → For ultra-low-latency (< 1ms requirement): evaluate carefully
   → Consider: encrypt only sensitive topics via separate listener
   → But: compliance usually mandates ALL traffic encrypted

2. WHEN RUNNING PURELY LOCAL DEVELOPMENT
   → localhost Kafka for developer testing
   → SSL adds setup complexity for zero security benefit
   → Use PLAINTEXT for local dev
   → Use SSL in staging and production

3. WHEN USING MANAGED KAFKA WITH BUILT-IN ENCRYPTION
   → Amazon MSK: encryption in transit enabled by default
   → Confluent Cloud: always encrypted
   → Azure Event Hubs: always encrypted
   → No need to manage certificates yourself

  TRUST VALIDATION:
  
  Client has TRUSTSTORE containing: CA certificate
  
  Client connects to Broker 1:
  → Broker presents: Broker 1 certificate
  → Client checks: Is this cert signed by a CA in my truststore?
  → CA cert in truststore → Kafka-CA
  → Broker 1 cert issuer → Kafka-CA
  → MATCH → Trust established ✅
  
  If attacker presents a cert signed by different CA:
  → Client checks: Is this cert signed by Kafka-CA?
  → NO → Connection rejected ❌

PERFORMANCE IMPACT — WHERE THE OVERHEAD COMES FROM
═══════════════════════════════════════════════════════════

  OVERHEAD 1: TLS HANDSHAKE (per connection)
  → Full handshake: ~5-15ms
  → Session resumption: ~2-5ms
  → Kafka uses long-lived connections → handshake is ONE-TIME
  → Impact: negligible after initial connection

  OVERHEAD 2: ENCRYPTION/DECRYPTION (per message)
  → AES-256-GCM: hardware-accelerated on modern CPUs (AES-NI)
  → Per-message overhead: ~0.01-0.05ms
  → At 100K messages/sec: 1-5ms total CPU per second
  → Impact: 5-15% throughput reduction typical

  OVERHEAD 3: MEMORY (per connection)
  → SSL session state: ~20-50KB per connection
  → 1000 connections: 20-50MB additional memory
  → Impact: minor — increase JVM heap if needed

  OVERHEAD 4: CPU (ongoing)
  → Encryption is CPU-intensive
  → With AES-NI hardware: minimal impact
  → Without AES-NI: significant (20-30% CPU increase)
  → Check: cat /proc/cpuinfo | grep aes

  MITIGATION:
  → Use compression BEFORE encryption (Project 39 tuning)
    Compressed data = less data to encrypt = less CPU
  → Increase batch.size (Project 39) to amortize overhead
  → Monitor CPU after enabling SSL (Project 28 metrics)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: HOSTNAME VERIFICATION FAILURE
═════════════════════════════════════════
  ERROR: "SSL handshake failed: hostname verification failed"
  
  → Certificate CN = "kafka-broker-1"
  → Client connects to: "localhost:9093"
  → Hostname "localhost" ≠ "kafka-broker-1"
  → VERIFICATION FAILS
  
  FIX: Use SAN (Subject Alternative Name) in certificate:
       SAN = DNS:kafka-broker-1, DNS:localhost, IP:127.0.0.1
       OR: Connect using the hostname that matches the certificate


PITFALL 2: WRONG KEYSTORE/TRUSTSTORE PASSWORD
══════════════════════════════════════════════
  ERROR: "Keystore was tampered with, or password was incorrect"
  
  → Java keytool uses TWO passwords:
    1. Keystore password (protects the keystore file)
    2. Key password (protects the private key inside)
  → If these don't match what's in server.properties → error
  
  FIX: Use the SAME password for keystore and key
       (simpler management, Kafka requires this for some versions)
       ssl.keystore.password=same-password
       ssl.key.password=same-password


PITFALL 3: TRUSTSTORE MISSING CA CERTIFICATE
═════════════════════════════════════════════
  ERROR: "unable to find valid certification path to requested target"
  
  → Client truststore doesn't contain the CA that signed broker cert
  → Client cannot verify broker identity
  → Happens when: using different CA for broker vs client certs
  
  FIX: Ensure SAME CA cert is in ALL truststores
       keytool -import -alias ca-cert -file ca-cert.pem -keystore truststore.jks


PITFALL 4: CERTIFICATE EXPIRED
═══════════════════════════════
  ERROR: "NotAfter: Thu Jan 15 00:00:00 UTC 2024"
  
  → Certificate validity period has passed
  → ALL connections using this cert fail
  → If broker cert expires: NO clients can connect
  → If CA cert expires: ALL certificates invalid

#!/bin/bash
# ============================================================
# FILE: generate_certs.sh
#
# Generates complete SSL certificate infrastructure:
# 1. Certificate Authority (CA)
# 2. Broker keystore + certificate (signed by CA)
# 3. Client keystore + certificate (signed by CA)
# 4. Truststores (containing CA cert)
# 5. PEM files for Python clients
# ============================================================

set -e

CERTS_DIR="/certs"
CA_DIR="${CERTS_DIR}/ca"
BROKER_DIR="${CERTS_DIR}/broker"
CLIENT_DIR="${CERTS_DIR}/client"

# Passwords
CA_PASSWORD="ca-secret-password"
BROKER_KS_PASSWORD="broker-ks-password"
CLIENT_KS_PASSWORD="client-ks-password"
TRUSTSTORE_PASSWORD="truststore-password"

# Certificate details
CA_CN="Kafka-Dev-CA"
BROKER_CN="kafka-ssl"
CLIENT_CN="kafka-client"
VALIDITY_CA=3650      # 10 years
VALIDITY_CERT=730     # 2 years

echo "╔══════════════════════════════════════════════════════╗"
echo "║  SSL CERTIFICATE GENERATION                          ║"
echo "╚══════════════════════════════════════════════════════╝"

# Check if certs already exist
if [ -f "${BROKER_DIR}/kafka-broker.keystore.jks" ]; then
    echo "  ✅ Certificates already exist. Skipping generation."
    exit 0
fi

mkdir -p ${CA_DIR} ${BROKER_DIR} ${CLIENT_DIR}

# ============================================================
# STEP 1: Generate Certificate Authority (CA)
# ============================================================
echo ""
echo "STEP 1: Generating Certificate Authority..."

# Generate CA key pair and self-signed certificate
openssl req -new -x509 \
    -keyout ${CA_DIR}/ca-key.pem \
    -out ${CA_DIR}/ca-cert.pem \
    -days ${VALIDITY_CA} \
    -subj "/CN=${CA_CN}/O=KafkaProject/C=US" \
    -passout pass:${CA_PASSWORD} \
    2>/dev/null

echo "  ✅ CA certificate generated: ${CA_DIR}/ca-cert.pem"
echo "  ✅ CA private key generated: ${CA_DIR}/ca-key.pem"
echo "     CN=${CA_CN}, Valid for ${VALIDITY_CA} days"

# ============================================================
# STEP 2: Generate Broker Keystore + Certificate
# ============================================================
echo ""
echo "STEP 2: Generating Broker keystore and certificate..."

# 2a: Create broker keystore with key pair
keytool -genkeypair \
    -alias broker \
    -keyalg RSA \
    -keysize 2048 \
    -dname "CN=${BROKER_CN},O=KafkaProject,C=US" \
    -ext "SAN=DNS:${BROKER_CN},DNS:localhost,IP:127.0.0.1" \
    -keystore ${BROKER_DIR}/kafka-broker.keystore.jks \
    -storepass ${BROKER_KS_PASSWORD} \
    -keypass ${BROKER_KS_PASSWORD} \
    -validity ${VALIDITY_CERT} \
    2>/dev/null

echo "  ✅ Broker keystore created"

# 2b: Generate Certificate Signing Request (CSR)
keytool -certreq \
    -alias broker \
    -keystore ${BROKER_DIR}/kafka-broker.keystore.jks \
    -storepass ${BROKER_KS_PASSWORD} \
    -file ${BROKER_DIR}/broker-csr.pem \
    -ext "SAN=DNS:${BROKER_CN},DNS:localhost,IP:127.0.0.1" \
    2>/dev/null

echo "  ✅ Broker CSR generated"

# 2c: Sign broker certificate with CA
# Create extensions file for SAN
cat > ${BROKER_DIR}/broker-ext.cnf << EOF
[v3_req]
subjectAltName = DNS:${BROKER_CN},DNS:localhost,IP:127.0.0.1
EOF

openssl x509 -req \
    -in ${BROKER_DIR}/broker-csr.pem \
    -CA ${CA_DIR}/ca-cert.pem \
    -CAkey ${CA_DIR}/ca-key.pem \
    -CAcreateserial \
    -out ${BROKER_DIR}/broker-cert-signed.pem \
    -days ${VALIDITY_CERT} \
    -passin pass:${CA_PASSWORD} \
    -extensions v3_req \
    -extfile ${BROKER_DIR}/broker-ext.cnf \
    2>/dev/null

echo "  ✅ Broker certificate signed by CA"

# 2d: Import CA cert into broker keystore (trust chain)
keytool -import -noprompt \
    -alias ca-cert \
    -file ${CA_DIR}/ca-cert.pem \
    -keystore ${BROKER_DIR}/kafka-broker.keystore.jks \
    -storepass ${BROKER_KS_PASSWORD} \
    2>/dev/null

echo "  ✅ CA cert imported into broker keystore"

# 2e: Import signed broker cert into keystore
keytool -import -noprompt \
    -alias broker \
    -file ${BROKER_DIR}/broker-cert-signed.pem \
    -keystore ${BROKER_DIR}/kafka-broker.keystore.jks \
    -storepass ${BROKER_KS_PASSWORD} \
    2>/dev/null

echo "  ✅ Signed broker cert imported into keystore"

# 2f: Create broker truststore with CA cert
keytool -import -noprompt \
    -alias ca-cert \
    -file ${CA_DIR}/ca-cert.pem \
    -keystore ${BROKER_DIR}/kafka-broker.truststore.jks \
    -storepass ${TRUSTSTORE_PASSWORD} \
    2>/dev/null

echo "  ✅ Broker truststore created with CA cert"

# ============================================================
# STEP 3: Generate Client Keystore + Certificate
# ============================================================
echo ""
echo "STEP 3: Generating Client keystore and certificate..."

# 3a: Create client keystore
keytool -genkeypair \

keytool -certreq \

openssl x509 -req \

    -CAcreateserial \

echo "  ✅ Client certificate signed by CA"

keytool -import -noprompt \

keytool -import -noprompt \

echo "  ✅ Client keystore complete"

                    empty += 1

    def check_ssl_handshake(self):
        """Perform SSL handshake and inspect certificate"""
        
        print("\n  ┌─ SSL HANDSHAKE CHECK")
        
        ca_cert = f'{CERTS_BASE}/ca-cert.pem'
        client_cert = f'{CERTS_BASE}/client-cert.pem'
        client_key = f'{CERTS_BASE}/client-key.pem'
        
        try:
            # Create SSL context
            context = python_ssl.create_default_context(
                python_ssl.Purpose.SERVER_AUTH,
                cafile=ca_cert
            )
            
            # Disable hostname verification for Docker
            context.check_hostname = False
            context.verify_mode = python_ssl.CERT_REQUIRED
            
            # Load client certificate (for mTLS)
            if os.path.exists(client_cert) and os.path.exists(client_key):
                context.load_cert_chain(
                    certfile=client_cert,
                    keyfile=client_key,
                    password='client-ks-password'
                )
                print(f"  │  ✅ Client certificate loaded (mTLS)")
            
            # Perform handshake
            start = time.time()
            
            raw_sock = socket.create_connection((BROKER_HOST, SSL_PORT), timeout=10)
            ssl_sock = context.wrap_socket(raw_sock, server_hostname=BROKER_HOST)
            
            handshake_ms = (time.time() - start) * 1000
            
            # Extract connection info
            cipher = ssl_sock.cipher()
            version = ssl_sock.version()
            peer_cert = ssl_sock.getpeercert()
            
            ssl_sock.close()
            
            self.add_result('HANDSHAKE', 'SSL Connection', 'PASS',
                          f'Handshake completed in {handshake_ms:.1f}ms')
            print(f"  │  ✅ Handshake successful ({handshake_ms:.1f}ms)")
            
            # TLS version
            if version in ('TLSv1.2', 'TLSv1.3'):
                self.add_result('HANDSHAKE', 'TLS Version', 'PASS', version)
                print(f"  │  ✅ TLS Version: {version}")
            else:
                self.add_result('HANDSHAKE', 'TLS Version', 'WARN', version)
                print(f"  │  ⚠️  TLS Version: {version} (should be 1.2+)")
            
            # Cipher suite
            if cipher:
                cipher_name = cipher[0]
                cipher_bits = cipher[2]
                
                strong = cipher_bits >= 128
                status = 'PASS' if strong else 'WARN'
                icon = '✅' if strong else '⚠️'
                
                self.add_result('HANDSHAKE', 'Cipher Suite', status,
                              f'{cipher_name} ({cipher_bits}-bit)')
                print(f"  │  {icon} Cipher: {cipher_name} ({cipher_bits}-bit)")
            
            # Peer certificate details
            if peer_cert:
                subject = dict(x[0] for x in peer_cert.get('subject', []))
                issuer = dict(x[0] for x in peer_cert.get('issuer', []))
                not_after = peer_cert.get('notAfter', 'unknown')
                
                cn = subject.get('commonName', 'unknown')
                issuer_cn = issuer.get('commonName', 'unknown')
                
                print(f"  │")
                print(f"  │  Broker Certificate:")
                print(f"  │    Subject CN: {cn}")
                print(f"  │    Issuer CN:  {issuer_cn}")
                print(f"  │    Expires:    {not_after}")
                
                # Check SAN
                san_list = peer_cert.get('subjectAltName', [])
                if san_list:
                    sans = [f"{t}:{v}" for t, v in san_list]
                    print(f"  │    SAN:        {', '.join(sans)}")
                    self.add_result('CERTIFICATE', 'SAN', 'PASS',
                                  f'{len(sans)} entries')
                else:
                    print(f"  │    SAN:        NONE")
                    self.add_result('CERTIFICATE', 'SAN', 'WARN',
                                  'No SAN entries — hostname verification may fail')
                
                # Check expiration
                self._check_cert_expiration(not_after)
                
                self.add_result('CERTIFICATE', 'Broker Cert', 'PASS',
                              f'CN={cn}, Issuer={issuer_cn}')
        
        except python_ssl.SSLError as e:
            self.add_result('HANDSHAKE', 'SSL Connection', 'FAIL', str(e))
            print(f"  │  ❌ SSL Error: {e}")
        
        except python_ssl.SSLCertVerificationError as e:
            self.add_result('HANDSHAKE', 'Certificate Verification', 'FAIL', str(e))
            print(f"  │  ❌ Certificate verification failed: {e}")
        
        except FileNotFoundError as e:
            self.add_result('HANDSHAKE', 'Certificate Files', 'FAIL', str(e))
            print(f"  │  ❌ Certificate file not found: {e}")
        
        except Exception as e:
            self.add_result('HANDSHAKE', 'SSL Connection', 'FAIL', str(e))
            print(f"  │  ❌ Connection failed: {e}")
        
        print(f"  └─{'─' * 55}")
    
    def _check_cert_expiration(self, not_after_str):
        """Check how many days until certificate expires"""
        try:
            from datetime import datetime
            
            # Parse OpenSSL date format
            formats = [
                '%b %d %H:%M:%S %Y %Z',
                '%b  %d %H:%M:%S %Y %Z',
                '%Y-%m-%dT%H:%M:%S',
            ]
            
            expiry = None
            for fmt in formats:
                try:
                    expiry = datetime.strptime(not_after_str, fmt)
                    break
                except ValueError:
                    continue
            
            if expiry:
                days_remaining = (expiry - datetime.now()).days
                
                if days_remaining < 0:
                    self.add_result('CERTIFICATE', 'Expiration', 'FAIL',
                                  f'EXPIRED {abs(days_remaining)} days ago')
                    print(f"  │  ❌ Certificate EXPIRED {abs(days_remaining)} days ago!")
                elif days_remaining < 30:
                    self.add_result('CERTIFICATE', 'Expiration', 'WARN',
                                  f'{days_remaining} days remaining')
                    print(f"  │  ⚠️  Expires in {days_remaining} days — RENEW SOON")
                elif days_remaining < 90:
                    self.add_result('CERTIFICATE', 'Expiration', 'PASS',
                                  f'{days_remaining} days remaining')
                    print(f"  │  ✅ Expires in {days_remaining} days (plan renewal)")
                else:
                    self.add_result('CERTIFICATE', 'Expiration', 'PASS',
                                  f'{days_remaining} days remaining')
                    print(f"  │  ✅ Expires in {days_remaining} days")
        
        except Exception:
            pass

# FILE: cert_monitor.py

# Monitors certificate expiration dates across all components

import os
import subprocess
import time
from datetime import datetime, timedelta

ALERT_THRESHOLDS = {
    'critical_days': 7,
    'warning_days': 30,
    'info_days': 90,
}

class CertificateMonitor:
    
    def __init__(self, certs_base='./certs'):
        self.certs_base = certs_base
        self.certificates = []
    
    def discover_certificates(self):
        """Find all PEM certificate files"""
        
        cert_files = []
        
        for root, dirs, files in os.walk(self.certs_base):
            for filename in files:
                if filename.endswith(('.pem', '.crt', '.cer')):
                    filepath = os.path.join(root, filename)
                    
                    # Check if it's actually a certificate (not a key)
                    try:
                        with open(filepath, 'r') as f:
                            content = f.read()
                        
                        if '-----BEGIN CERTIFICATE-----' in content:
                            cert_files.append(filepath)
                    except Exception:
                        pass
        
        return cert_files
    
    def get_cert_info(self, cert_path):
        """Extract certificate details using openssl"""
        
        try:
            # Get subject
            result = subprocess.run(
                ['openssl', 'x509', '-in', cert_path, '-subject', '-noout'],
                capture_output=True, text=True, timeout=5
            )
            subject = result.stdout.strip().replace('subject=', '').strip()
            
            # Get issuer
            result = subprocess.run(
                ['openssl', 'x509', '-in', cert_path, '-issuer', '-noout'],
                capture_output=True, text=True, timeout=5
            )
            issuer = result.stdout.strip().replace('issuer=', '').strip()
            
            # Get dates
            result = subprocess.run(
                ['openssl', 'x509', '-in', cert_path, '-startdate', '-enddate', '-noout'],
                capture_output=True, text=True, timeout=5
            )
            
            dates = result.stdout.strip().split('\n')
            not_before = dates[0].replace('notBefore=', '').strip() if len(dates) > 0 else 'unknown'
            not_after = dates[1].replace('notAfter=', '').strip() if len(dates) > 1 else 'unknown'
            
            # Parse expiration date
            expiry = None
            for fmt in ['%b %d %H:%M:%S %Y %Z', '%b  %d %H:%M:%S %Y %Z']:
                try:
                    expiry = datetime.strptime(not_after, fmt)
                    break
                except ValueError:
                    continue
            
            days_remaining = (expiry - datetime.now()).days if expiry else -1
            
            # Get serial number
            result = subprocess.run(
                ['openssl', 'x509', '-in', cert_path, '-serial', '-noout'],
                capture_output=True, text=True, timeout=5
            )
            serial = result.stdout.strip().replace('serial=', '').strip()
            
            # Determine if CA or entity cert
            result = subprocess.run(
                ['openssl', 'x509', '-in', cert_path, '-text', '-noout'],
                capture_output=True, text=True, timeout=5
            )
            is_ca = 'CA:TRUE' in result.stdout
            
            return {
                'path': cert_path,
                'subject': subject,
                'issuer': issuer,
                'not_before': not_before,
                'not_after': not_after,
                'days_remaining': days_remaining,
                'serial': serial,
                'is_ca': is_ca,
                'type': 'CA' if is_ca else 'Entity',
            }
        
        except FileNotFoundError:
            return {
                'path': cert_path,
                'error': 'openssl not found',
                'days_remaining': -1,
            }
        except Exception as e:
            return {
                'path': cert_path,
                'error': str(e),
                'days_remaining': -1,
            }
    
    def check_all(self):
        """Check all discovered certificates"""
        
        print("=" * 70)
        print("  🔐 CERTIFICATE EXPIRATION MONITOR")
        print("=" * 70)
        print(f"  Scanning: {self.certs_base}")
        print(f"  Time: {datetime.now().strftime('%Y-%m-%d %H:%M:%S')}")
        print(f"  Alert thresholds:")
        print(f"    Critical: < {ALERT_THRESHOLDS['critical_days']} days")
        print(f"    Warning:  < {ALERT_THRESHOLDS['warning_days']} days")
        print(f"    Info:     < {ALERT_THRESHOLDS['info_days']} days")
        print("=" * 70)
        
        cert_files = self.discover_certificates()
        
        if not cert_files:
            print(f"\n  No certificate files found in {self.certs_base}")
            return
        
        print(f"\n  Found {len(cert_files)} certificate files")
        
        alerts = {'critical': [], 'warning': [], 'info': [], 'ok': []}
        
        for cert_path in sorted(cert_files):
            info = self.get_cert_info(cert_path)
            self.certificates.append(info)
            
            if 'error' in info:
                print(f"\n  ⚠️  {cert_path}: {info['error']}")
                continue
            
            days = info['days_remaining']
            relative_path = os.path.relpath(cert_path, self.certs_base)
            
            if days < 0:
                status = '💀 EXPIRED'
                alerts['critical'].append(info)
            elif days < ALERT_THRESHOLDS['critical_days']:
                status = '🔴 CRITICAL'
                alerts['critical'].append(info)
            elif days < ALERT_THRESHOLDS['warning_days']:
                status = '🟡 WARNING'
                alerts['warning'].append(info)
            elif days < ALERT_THRESHOLDS['info_days']:
                status = '🔵 INFO'
                alerts['info'].append(info)
            else:
                status = '🟢 OK'
                alerts['ok'].append(info)
            
            print(f"\n  ┌─ {relative_path}")
            print(f"  │  Type:    {info['type']}")
            print(f"  │  Subject: {info['subject']}")
            print(f"  │  Issuer:  {info['issuer']}")
            print(f"  │  Expires: {info['not_after']}")
            print(f"  │  Status:  {status} ({days} days remaining)")
            print(f"  └─{'─' * 55}")
        
        # Summary
        print(f"\n{'=' * 70}")
        print(f"  📊 CERTIFICATE STATUS SUMMARY")
        print(f"{'=' * 70}")
        print(f"  🟢 OK:       {len(alerts['ok'])}")
        print(f"  🔵 Info:     {len(alerts['info'])}")
        print(f"  🟡 Warning:  {len(alerts['warning'])}")
        print(f"  🔴 Critical: {len(alerts['critical'])}")
        
        if alerts['critical']:
            print(f"\n  🚨 IMMEDIATE ACTION REQUIRED:")
            for cert in alerts['critical']:
                days = cert['days_remaining']
                if days < 0:
                    print(f"     💀 {cert['subject']} — EXPIRED {abs(days)} days ago!")
                else:
                    print(f"     🔴 {cert['subject']} — expires in {days} days!")
        
        if alerts['warning']:
            print(f"\n  ⚠️  RENEWAL NEEDED SOON:")
            for cert in alerts['warning']:
                print(f"     🟡 {cert['subject']} — expires in {cert['days_remaining']} days")
        
        # Next expiration
        valid_certs = [c for c in self.certificates 
                      if 'days_remaining' in c and c['days_remaining'] >= 0]
        
        if valid_certs:
            soonest = min(valid_certs, key=lambda c: c['days_remaining'])
            print(f"\n  ⏰ Next expiration: {soonest['subject']} "
                  f"in {soonest['days_remaining']} days")
        
        print(f"{'=' * 70}")
        
        return alerts
