🏗️ HYPOTHETICAL PROBLEM

Your e-commerce platform processes 500,000 events/second during normal hours.
Every year, Black Friday hits and traffic spikes to 5,000,000 events/second

→ System must gracefully degrade — slow down, not crash

→ Traffic spikes must be absorbed, not rejected
→ Recovery from backlog must be controlled, not thundering-herd
→ All of this must work automatically — no 2 AM engineer scaling

🔍 HOW TO APPROACH IT

DECOMPOSE THE BACKPRESSURE PROBLEM:

Backpressure = when a downstream component cannot keep up
with the rate of data from an upstream component.

In Kafka, backpressure manifests at THREE levels:

Level 1 — PRODUCER → BROKER
├── Producer sends faster than broker can accept
├── Broker-side quota throttling (Project 42)
├── Producer buffer fills up → block or drop
└── Cross-region replication adds latency (Project 47)

Level 2 — BROKER → CONSUMER
├── Messages accumulate faster than consumer processes
├── Consumer lag grows continuously
├── Consumer fetches too many records → OOM
├── max.poll.interval.ms exceeded → rebalance (Project 10)
└── Rebalance during backlog = death spiral

Level 3 — CONSUMER → DOWNSTREAM SYSTEM
├── Consumer reads from Kafka fast but downstream is slow
├── Database insert rate < Kafka consumption rate
├── HTTP API call latency spikes under load
├── Consumer blocks on downstream → poll timeout → rebalance
└── Need to decouple consumption rate from processing rate

EACH LEVEL NEEDS DIFFERENT SOLUTIONS:

Level 1: Producer-side flow control
→ Buffering, compression, batching, blocking vs dropping

Level 2: Consumer-side flow control
→ pause()/resume(), max.poll.records, adaptive fetching

Level 3: Processing-side flow control
→ Internal queues, rate limiters, circuit breakers, spillover topics

WHAT MAKES THIS DIFFERENT FROM PREVIOUS PROJECTS:

→ Project 10: Covered rebalancing mechanics
→ Project 39: Covered performance tuning configs
→ Project 42: Covered quota management
→ Project 47: Created the DR failover burst scenario

→ THIS PROJECT: Combines all of them into a unified
  backpressure strategy that prevents system collapse

✅ WHAT COULD BE DONE

OPTION A: SCALE HORIZONTALLY (Brute Force)
├── Add more consumer instances to match producer rate
├── Simple but expensive — 10x consumers for 10x spike
├── Requires partition count >= consumer count
├── Scaling takes time — spike may pass before scale completes
└── Verdict: Necessary but not sufficient alone

OPTION B: SHED LOAD (Drop Messages)
├── Drop non-critical messages during overload
├── Fast but lossy — unacceptable for orders/payments
├── Works for analytics, logging, metrics
└── Verdict: Only for explicitly non-critical streams

OPTION C: ABSORB AND DEGRADE GRACEFULLY (Chosen)
├── Accept all messages into Kafka (Kafka handles this well)
├── Control consumption rate to match processing capacity
├── Prioritize critical partitions over non-critical
├── Spillover excess to retry topics for later processing
├── Auto-recover when spike subsides
└── Verdict: Best for mixed-criticality workloads

OPTION D: UPSTREAM RATE LIMITING (API Gateway)
├── Reject excess requests at the API layer
├── Users see 429 Too Many Requests
├── Protects everything downstream
├── Bad UX during Black Friday — customers leave
└── Verdict: Last resort — combine with Option C

WE CHOOSE: OPTION C as primary + OPTION A for critical paths
→ Graceful degradation for all consumers
→ Horizontal scaling for payment/order consumers specifically
→ Spillover topics for non-critical processing
→ Never drop, never crash, never OOM

✅ WHAT SHOULD DO

1. IMPLEMENT CONSUMER pause()/resume() FLOW CONTROL
   → Kafka's built-in mechanism for per-partition flow control
   → Pause partitions when internal queue is full
   → Resume when queue drains below threshold
   → Consumer keeps calling poll() to avoid rebalance

2. USE ADAPTIVE max.poll.records
   → Start with conservative value (100)
   → Increase when processing is fast
   → Decrease when processing is slow or memory is high
   → Never fetch more than you can process in max.poll.interval.ms

3. IMPLEMENT BOUNDED INTERNAL WORK QUEUE
   → Decouple fetching from processing
   → Fixed-size queue between consumer thread and worker threads
   → Queue full → pause partition
   → Queue drains → resume partition

4. ADD CIRCUIT BREAKER FOR DOWNSTREAM SYSTEMS
   → Database overloaded → open circuit → pause consumption
   → Database recovers → close circuit → resume consumption
   → Prevents cascading failures

5. CREATE SPILLOVER TOPICS FOR EXCESS TRAFFIC
   → Normal path: consume → process → commit
   → Overloaded path: consume → spillover topic → commit
   → Background job processes spillover when load drops
   → No data loss, just delayed processing

6. IMPLEMENT PRODUCER-SIDE RATE LIMITING
   → Token bucket algorithm for production rate
   → Prevents producer from overwhelming broker
   → Adaptive: increase bucket during low traffic, decrease during high

7. MONITOR BUFFER MEMORY USAGE ON PRODUCER
   → buffer.memory usage > 80% = warning
   → buffer.memory usage > 95% = critical backoff
   → Prevents producer blocking on send()

8. USE LAG-BASED PRIORITY SYSTEM
   → Monitor lag per partition
   → High-priority partitions (payments) always processed
   → Low-priority partitions (analytics) paused during overload
   → Resume low-priority when high-priority lag is acceptable

❌ WHAT SHOULD NOT DO

1. DO NOT SET max.poll.records TOO HIGH
   → 10,000 records per poll during spike = OOM
   → Processing 10,000 records may exceed max.poll.interval.ms
   → Consumer gets kicked from group → rebalance → death spiral
   → Start with 100, increase carefully

2. DO NOT RELY ON max.poll.interval.ms ALONE
   → Default 5 minutes is too generous
   → Consumer appears "alive" but is actually stuck processing
   → Other consumers can't get those partitions
   → Use shorter interval + pause/resume instead

3. DO NOT FETCH UNLIMITED DATA WITH fetch.max.bytes
   → Default 50MB per fetch can overwhelm consumer memory
   → During backlog: every fetch returns maximum data
   → Reduce to match processing capacity

4. DO NOT BLOCK IN poll() CALLBACK
   → Long-running processing inside poll loop
   → Blocks heartbeat → session timeout → rebalance
   → Use separate worker threads for processing

5. DO NOT IGNORE PRODUCER BUFFER METRICS
   → Producer silently blocks when buffer is full
   → Looks like "slow producing" but actually backpressure
   → Monitor buffer-available-bytes metric

6. DO NOT USE UNLIMITED RETRIES WITHOUT BACKOFF
   → Retry storms amplify traffic during overload
   → Each retry adds more load to already-overloaded system
   → Exponential backoff (Project 26) is essential

7. DO NOT PROCESS BACKLOG AT FULL SPEED
   → After outage: consumer has 10M record backlog
   → Processing at full speed = overwhelming downstream
   → Use controlled catch-up rate (e.g., 2x normal throughput)
   → Gradually increase as downstream proves stable

8. DO NOT MIX CRITICAL AND NON-CRITICAL IN SAME CONSUMER GROUP
   → During overload, all partitions compete for same resources
   → Critical payment events delayed by analytics processing
   → Separate consumer groups for different priority levels

🚫 WHEN NOT TO USE

1. LOW-LATENCY STREAM PROCESSING (< 10ms)
   → pause()/resume() adds latency (poll cycle delay)
   → Bounded queue adds latency (queue wait time)
   → For ultra-low latency: over-provision resources instead
   → Accept higher cost for guaranteed low latency

2. EXACTLY-ONCE STREAMS WITH TIGHT WINDOWS (Project 43)
   → Pausing partitions disrupts windowed operations (Project 22)
   → Window grace periods may expire during pause
   → State stores may grow unbounded during pause
   → Better: scale Kafka Streams instances instead

3. WHEN DATA FRESHNESS IS CRITICAL
   → Spillover topics add processing delay
   → Pause/resume adds consumption delay
   → For real-time dashboards: over-provision or sample

4. SIMPLE SINGLE-PARTITION TOPICS
   → Can't pause individual partitions if there's only one
   → Pause = stop everything
   → For single-partition: use internal queue only

5. WHEN PRODUCERS CAN BE RATE-LIMITED AT SOURCE
   → If API gateway can reject excess traffic
   → Backpressure in Kafka is unnecessary
   → Push rate limiting as far upstream as possible

⚙️ HOW IT ONLY WORKS (INTERNAL MECHANICS)

KAFKA CONSUMER pause()/resume() INTERNALS:

The most important backpressure mechanism in Kafka.
Understanding exactly how it works:

NORMAL POLL CYCLE:
┌──────────────────────────────────────────────────────────┐
│  consumer.poll(timeout=1000)                             │
│                                                          │
│  1. Send heartbeat to group coordinator                  │
│  2. Fetch data from ALL assigned partitions              │
│  3. Return records from ALL partitions                   │
│  4. Application processes records                        │
│  5. consumer.commit()                                    │
│  6. Back to step 1                                       │
└──────────────────────────────────────────────────────────┘

WITH pause() ACTIVE:
┌──────────────────────────────────────────────────────────┐
│  consumer.pause([TopicPartition("orders", 2)])           │
│                                                          │
│  Internally:                                             │
│  → Partition orders:2 added to "paused" set              │
│  → NO network call — purely client-side state            │
│  → Partition is still ASSIGNED to this consumer          │
│  → No rebalance triggered                                │
│                                                          │
│  Next consumer.poll(timeout=1000):                       │
│  1. Send heartbeat ✓ (keeps consumer in group)           │
│  2. Fetch from orders:0, orders:1 (NOT orders:2)         │
│  3. Return records from non-paused partitions only       │
│  4. Application processes reduced record set             │
│  5. consumer.commit()                                    │
│  6. Back to step 1                                       │
│                                                          │
│  KEY: Consumer stays in group, heartbeat continues,      │
│  but paused partitions are not fetched.                   │
│  No data loss — records accumulate in Kafka.             │
└──────────────────────────────────────────────────────────┘

resume() RESTORES NORMAL FETCHING:
┌──────────────────────────────────────────────────────────┐
│  consumer.resume([TopicPartition("orders", 2)])          │
│                                                          │
│  → Partition removed from "paused" set                   │
│  → Next poll() will fetch from orders:2 again            │
│  → Resumes from last committed offset                    │
│  → Accumulated records are fetched in subsequent polls   │
└──────────────────────────────────────────────────────────┘


WHY pause()/resume() IS BETTER THAN ALTERNATIVES:

Alternative 1: Reduce poll frequency (sleep between polls)
→ Problem: Heartbeat is sent during poll()
→ If sleep > session.timeout.ms → consumer kicked → rebalance
→ pause() keeps heartbeat alive while stopping data fetch

Alternative 2: Reduce max.poll.records to 1
→ Problem: Still fetches from ALL partitions
→ Can't prioritize critical partitions over non-critical
→ pause() gives per-partition granularity

Alternative 3: Stop calling poll() entirely
→ Problem: max.poll.interval.ms expires → rebalance
→ Rebalance during overload = cascading failure
→ pause() + continue poll() = no rebalance risk

WHAT HAPPENS DURING BROKER THROTTLING:

Broker quota (Project 42): producer_byte_rate = 10MB/s

Producer sends at 15MB/s:
1. Broker accepts first 10MB
2. Broker returns ThrottleTime in response header
3. Client respects ThrottleTime → delays next send
4. Effective rate drops to 10MB/s
5. Producer buffer starts filling (producing 15, sending 10)
6. Eventually buffer fills → send() blocks
7. Application thread blocks on send() → natural backpressure

THIS IS HEALTHY BACKPRESSURE — system self-regulates.


CONSUMER-SIDE MEMORY MECHANICS:

Why consumers OOM during backlog:

Normal: Consumer fetches 500 records, processes in 2 seconds
Backlog: Consumer fetches 500 records, each is 10KB = 5MB per poll
         Process + in-memory state + deserialization overhead
         With 12 partitions: 60MB per poll cycle
         Add business objects, caches, framework overhead
         JVM heap usage spikes to 80%+

With max.poll.records = 10000 during backlog:
         100MB per poll cycle just for raw records
         Plus deserialization: 200MB+
         Plus business logic state: 300MB+
         JVM: OOM → kill → rebalance → repeat

SOLUTION: Adaptive max.poll.records
→ Monitor JVM heap usage
→ heap > 70%: reduce max.poll.records by 50%
→ heap < 40%: increase max.poll.records by 25%
→ Never exceed safe maximum (e.g., 500)
→ Never go below minimum (e.g., 10)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: PAUSE WITHOUT CONTINUED POLLING
├── Problem: Developer pauses partitions and stops calling poll()
├── Impact: max.poll.interval.ms expires → rebalance
├── Symptom: Consumer keeps getting kicked from group
├── Solution: ALWAYS call poll() even when all partitions paused
└── poll() with all paused returns empty — but sends heartbeat

PITFALL 2: COMMIT BEFORE PROCESSING COMPLETE
├── Problem: Commit offsets then process — crash loses data
├── Impact: Messages silently lost during backpressure events
├── Symptom: Data gaps discovered hours later
├── Solution: Process → commit (at-least-once) with idempotent sink
└── Reference: Project 11 delivery guarantees

PITFALL 3: SPILLOVER TOPIC GROWS UNBOUNDED
├── Problem: Spillover rate > background processing rate
├── Impact: Spillover topic fills disk, broker runs out of storage
├── Symptom: Broker log.retention triggers → spillover data deleted
├── Solution: Monitor spillover lag, alert when growing
└── Set retention on spillover topic to match SLA

PITFALL 4: CIRCUIT BREAKER NEVER CLOSES
├── Problem: Downstream is "partially" healthy
├── Impact: Circuit breaker stays open → consumption stops forever
├── Symptom: Consumer lag grows indefinitely, no processing
├── Solution: Half-open state: periodically test downstream
└── Time-based forced close with health check

PITFALL 5: RATE LIMITER TOO AGGRESSIVE
├── Problem: Rate limit set for normal traffic, not spike
├── Impact: Producer drops/blocks legitimate traffic during spike
├── Symptom: Data loss or extreme latency during Black Friday
├── Solution: Adaptive rate limits based on current conditions
└── Set rate limit to 2-3x normal, not 1x

PITFALL 6: PAUSE/RESUME THRASHING
├── Problem: Queue oscillates around threshold rapidly
├── Impact: Partitions paused/resumed every poll cycle
├── Symptom: Inconsistent throughput, high CPU from state changes
├── Solution: Hysteresis — pause at 80% full, resume at 40% full
└── Different thresholds for pause and resume

PITFALL 7: REBALANCE DURING BACKPRESSURE
├── Problem: New consumer joins during overload → rebalance
├── Impact: All consumers stop processing during rebalance
├── Symptom: Brief total stop + offset re-fetch + more memory pressure
├── Solution: Use CooperativeStickyAssignor (Project 10)
└── Incremental rebalance minimizes disruption

PITFALL 8: IGNORING PRODUCER CALLBACK ERRORS
├── Problem: Producer send() callback has error during throttling
├── Impact: Messages silently lost
├── Symptom: Data gaps in topics
├── Solution: Always check callback errors, retry on retriable errors
└── Monitor producer error rate metrics

PARTITION COUNT REQUIREMENTS:
→ Consumer count ≤ partition count (Project 5)
→ For 12 consumers: need ≥ 12 partitions
→ Plan partitions for PEAK consumer count, not normal
→ Cannot add partitions without key distribution change (Project 4)

MEMORY SCALING PER CONSUMER:
→ Base memory: 512MB
→ Per-partition fetch buffer: ~1MB × partitions assigned
→ Internal work queue: bounded_size × avg_message_size
→ Processing overhead: depends on business logic
→ Formula: 512MB + (partitions × 1MB) + (queue_size × msg_size) + 30% headroom

SPILLOVER TOPIC SIZING:
→ Capacity: max_spike_rate × max_spike_duration × avg_message_size
→ Example: 5M/s × 18 hours × 1KB = 324TB (!!!)
→ Reality: spillover captures overflow, not full traffic
→ Realistic: 10% of spike = 32.4TB
→ Set retention to match processing SLA (e.g., 48 hours)

PRODUCER BUFFER SIZING:
→ Default 32MB is sufficient for most cases
→ During broker throttling: buffer fills at (produce_rate - broker_rate)
→ Example: producing 15MB/s, broker accepts 10MB/s
→ Buffer fills at 5MB/s → 32MB lasts ~6 seconds
→ For longer throttle tolerance: increase buffer.memory
→ 128MB buffer → 25 seconds of throttle absorption

# common/rate_limiter.py
# ─────────────────────────────────────────────────────────────
# Token bucket rate limiter for controlling production/consumption rate
# Used by both producers and consumers for flow control
#
# How token bucket works:
#   - Bucket holds tokens (max = burst_size)
#   - Tokens added at fixed rate (tokens_per_second)
#   - Each operation consumes 1 token
#   - If no tokens available: wait or reject
#   - Allows bursts up to burst_size, then limits to steady rate
# ─────────────────────────────────────────────────────────────

import threading
import time
import logging
from dataclasses import dataclass
from typing import Optional

logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger("rate-limiter")


@dataclass
class RateLimiterStats:
    """Statistics for rate limiter observability."""
    total_acquired: int = 0
    total_rejected: int = 0
    total_waited_ms: float = 0.0
    current_tokens: float = 0.0
    current_rate: float = 0.0

    def to_dict(self) -> dict:
        return {
            "total_acquired": self.total_acquired,
            "total_rejected": self.total_rejected,
            "total_waited_ms": round(self.total_waited_ms, 2),
            "current_tokens": round(self.current_tokens, 2),
            "current_rate": round(self.current_rate, 2),
        }


class TokenBucketRateLimiter:
    """
    Thread-safe token bucket rate limiter with adaptive rate support.

    Usage:
        limiter = TokenBucketRateLimiter(rate=1000, burst=2000)

        # Blocking — waits until token available
        limiter.acquire()
        producer.send(...)

        # Non-blocking — returns immediately
        if limiter.try_acquire():
            producer.send(...)
        else:
            handle_rate_limited()

        # Adaptive — change rate at runtime
        limiter.set_rate(5000)  # Increase during off-peak
    """

    def __init__(
        self,
        rate: float,
        burst: Optional[int] = None,
        name: str = "default"
    ):
        """
        Args:
            rate: Tokens per second (sustained throughput)
            burst: Maximum tokens (burst capacity). Default = 2x rate
            name: Identifier for logging
        """
        self.name = name
        self._rate = rate
        self._burst = burst or int(rate * 2)
        self._tokens = float(self._burst)  # Start full
        self._last_refill = time.monotonic()
        self._lock = threading.Lock()
        self._stats = RateLimiterStats()

        logger.info(
            f"RateLimiter '{name}' created: "
            f"rate={rate}/s, burst={self._burst}"
        )

    def _refill(self):
        """Add tokens based on elapsed time since last refill."""
        now = time.monotonic()
        elapsed = now - self._last_refill
        new_tokens = elapsed * self._rate
        self._tokens = min(self._burst, self._tokens + new_tokens)
        self._last_refill = now
        self._stats.current_tokens = self._tokens

    def acquire(self, timeout: float = 30.0) -> bool:
        """
        Acquire a token, blocking until available or timeout.

        Returns:
            True if token acquired, False if timeout exceeded
        """
        deadline = time.monotonic() + timeout

        while True:
            with self._lock:
                self._refill()

                if self._tokens >= 1.0:
                    self._tokens -= 1.0
                    self._stats.total_acquired += 1
                    self._stats.current_tokens = self._tokens
                    return True

                # Calculate wait time for next token
                wait_time = (1.0 - self._tokens) / self._rate

            # Check timeout
            remaining = deadline - time.monotonic()
            if remaining <= 0:
                with self._lock:
                    self._stats.total_rejected += 1
                return False

            # Sleep for minimum of wait_time and remaining timeout
            sleep_time = min(wait_time, remaining, 0.01)  # Max 10ms sleep
            self._stats.total_waited_ms += sleep_time * 1000
            time.sleep(sleep_time)

    def try_acquire(self, count: int = 1) -> bool:
        """
        Try to acquire token(s) without blocking.

        Returns:
            True if token(s) acquired, False if not available
        """
        with self._lock:
            self._refill()

            if self._tokens >= count:
                self._tokens -= count
                self._stats.total_acquired += count
                self._stats.current_tokens = self._tokens
                return True

            self._stats.total_rejected += count
            return False

    def set_rate(self, new_rate: float):
        """
        Dynamically adjust the rate limit.
        Used for adaptive rate limiting based on system pressure.
        """
        with self._lock:
            self._refill()  # Settle current state before changing rate
            old_rate = self._rate
            self._rate = new_rate
            self._stats.current_rate = new_rate

        logger.info(
            f"RateLimiter '{self.name}' rate changed: "
            f"{old_rate}/s → {new_rate}/s"
        )

    def set_burst(self, new_burst: int):
        """Dynamically adjust burst capacity."""
        with self._lock:
            self._burst = new_burst
            self._tokens = min(self._tokens, float(new_burst))

    @property
    def stats(self) -> RateLimiterStats:
        with self._lock:
            self._refill()
            return RateLimiterStats(
                total_acquired=self._stats.total_acquired,
                total_rejected=self._stats.total_rejected,
                total_waited_ms=self._stats.total_waited_ms,
                current_tokens=self._tokens,
                current_rate=self._rate,
            )

    @property
    def utilization(self) -> float:
        """How much of the rate limit is being used (0.0 to 1.0)."""
        with self._lock:
            self._refill()
            return 1.0 - (self._tokens / self._burst)


class AdaptiveRateLimiter:
    """
    Rate limiter that automatically adjusts based on system metrics.

    Watches a pressure_fn callback and adjusts rate accordingly:
    - Low pressure (<40%): increase rate toward max
    - Medium pressure (40-70%): maintain current rate
    - High pressure (70-90%): decrease rate
    - Critical pressure (>90%): decrease rate aggressively
    """

    def __init__(
        self,
        base_rate: float,
        min_rate: float,
        max_rate: float,
        pressure_fn: callable,
        adjustment_interval: float = 5.0,
        name: str = "adaptive"
    ):
        self.base_rate = base_rate
        self.min_rate = min_rate
        self.max_rate = max_rate
        self.pressure_fn = pressure_fn
        self.adjustment_interval = adjustment_interval

        self.limiter = TokenBucketRateLimiter(
            rate=base_rate,
            burst=int(max_rate * 2),
            name=name
        )

        self._running = True
        self._adjuster_thread = threading.Thread(
            target=self._adjustment_loop,
            daemon=True,
            name=f"rate-adjuster-{name}"
        )
        self._adjuster_thread.start()

    def _adjustment_loop(self):
        """Periodically adjust rate based on pressure."""
        while self._running:
            try:
                pressure = self.pressure_fn()
                current_rate = self.limiter._rate

                if pressure < 0.4:
                    # Low pressure: increase rate by 20%
                    new_rate = min(self.max_rate, current_rate * 1.2)
                elif pressure < 0.7:
                    # Medium pressure: hold steady
                    new_rate = current_rate
                elif pressure < 0.9:
                    # High pressure: decrease by 30%
                    new_rate = max(self.min_rate, current_rate * 0.7)
                else:
                    # Critical: decrease by 50%
                    new_rate = max(self.min_rate, current_rate * 0.5)

                if abs(new_rate - current_rate) / max(current_rate, 1) > 0.05:
                    self.limiter.set_rate(new_rate)
                    logger.info(
                        f"Adaptive rate adjustment: pressure={pressure:.1%}, "
                        f"rate={current_rate:.0f} → {new_rate:.0f}/s"
                    )

            except Exception as e:
                logger.warning(f"Rate adjustment error: {e}")

            time.sleep(self.adjustment_interval)

    def acquire(self, timeout: float = 30.0) -> bool:
        return self.limiter.acquire(timeout)

    def try_acquire(self, count: int = 1) -> bool:
        return self.limiter.try_acquire(count)

    @property
    def stats(self) -> RateLimiterStats:
        return self.limiter.stats

    def stop(self):
        self._running = False

# common/circuit_breaker.py
# ─────────────────────────────────────────────────────────────
# Circuit breaker for downstream system protection
# Prevents consumer from sending traffic to unhealthy downstream
#
# States:
#   CLOSED  → Normal operation, requests flow through
#   OPEN    → Downstream is down, all requests rejected immediately
#   HALF_OPEN → Testing if downstream recovered
# ─────────────────────────────────────────────────────────────

import threading
import time
import logging
from enum import Enum
from dataclasses import dataclass, field
from typing import Callable, Optional

logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger("circuit-breaker")


class CircuitState(Enum):
    CLOSED = "closed"
    OPEN = "open"
    HALF_OPEN = "half_open"


@dataclass
class CircuitBreakerStats:
    """Circuit breaker statistics."""
    state: CircuitState = CircuitState.CLOSED
    total_calls: int = 0
    successful_calls: int = 0
    failed_calls: int = 0
    rejected_calls: int = 0
    consecutive_failures: int = 0
    last_failure_time: Optional[float] = None
    last_success_time: Optional[float] = None
    state_transitions: list = field(default_factory=list)


class CircuitBreaker:
    """
    Circuit breaker pattern implementation.

    Usage:
        cb = CircuitBreaker(
            failure_threshold=5,
            recovery_timeout=30,
            half_open_max_calls=3,
            name="database"
        )

        if cb.can_execute():
            try:
                result = database.insert(record)
                cb.record_success()
            except Exception as e:
                cb.record_failure()
        else:
            # Circuit is open — don't even try
            spillover_queue.put(record)
    """

    def __init__(
        self,
        failure_threshold: int = 5,
        recovery_timeout: float = 30.0,
        half_open_max_calls: int = 3,
        success_threshold: int = 3,
        name: str = "default"
    ):
        """
        Args:
            failure_threshold: Consecutive failures to trip circuit
            recovery_timeout: Seconds to wait before trying again
            half_open_max_calls: Max concurrent calls in half-open state
            success_threshold: Consecutive successes to close circuit
            name: Identifier for logging
        """
        self.failure_threshold = failure_threshold
        self.recovery_timeout = recovery_timeout
        self.half_open_max_calls = half_open_max_calls
        self.success_threshold = success_threshold
        self.name = name

        self._state = CircuitState.CLOSED
        self._lock = threading.Lock()
        self._stats = CircuitBreakerStats()
        self._consecutive_failures = 0
        self._consecutive_successes = 0
        self._last_failure_time: Optional[float] = None
        self._half_open_calls = 0

        # Callbacks for state change notifications
        self._on_state_change: Optional[Callable] = None

        logger.info(
            f"CircuitBreaker '{name}' created: "
            f"threshold={failure_threshold}, "
            f"recovery={recovery_timeout}s"
        )

    @property
    def state(self) -> CircuitState:
        with self._lock:
            # Check if we should transition from OPEN to HALF_OPEN
            if self._state == CircuitState.OPEN:
                if self._last_failure_time is not None:
                    elapsed = time.monotonic() - self._last_failure_time
                    if elapsed >= self.recovery_timeout:
                        self._transition_to(CircuitState.HALF_OPEN)
            return self._state

    def can_execute(self) -> bool:
        """Check if a call is allowed through the circuit."""
        current_state = self.state  # This checks for OPEN → HALF_OPEN transition

        with self._lock:
            if current_state == CircuitState.CLOSED:
                self._stats.total_calls += 1
                return True

            elif current_state == CircuitState.HALF_OPEN:
                if self._half_open_calls < self.half_open_max_calls:
                    self._half_open_calls += 1
                    self._stats.total_calls += 1
                    return True
                else:
                    self._stats.rejected_calls += 1
                    return False

            else:  # OPEN
                self._stats.rejected_calls += 1
                return False

    def record_success(self):
        """Record a successful call."""
        with self._lock:
            self._stats.successful_calls += 1
            self._stats.last_success_time = time.monotonic()
            self._consecutive_failures = 0
            self._consecutive_successes += 1

            if self._state == CircuitState.HALF_OPEN:
                if self._consecutive_successes >= self.success_threshold:
                    self._transition_to(CircuitState.CLOSED)

    def record_failure(self):
        """Record a failed call."""
        with self._lock:
            self._stats.failed_calls += 1
            self._stats.last_failure_time = time.monotonic()
            self._last_failure_time = time.monotonic()
            self._consecutive_failures += 1
            self._consecutive_successes = 0
            self._stats.consecutive_failures = self._consecutive_failures

            if self._state == CircuitState.CLOSED:
                if self._consecutive_failures >= self.failure_threshold:
                    self._transition_to(CircuitState.OPEN)

            elif self._state == CircuitState.HALF_OPEN:
                # Any failure in half-open → back to open
                self._transition_to(CircuitState.OPEN)

    def _transition_to(self, new_state: CircuitState):
        """Execute state transition."""
        old_state = self._state
        self._state = new_state
        self._stats.state = new_state

        if new_state == CircuitState.CLOSED:
            self._consecutive_failures = 0
            self._half_open_calls = 0
        elif new_state == CircuitState.HALF_OPEN:
            self._half_open_calls = 0
            self._consecutive_successes = 0
        elif new_state == CircuitState.OPEN:
            self._half_open_calls = 0

        transition = {
            "from": old_state.value,
            "to": new_state.value,
            "time": time.time(),
        }
        self._stats.state_transitions.append(transition)

        logger.warning(
            f"CircuitBreaker '{self.name}': "
            f"{old_state.value} → {new_state.value}"
        )

        if self._on_state_change:
            try:
                self._on_state_change(old_state, new_state)
            except Exception:
                pass

    def on_state_change(self, callback: Callable):
        """Register callback for state transitions."""
        self._on_state_change = callback

    @property
    def is_open(self) -> bool:
        return self.state == CircuitState.OPEN

    @property
    def is_closed(self) -> bool:
        return self.state == CircuitState.CLOSED

    def get_stats(self) -> dict:
        with self._lock:
            return {
                "name": self.name,
                "state": self._state.value,
                "total_calls": self._stats.total_calls,
                "successful_calls": self._stats.successful_calls,
                "failed_calls": self._stats.failed_calls,
                "rejected_calls": self._stats.rejected_calls,
                "consecutive_failures": self._consecutive_failures,
                "transitions": len(self._stats.state_transitions),
            }

    def force_open(self):
        """Manually trip the circuit (for testing/maintenance)."""
        with self._lock:
            self._transition_to(CircuitState.OPEN)

    def force_close(self):
        """Manually close the circuit (for recovery)."""
        with self._lock:
            self._transition_to(CircuitState.CLOSED)

# common/pressure_monitor.py
# ─────────────────────────────────────────────────────────────
# Central pressure monitoring system
# Aggregates pressure signals from producer, consumer, downstream
# Provides unified pressure score for adaptive decisions
# ─────────────────────────────────────────────────────────────

import threading
import time
import logging
import psutil
from dataclasses import dataclass
from typing import Optional, Callable

logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger("pressure-monitor")


@dataclass
class PressureReading:
    """Single pressure measurement."""
    timestamp: float
    producer_pressure: float      # 0.0 to 1.0 (buffer usage)
    consumer_pressure: float      # 0.0 to 1.0 (lag / threshold)
    downstream_pressure: float    # 0.0 to 1.0 (response time / SLA)
    memory_pressure: float        # 0.0 to 1.0 (heap usage)
    overall_pressure: float       # 0.0 to 1.0 (weighted composite)

    def to_dict(self) -> dict:
        return {
            "producer": round(self.producer_pressure, 3),
            "consumer": round(self.consumer_pressure, 3),
            "downstream": round(self.downstream_pressure, 3),
            "memory": round(self.memory_pressure, 3),
            "overall": round(self.overall_pressure, 3),
        }


class PressureMonitor:
    """
    Monitors system pressure from multiple sources.

    Pressure sources:
    1. Producer buffer usage (how full is the send buffer)
    2. Consumer lag (how far behind are consumers)
    3. Downstream latency (how slow is the sink)
    4. JVM/process memory usage

    Each source contributes to an overall pressure score
    used by adaptive components (rate limiter, poller, etc.)
    """

    def __init__(
        self,
        consumer_lag_threshold: int = 100000,
        downstream_latency_sla_ms: float = 500.0,
        memory_threshold_pct: float = 0.80,
        weights: Optional[dict] = None
    ):
        self.consumer_lag_threshold = consumer_lag_threshold
        self.downstream_latency_sla_ms = downstream_latency_sla_ms
        self.memory_threshold_pct = memory_threshold_pct

        # Weights for overall pressure calculation
        self.weights = weights or {
            "producer": 0.15,
            "consumer": 0.35,
            "downstream": 0.30,
            "memory": 0.20,
        }

        # Current values (updated by external components)
        self._lock = threading.Lock()
        self._producer_buffer_pct: float = 0.0
        self._consumer_lag: int = 0
        self._downstream_latency_ms: float = 0.0
        self._internal_queue_fill_pct: float = 0.0

        # History for trend detection
        self._history: list[PressureReading] = []
        self._max_history = 120  # 2 minutes at 1s intervals

    def update_producer_buffer(self, used_bytes: int, total_bytes: int):
        """Update producer buffer usage metric."""
        with self._lock:
            if total_bytes > 0:
                self._producer_buffer_pct = used_bytes / total_bytes
            else:
                self._producer_buffer_pct = 0.0

    def update_consumer_lag(self, total_lag: int):
        """Update total consumer lag across all partitions."""
        with self._lock:
            self._consumer_lag = total_lag

    def update_downstream_latency(self, latency_ms: float):
        """Update downstream processing latency."""
        with self._lock:
            self._downstream_latency_ms = latency_ms

    def update_queue_fill(self, fill_pct: float):
        """Update internal work queue fill percentage."""
        with self._lock:
            self._internal_queue_fill_pct = fill_pct

    def get_reading(self) -> PressureReading:
        """Calculate current pressure reading from all sources."""
        with self._lock:
            # Producer pressure: buffer fill percentage
            producer_pressure = min(1.0, self._producer_buffer_pct)

            # Consumer pressure: lag relative to threshold
            consumer_pressure = min(
                1.0,
                self._consumer_lag / max(1, self.consumer_lag_threshold)
            )

            # Downstream pressure: latency relative to SLA
            downstream_pressure = min(
                1.0,
                self._downstream_latency_ms / max(1, self.downstream_latency_sla_ms)
            )

            # Memory pressure: process memory usage
            try:
                process = psutil.Process()
                memory_info = process.memory_info()
                # Use RSS relative to available system memory
                total_memory = psutil.virtual_memory().total
                memory_pressure = min(
                    1.0,
                    memory_info.rss / (total_memory * self.memory_threshold_pct)
                )
            except Exception:
                memory_pressure = 0.0

            # Weighted overall pressure
            overall = (
                self.weights["producer"] * producer_pressure
                + self.weights["consumer"] * consumer_pressure
                + self.weights["downstream"] * downstream_pressure
                + self.weights["memory"] * memory_pressure
            )
            overall = min(1.0, overall)

            reading = PressureReading(
                timestamp=time.time(),
                producer_pressure=producer_pressure,
                consumer_pressure=consumer_pressure,
                downstream_pressure=downstream_pressure,
                memory_pressure=memory_pressure,
                overall_pressure=overall,
            )

            self._history.append(reading)
            if len(self._history) > self._max_history:
                self._history.pop(0)

            return reading

    @property
    def overall_pressure(self) -> float:
        """Quick access to current overall pressure (0.0 to 1.0)."""
        return self.get_reading().overall_pressure

    def get_trend(self, window_seconds: int = 30) -> str:
        """
        Determine pressure trend over recent window.
        Returns: "increasing", "stable", "decreasing"
        """
        with self._lock:
            if len(self._history) < 10:
                return "stable"

            cutoff = time.time() - window_seconds
            recent = [r for r in self._history if r.timestamp >= cutoff]

            if len(recent) < 5:
                return "stable"

            first_half = recent[:len(recent) // 2]
            second_half = recent[len(recent) // 2:]

            avg_first = sum(r.overall_pressure for r in first_half) / len(first_half)
            avg_second = sum(r.overall_pressure for r in second_half) / len(second_half)

            diff = avg_second - avg_first

            if diff > 0.1:
                return "increasing"
            elif diff < -0.1:
                return "decreasing"
            else:
                return "stable"

    def get_pressure_level(self) -> str:
        """Get human-readable pressure level."""
        pressure = self.overall_pressure
        if pressure < 0.4:
            return "normal"
        elif pressure < 0.7:
            return "elevated"
        elif pressure < 0.9:
            return "high"
        else:
            return "critical"

from typing import Optional, Set
from collections import defaultdict

from confluent_kafka import Consumer, TopicPartition

logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger("flow-controller")


class PartitionPriority(Enum):
    """Priority levels for partitions."""
    CRITICAL = 1     # Never pause (payments)
    HIGH = 2         # Pause only under extreme pressure
    NORMAL = 3       # Pause under high pressure
    LOW = 4          # First to pause under any pressure
    BACKGROUND = 5   # Pause whenever system is not idle


@dataclass
class PartitionState:
    """Tracks state for a single partition."""
    topic_partition: TopicPartition
    priority: PartitionPriority
    is_paused: bool = False
    queue_fill_pct: float = 0.0
    last_pause_time: Optional[float] = None
    last_resume_time: Optional[float] = None
    pause_count: int = 0
    messages_since_resume: int = 0


class PartitionFlowController:
    """
    Controls per-partition consumption using pause()/resume().

    HYSTERESIS DESIGN (prevents thrashing):
    ─────────────────────────────────────
    Partition is PAUSED when:
      → queue_fill_pct > PAUSE_THRESHOLD (80%)
      → AND system pressure warrants it for this priority level

    Partition is RESUMED when:
      → queue_fill_pct < RESUME_THRESHOLD (40%)
      → AND minimum pause duration has elapsed (5 seconds)

    Gap between 80% and 40% prevents rapid pause/resume oscillation.

    PRIORITY-BASED PAUSING:
    ─────────────────────────
    Under increasing pressure, partitions pause in this order:
      BACKGROUND → LOW → NORMAL → HIGH
      CRITICAL partitions are NEVER paused.

    Under decreasing pressure, partitions resume in reverse order:
      HIGH → NORMAL → LOW → BACKGROUND
    """

    # Thresholds with hysteresis gap
    PAUSE_THRESHOLD = 0.80     # Pause when queue > 80% full
    RESUME_THRESHOLD = 0.40    # Resume when queue < 40% full
    MIN_PAUSE_DURATION = 5.0   # Minimum seconds a partition stays paused

    # Pressure thresholds per priority level
    # "At what overall pressure level should this priority start pausing?"
    PRIORITY_PRESSURE_THRESHOLDS = {
        PartitionPriority.CRITICAL: 999.0,    # Never pause
        PartitionPriority.HIGH: 0.90,         # Only under extreme pressure
        PartitionPriority.NORMAL: 0.70,       # Under high pressure
        PartitionPriority.LOW: 0.50,          # Under moderate pressure
        PartitionPriority.BACKGROUND: 0.30,   # Under any elevated pressure
    }

    def __init__(
        self,
        consumer: Consumer,
        pressure_monitor,
        priority_map: Optional[dict] = None
    ):
        """
        Args:
            consumer: Kafka Consumer instance
            pressure_monitor: PressureMonitor instance for system pressure
            priority_map: Dict mapping "topic:partition" → PartitionPriority
                          Default: all partitions are NORMAL priority
        """
        self.consumer = consumer
        self.pressure_monitor = pressure_monitor
        self.priority_map = priority_map or {}

        self._lock = threading.Lock()
        self._partition_states: dict[str, PartitionState] = {}
        self._paused_partitions: Set[str] = set()

    def _partition_key(self, tp: TopicPartition) -> str:
        """Create unique key for a topic-partition."""
        return f"{tp.topic}:{tp.partition}"

    def register_partition(self, tp: TopicPartition):
        """Register a partition for flow control (called on assignment)."""
        key = self._partition_key(tp)
        priority_str = self.priority_map.get(key, "NORMAL")

        if isinstance(priority_str, str):
            try:
                priority = PartitionPriority[priority_str.upper()]
            except KeyError:
                priority = PartitionPriority.NORMAL
        elif isinstance(priority_str, PartitionPriority):
            priority = priority_str
        else:
            priority = PartitionPriority.NORMAL

        with self._lock:
            self._partition_states[key] = PartitionState(
                topic_partition=tp,
                priority=priority,
            )

        logger.info(
            f"Registered partition {key} with priority {priority.name}"
        )

    def unregister_partition(self, tp: TopicPartition):
        """Unregister a partition (called on revocation)."""
        key = self._partition_key(tp)
        with self._lock:
            self._partition_states.pop(key, None)
            self._paused_partitions.discard(key)

    def update_queue_fill(self, tp: TopicPartition, fill_pct: float):
        """
        Update queue fill percentage for a specific partition.
        Called by the consumer after each poll/processing cycle.
        """
        key = self._partition_key(tp)
        with self._lock:
            state = self._partition_states.get(key)
            if state:
                state.queue_fill_pct = fill_pct

    def evaluate_and_apply(self) -> dict:
        """
        Main control loop method — evaluate all partitions and
        apply pause/resume decisions.

        Called once per poll cycle by the consumer.

        Returns:
            Dict with control actions taken
        """
        now = time.monotonic()
        reading = self.pressure_monitor.get_reading()
        overall_pressure = reading.overall_pressure

        actions = {
            "paused": [],
            "resumed": [],
            "unchanged": [],
            "overall_pressure": overall_pressure,
        }

        with self._lock:
            for key, state in self._partition_states.items():
                action = self._evaluate_partition(state, overall_pressure, now)

                if action == "pause":
                    self._pause_partition(state, now)
                    actions["paused"].append(key)

                elif action == "resume":
                    self._resume_partition(state, now)
                    actions["resumed"].append(key)

                else:
                    actions["unchanged"].append(key)

        # Log significant changes
        if actions["paused"] or actions["resumed"]:
            paused_count = len(self._paused_partitions)
            total_count = len(self._partition_states)
            logger.info(
                f"Flow control: pressure={overall_pressure:.1%}, "
                f"paused={paused_count}/{total_count}, "
                f"actions: +paused={actions['paused']}, "
                f"+resumed={actions['resumed']}"
            )

        return actions

    def _evaluate_partition(
        self,
        state: PartitionState,
        overall_pressure: float,
        now: float
    ) -> str:
        """
        Evaluate whether a single partition should be paused or resumed.

        Decision tree:
        1. CRITICAL priority → never pause
        2. Already paused → check if should resume
        3. Not paused → check if should pause
        """
        key = self._partition_key(state.topic_partition)

        # Rule 1: Never pause CRITICAL partitions
        if state.priority == PartitionPriority.CRITICAL:
            if state.is_paused:
                return "resume"  # Force resume if somehow paused
            return "unchanged"

        # Rule 2: Already paused — check resume conditions
        if state.is_paused:
            # Check minimum pause duration (anti-thrash)
            if state.last_pause_time is not None:
                pause_elapsed = now - state.last_pause_time
                if pause_elapsed < self.MIN_PAUSE_DURATION:
                    return "unchanged"  # Too soon to resume

            # Check queue has drained below RESUME threshold (hysteresis)
            if state.queue_fill_pct < self.RESUME_THRESHOLD:
                # Also check that pressure level allows this priority
                pressure_threshold = self.PRIORITY_PRESSURE_THRESHOLDS[state.priority]
                if overall_pressure < pressure_threshold:
                    return "resume"

            return "unchanged"

        # Rule 3: Not paused — check pause conditions
        else:
            # Check if queue is too full
            if state.queue_fill_pct > self.PAUSE_THRESHOLD:
                return "pause"

            # Check if system pressure warrants pausing this priority
            pressure_threshold = self.PRIORITY_PRESSURE_THRESHOLDS[state.priority]
            if overall_pressure >= pressure_threshold:
                return "pause"

            return "unchanged"

    def _pause_partition(self, state: PartitionState, now: float):
        """Execute partition pause."""
        key = self._partition_key(state.topic_partition)

        self.consumer.pause([state.topic_partition])
        state.is_paused = True
        state.last_pause_time = now
        state.pause_count += 1
        state.messages_since_resume = 0
        self._paused_partitions.add(key)

        logger.info(
            f"PAUSED {key} "
            f"(priority={state.priority.name}, "
            f"queue={state.queue_fill_pct:.0%}, "
            f"pause_count={state.pause_count})"
        )

    def _resume_partition(self, state: PartitionState, now: float):
        """Execute partition resume."""
        key = self._partition_key(state.topic_partition)

        self.consumer.resume([state.topic_partition])
        state.is_paused = False
        state.last_resume_time = now
        self._paused_partitions.discard(key)

        pause_duration = now - state.last_pause_time if state.last_pause_time else 0
        logger.info(
            f"RESUMED {key} "
            f"(priority={state.priority.name}, "
            f"queue={state.queue_fill_pct:.0%}, "
            f"was paused {pause_duration:.1f}s)"
        )

    # ── QUERY METHODS ──
    @property
    def paused_count(self) -> int:
        with self._lock:
            return len(self._paused_partitions)

    @property
    def total_count(self) -> int:
        with self._lock:
            return len(self._partition_states)

    @property
    def active_ratio(self) -> float:
        """Ratio of active (non-paused) partitions."""
        total = self.total_count
        if total == 0:
            return 1.0
        return (total - self.paused_count) / total

    def get_partition_status(self) -> list[dict]:
        """Get status of all partitions for dashboard."""
        with self._lock:
            return [
                {
                    "partition": key,
                    "priority": state.priority.name,
                    "is_paused": state.is_paused,
                    "queue_fill_pct": round(state.queue_fill_pct, 3),
                    "pause_count": state.pause_count,
                }
                for key, state in sorted(self._partition_states.items())
            ]

    def pause_all_non_critical(self):
        """Emergency: pause everything except CRITICAL partitions."""
        with self._lock:
            now = time.monotonic()
            for key, state in self._partition_states.items():
                if state.priority != PartitionPriority.CRITICAL and not state.is_paused:
                    self._pause_partition(state, now)
        logger.warning("EMERGENCY: All non-critical partitions paused")

    def resume_all(self):
        """Resume all paused partitions."""
        with self._lock:
            now = time.monotonic()
            for key, state in self._partition_states.items():
                if state.is_paused:
                    self._resume_partition(state, now)
        logger.info("All partitions resumed")

5. Spillover Handler

# consumer/spillover_handler.py
# ─────────────────────────────────────────────────────────────
# Manages spillover topics for excess traffic
# When consumer can't process in real-time, messages are
# redirected to a spillover topic for later processing
#
# Pattern:
#   Main topic → Consumer → [Can process?]
#                              YES → Process normally
#                              NO  → Write to spillover topic
#                                    Commit main topic offset
#                                    Background job processes later
# ─────────────────────────────────────────────────────────────

import json
import logging
import time
import threading
from dataclasses import dataclass
from typing import Optional, Callable
from collections import defaultdict

from confluent_kafka import Producer, Consumer, KafkaError, TopicPartition

logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger("spillover-handler")


@dataclass
class SpilloverStats:
    """Spillover statistics per topic."""
    topic: str
    spillover_topic: str
    total_spilled: int = 0
    total_recovered: int = 0
    current_spillover_lag: int = 0
    last_spill_time: Optional[float] = None
    last_recover_time: Optional[float] = None


class SpilloverHandler:
    """
    Manages spillover topics for backpressure relief.

    When the consumer's processing pipeline is overwhelmed:
    1. Messages that can't be processed immediately are written
       to a spillover topic (e.g., orders-events-spillover)
    2. The original offset is committed (message is "handled")
    3. A background consumer processes the spillover topic at
       a controlled rate during off-peak hours

    Spillover message format:
    - Key: Original message key
    - Value: Original message value
    - Headers: Original headers + spillover metadata
      - spillover.source.topic: Original topic name
      - spillover.source.partition: Original partition
      - spillover.source.offset: Original offset
      - spillover.timestamp: When it was spilled
      - spillover.reason: Why it was spilled (queue_full, circuit_open, etc.)
    """

    def __init__(
        self,
        bootstrap_servers: str,
        spillover_suffix: str = "-spillover",
        producer_config: Optional[dict] = None
    ):
        self.bootstrap_servers = bootstrap_servers
        self.spillover_suffix = spillover_suffix

        # Spillover producer — writes to spillover topics
        base_config = {
            "bootstrap.servers": bootstrap_servers,
            "acks": "all",                    # No data loss on spillover
            "retries": 2147483647,
            "delivery.timeout.ms": 120000,
            "linger.ms": 50,                  # Small batch for latency
            "compression.type": "lz4",
            "enable.idempotence": True,       # Project 17: exactly-once
        }
        if producer_config:
            base_config.update(producer_config)

        self._producer = Producer(base_config)
        self._stats: dict[str, SpilloverStats] = {}
        self._lock = threading.Lock()
        self._running = True

        # Background flush thread
        self._flush_thread = threading.Thread(
            target=self._flush_loop,
            daemon=True,
            name="spillover-flush"
        )
        self._flush_thread.start()

    def get_spillover_topic(self, source_topic: str) -> str:
        """Derive spillover topic name from source topic."""
        return f"{source_topic}{self.spillover_suffix}"

    def spill(
        self,
        source_topic: str,
        source_partition: int,
        source_offset: int,
        key: Optional[bytes],
        value: Optional[bytes],
        headers: Optional[list] = None,
        reason: str = "backpressure"
    ) -> bool:
        """
        Write a message to the spillover topic.

        Args:
            source_topic: Original topic the message came from
            source_partition: Original partition number
            source_offset: Original offset
            key: Original message key (bytes)
            value: Original message value (bytes)
            headers: Original message headers
            reason: Why this message was spilled

        Returns:
            True if successfully queued for spillover
        """
        spillover_topic = self.get_spillover_topic(source_topic)

        # Build headers with spillover metadata
        spillover_headers = list(headers or [])
        spillover_headers.extend([
            ("spillover.source.topic", source_topic.encode("utf-8")),
            ("spillover.source.partition", str(source_partition).encode("utf-8")),
            ("spillover.source.offset", str(source_offset).encode("utf-8")),
            ("spillover.timestamp", str(int(time.time() * 1000)).encode("utf-8")),
            ("spillover.reason", reason.encode("utf-8")),
        ])

        try:
            self._producer.produce(
                topic=spillover_topic,
                key=key,
                value=value,
                headers=spillover_headers,
                on_delivery=self._delivery_callback,
            )

            # Update stats
            with self._lock:
                if source_topic not in self._stats:
                    self._stats[source_topic] = SpilloverStats(
                        topic=source_topic,
                        spillover_topic=spillover_topic,
                    )
                stats = self._stats[source_topic]
                stats.total_spilled += 1
                stats.last_spill_time = time.time()

            return True

        except BufferError:
            logger.error(
                f"Spillover producer buffer full for {spillover_topic} — "
                f"message from {source_topic}:{source_partition}:{source_offset} "
                f"could not be spilled"
            )
            return False
        except Exception as e:
            logger.error(f"Spillover write failed: {e}")
            return False

    def _delivery_callback(self, err, msg):
        """Handle spillover message delivery confirmation."""
        if err:
            logger.error(
                f"Spillover delivery FAILED for {msg.topic()}: {err}"
            )
        else:
            logger.debug(
                f"Spillover delivered: {msg.topic()}:{msg.partition()}:{msg.offset()}"
            )

    def _flush_loop(self):
        """Background thread to flush spillover producer."""
        while self._running:
            try:
                remaining = self._producer.flush(timeout=1.0)
                if remaining > 0:
                    logger.debug(f"Spillover flush: {remaining} messages pending")
            except Exception as e:
                logger.warning(f"Spillover flush error: {e}")
            time.sleep(0.5)

    def get_stats(self) -> dict:
        """Get spillover statistics for all topics."""
        with self._lock:
            return {
                topic: {
                    "spillover_topic": stats.spillover_topic,
                    "total_spilled": stats.total_spilled,
                    "total_recovered": stats.total_recovered,
                    "pending": stats.total_spilled - stats.total_recovered,
                    "last_spill_time": stats.last_spill_time,
                }
                for topic, stats in self._stats.items()
            }

    def close(self):
        """Shutdown spillover handler."""
        self._running = False
        self._producer.flush(timeout=10)


class SpilloverRecoveryConsumer:
    """
    Background consumer that processes spillover topics at controlled rate.

    Runs during off-peak hours or whenever system pressure is low.
    Processes at a fraction of normal throughput to avoid re-overwhelming.
    """

    def __init__(
        self,
        bootstrap_servers: str,
        spillover_topics: list[str],
        process_fn: Callable,
        rate_limit: float = 1000.0,
        group_id: str = "spillover-recovery"
    ):
        """
        Args:
            bootstrap_servers: Kafka bootstrap servers
            spillover_topics: List of spillover topics to process
            process_fn: Function to process each message
            rate_limit: Maximum messages per second to process
            group_id: Consumer group ID for recovery consumer
        """
        self.process_fn = process_fn
        self.rate_limit = rate_limit
        self._running = False

        from rate_limiter import TokenBucketRateLimiter
        self._rate_limiter = TokenBucketRateLimiter(
            rate=rate_limit,
            burst=int(rate_limit * 2),
            name="spillover-recovery"
        )

        self._consumer = Consumer({
            "bootstrap.servers": bootstrap_servers,
            "group.id": group_id,
            "auto.offset.reset": "earliest",
            "enable.auto.commit": False,
            "max.poll.records": 100,         # Small batches for rate control
            "max.poll.interval.ms": 300000,
        })
        self._consumer.subscribe(spillover_topics)

        self._stats = {
            "processed": 0,
            "failed": 0,
            "total_latency_ms": 0.0,
        }

    def run(self, pressure_fn: Optional[Callable] = None):
        """
        Process spillover messages at controlled rate.

        Args:
            pressure_fn: Optional callable returning current system pressure.
                         If pressure > 0.6, pause recovery to avoid adding load.
        """
        self._running = True
        logger.info(
            f"Spillover recovery started "
            f"(rate_limit={self.rate_limit}/s)"
        )

        while self._running:
            # Check system pressure — pause recovery if system is busy
            if pressure_fn:
                pressure = pressure_fn()
                if pressure > 0.6:
                    logger.info(
                        f"System pressure {pressure:.1%} > 60% — "
                        f"pausing spillover recovery"
                    )
                    time.sleep(10)
                    continue

            msg = self._consumer.poll(timeout=1.0)

            if msg is None:
                continue

            if msg.error():
                if msg.error().code() == KafkaError._PARTITION_EOF:
                    continue
                logger.warning(f"Spillover consumer error: {msg.error()}")
                continue

            # Rate limit processing
            self._rate_limiter.acquire(timeout=5.0)

            # Extract original message info from headers
            original_info = self._extract_spillover_headers(msg)

            try:
                start = time.time()
                self.process_fn(msg, original_info)
                elapsed_ms = (time.time() - start) * 1000

                self._stats["processed"] += 1
                self._stats["total_latency_ms"] += elapsed_ms

                self._consumer.commit(message=msg)

            except Exception as e:
                self._stats["failed"] += 1
                logger.error(
                    f"Spillover processing failed for "
                    f"{msg.topic()}:{msg.partition()}:{msg.offset()}: {e}"
                )
                # Commit anyway to avoid infinite retry
                # Failed messages logged for manual investigation
                self._consumer.commit(message=msg)

        self._consumer.close()
        logger.info("Spillover recovery stopped")

    def _extract_spillover_headers(self, msg) -> dict:
        """Extract spillover metadata from message headers."""
        info = {}
        if msg.headers():
            for header_key, header_value in msg.headers():
                if header_key.startswith("spillover."):
                    info[header_key] = header_value.decode("utf-8") if header_value else None
        return info

    def stop(self):
        self._running = False

    def get_stats(self) -> dict:
        avg_latency = 0.0
        if self._stats["processed"] > 0:
            avg_latency = self._stats["total_latency_ms"] / self._stats["processed"]
        return {
            **self._stats,
            "avg_latency_ms": round(avg_latency, 2),
            "rate_limiter": self._rate_limiter.stats.to_dict(),
        }

6. Adaptive Poller

# consumer/adaptive_poller.py
# ─────────────────────────────────────────────────────────────
# Dynamically adjusts max.poll.records based on system pressure
# Prevents OOM by reducing fetch size when memory is high
# Increases fetch size when system is healthy for throughput
# ─────────────────────────────────────────────────────────────

import logging
import time
import threading
from dataclasses import dataclass

logging.basicConfig(level=logging.INFO, format="%(asctime)s [%(levelname)s] %(message)s")
logger = logging.getLogger("adaptive-poller")


@dataclass
class PollerConfig:
    """Configuration for adaptive polling."""
    min_poll_records: int = 10
    max_poll_records: int = 1000
    initial_poll_records: int = 200
    adjustment_interval_seconds: float = 10.0

    # Pressure thresholds for adjustment
    increase_below_pressure: float = 0.3    # Increase when pressure < 30%
    decrease_above_pressure: float = 0.6    # Decrease when pressure > 60%
    emergency_above_pressure: float = 0.85  # Emergency reduction > 85%

    # Adjustment factors
    increase_factor: float = 1.25           # Increase by 25%
    decrease_factor: float = 0.7            # Decrease by 30%
    emergency_factor: float = 0.4           # Emergency: cut to 40%


class AdaptivePoller:
    """
    Dynamically adjusts the effective number of records to process per poll.

    NOTE: confluent-kafka doesn't support changing max.poll.records
    after consumer creation. So we implement this at the application level:
    - Consumer fetches up to max.poll.records (set at creation)
    - Application only PROCESSES up to current_poll_records
    - Remaining records stay in the consumer's internal buffer
    - Next poll returns them (no re-fetch needed)

    This gives us adaptive batch sizing without reconfiguring the consumer.
    """

    def __init__(self, config: Optional[PollerConfig] = None, pressure_monitor=None):
        self.config = config or PollerConfig()
        self.pressure_monitor = pressure_monitor

        self._current_records = self.config.initial_poll_records
        self._lock = threading.Lock()
        self._last_adjustment = time.monotonic()

        # Processing time tracking for smart adjustment
        self._recent_processing_times: list[float] = []
        self._max_tracking = 20

        logger.info(
            f"AdaptivePoller initialized: "
            f"current={self._current_records}, "
            f"range=[{self.config.min_poll_records}, {self.config.max_poll_records}]"
        )

    @property
    def current_poll_records(self) -> int:
        """Current number of records to process per poll cycle."""
        with self._lock:
            return self._current_records

    def record_processing_time(self, records_count: int, elapsed_seconds: float):
        """
        Record how long it took to process a batch.
        Used for intelligent adjustment decisions.
        """
        if records_count > 0:
            per_record_ms = (elapsed_seconds * 1000) / records_count
            with self._lock:
                self._recent_processing_times.append(per_record_ms)
                if len(self._recent_processing_times) > self._max_tracking:
                    self._recent_processing_times.pop(0)

    def adjust(self) -> int:
        """
        Evaluate and adjust poll records count based on pressure.
        Called periodically (every adjustment_interval).

        Returns:
            New current_poll_records value
        """
        now = time.monotonic()

        with self._lock:
            # Check if enough time has passed since last adjustment
            elapsed = now - self._last_adjustment
            if elapsed < self.config.adjustment_interval_seconds:
                return self._current_records

            self._last_adjustment = now
            old_value = self._current_records

            # Get current pressure
            if self.pressure_monitor:
                pressure = self.pressure_monitor.overall_pressure
            else:
                pressure = 0.0

            # Emergency reduction
            if pressure > self.config.emergency_above_pressure:
                new_value = max(
                    self.config.min_poll_records,
                    int(self._current_records * self.config.emergency_factor)
                )
                if new_value != old_value:
                    logger.warning(
                        f"AdaptivePoller EMERGENCY reduction: "
                        f"{old_value} → {new_value} "
                        f"(pressure={pressure:.1%})"
                    )

            # Normal decrease
            elif pressure > self.config.decrease_above_pressure:
                new_value = max(
                    self.config.min_poll_records,
                    int(self._current_records * self.config.decrease_factor)
                )
                if new_value != old_value:
                    logger.info(
                        f"AdaptivePoller decrease: "
                        f"{old_value} → {new_value} "
                        f"(pressure={pressure:.1%})"
                    )

            # Increase (only when pressure is low)
            elif pressure < self.config.increase_below_pressure:
                new_value = min(
                    self.config.max_poll_records,
                    int(self._current_records * self.config.increase_factor)
                )
                if new_value != old_value:
                    logger.info(
                        f"AdaptivePoller increase: "
                        f"{old_value} → {new_value} "
                        f"(pressure={pressure:.1%})"
                    )

            # Stable — no change
            else:
                new_value = self._current_records

            self._current_records = new_value
            return new_value

    def get_stats(self) -> dict:
        """Get adaptive poller statistics."""
        with self._lock:
            avg_processing_time = 0.0
            if self._recent_processing_times:
                avg_processing_time = (
                    sum(self._recent_processing_times)
                    / len(self._recent_processing_times)
                )

            return {
                "current_poll_records": self._current_records,
                "min": self.config.min_poll_records,
                "max": self.config.max_poll_records,
                "avg_processing_time_per_record_ms": round(avg_processing_time, 2),
            }

    Production-grade Kafka consumer with multi-layer backpressure handling.

    Architecture:
    ┌─────────────────────────────────────────────────────────┐
    │  Poll Thread (main)                                     │
    │  ┌─────────────┐    ┌──────────────┐                   │
    │  │ Kafka       │───▶│ Partition    │                   │
    │  │ Consumer    │    │ Flow Control │                   │
    │  │ poll()      │    │ (pause/      │                   │
    │  │             │    │  resume)     │                   │
    │  └─────────────┘    └──────┬───────┘                   │
    │                            │                            │
    │                   ┌────────▼────────┐                  │
    │                   │ Bounded Work    │                  │
    │                   │ Queue (1000)    │                  │
    │                   └────────┬────────┘                  │
    │                            │                            │
    │  ┌─────────────────────────┼───────────────────────┐   │
    │  │ Worker Thread Pool (4 threads)                  │   │
    │  │                         │                        │   │
    │  │  ┌──────────────┐ ┌────▼─────┐ ┌────────────┐  │   │
    │  │  │ Circuit      │ │ Process  │ │ Spillover  │  │   │
    │  │  │ Breaker      │ │ Function │ │ Handler    │  │   │
    │  │  │ (downstream) │ │ (user)   │ │ (overflow) │  │   │
    │  │  └──────────────┘ └──────────┘ └────────────┘  │   │
    │  └─────────────────────────────────────────────────┘   │
    │                                                         │
    │  ┌─────────────────────────────────────────────────┐   │
    │  │ Pressure Monitor (aggregates all metrics)       │   │
    │  │ → feeds back to Partition Flow Controller       │   │
    │  │ → feeds back to Adaptive Poller                 │   │
    │  └─────────────────────────────────────────────────┘   │
    └─────────────────────────────────────────────────────────┘
    """

    def __init__(
        self,
        config: BackpressureConsumerConfig,
        process_fn: Callable[[Any], None]
    ):

        Args:
            config: Consumer configuration
            process_fn: Function to process each message.
                        Signature: process_fn(message) -> None
                        Raises exception on failure.
        """
        self.config = config
        self.process_fn = process_fn
        self._running = False

        consumer_conf = {
            "bootstrap.servers": config.bootstrap_servers,

            "session.timeout.ms": config.session_timeout_ms,
            "heartbeat.interval.ms": config.heartbeat_interval_ms,
            "fetch.max.bytes": config.fetch_max_bytes,

        self._consumer = Consumer(consumer_conf)

        # ── Initialize Components ──
        # 1. Pressure Monitor
        self._pressure_monitor = PressureMonitor(
            consumer_lag_threshold=config.consumer_lag_threshold,
            downstream_latency_sla_ms=config.downstream_latency_sla_ms,

        self._flow_controller = PartitionFlowController(
            consumer=self._consumer,
            pressure_monitor=self._pressure_monitor,
            priority_map=config.partition_priorities,
        )

        # 3. Adaptive Poller
        self._adaptive_poller = AdaptivePoller(

        self._circuit_breaker = CircuitBreaker(
            failure_threshold=config.circuit_failure_threshold,
            recovery_timeout=config.circuit_recovery_timeout,

                spillover_suffix=config.spillover_suffix,

        self._work_queue = queue.Queue(maxsize=config.work_queue_size)

        # 7. Worker threads
        self._workers: list[threading.Thread] = []

        # ── Metrics ──

            "poll_cycles": 0,
            "empty_polls": 0,
            "total_processing_time_ms": 0.0,

        self._circuit_breaker.on_state_change(self._on_circuit_state_change)

    def _on_circuit_state_change(self, old_state, new_state):
        """React to circuit breaker state changes."""
        if new_state == CircuitState.OPEN:
            logger.warning(
                "Circuit breaker OPEN — pausing all non-critical partitions"

            self._flow_controller.pause_all_non_critical()
        elif new_state == CircuitState.CLOSED:
            logger.info(
                "Circuit breaker CLOSED — resuming partitions"
            )
            # Don't resume all immediately — let flow controller decide
            # based on queue fill levels

    def _on_assign(self, consumer, partitions):
        """Callback when partitions are assigned to this consumer."""
        logger.info(f"Assigned partitions: {[f'{p.topic}:{p.partition}' for p in partitions]}")
        for tp in partitions:
            self._flow_controller.register_partition(tp)

    def _on_revoke(self, consumer, partitions):

        logger.info(f"Revoked partitions: {[f'{p.topic}:{p.partition}' for p in partitions]}")
        for tp in partitions:

        # Commit any pending offsets before revocation
        try:
            consumer.commit(asynchronous=False)
        except Exception as e:

    def _worker_loop(self, worker_id: int):
        """
        Worker thread: processes messages from the bounded work queue.

        Each worker:
        1. Takes a message from the queue
        2. Checks circuit breaker
        3. If circuit closed: process normally
        4. If circuit open: spillover (if enabled) or re-queue
        """
        logger.info(f"Worker-{worker_id} started")

        while self._running:
            try:
                # Get message from queue with timeout
                # Timeout ensures thread can check self._running
                try:
                    item = self._work_queue.get(timeout=1.0)
                except queue.Empty:
                    continue

                msg, tp = item

                # Check circuit breaker
                if self._circuit_breaker.can_execute():
                    # Normal processing path
                    try:
                        start = time.time()
                        self.process_fn(msg)
                        elapsed_ms = (time.time() - start) * 1000

                        self._circuit_breaker.record_success()

                        with self._metrics_lock:
                            self._metrics["messages_processed"] += 1
                            self._metrics["total_processing_time_ms"] += elapsed_ms

                        # Update downstream latency for pressure monitor
                        self._pressure_monitor.update_downstream_latency(elapsed_ms)

                    except Exception as e:
                        self._circuit_breaker.record_failure()
                        with self._metrics_lock:
                            self._metrics["messages_failed"] += 1

                        logger.warning(
                            f"Worker-{worker_id} processing failed: {e}"
                        )

                        # Spillover on processing failure
                        if self._spillover:
                            self._spillover.spill(
                                source_topic=msg.topic(),
                                source_partition=msg.partition(),
                                source_offset=msg.offset(),
                                key=msg.key(),
                                value=msg.value(),
                                headers=msg.headers(),
                                reason="processing_error",
                            )
                            with self._metrics_lock:
                                self._metrics["messages_spilled"] += 1

                else:
                    # Circuit is OPEN — spillover immediately
                    if self._spillover:
                        self._spillover.spill(
                            source_topic=msg.topic(),
                            source_partition=msg.partition(),
                            source_offset=msg.offset(),
                            key=msg.key(),
                            value=msg.value(),
                            headers=msg.headers(),
                            reason="circuit_open",
                        )
                        with self._metrics_lock:
                            self._metrics["messages_spilled"] += 1
                    else:
                        # No spillover — re-queue with backoff
                        logger.warning(
                            f"Circuit open, no spillover — "
                            f"dropping message {msg.topic()}:{msg.partition()}:{msg.offset()}"
                        )
                        with self._metrics_lock:
                            self._metrics["messages_failed"] += 1

                self._work_queue.task_done()

            except Exception as e:
                logger.error(f"Worker-{worker_id} unexpected error: {e}")

        logger.info(f"Worker-{worker_id} stopped")

    def run(self):
        """
        Main consumer loop.

        This is the heart of the backpressure system:
        1. Poll Kafka for messages
        2. Limit batch size via adaptive poller
        3. Enqueue messages to bounded work queue
        4. If queue full → partition gets paused (flow controller)
        5. Workers process from queue
        6. As queue drains → partitions resume
        7. Commit offsets periodically
        """
        self._running = True

        # Subscribe with rebalance callbacks
        self._consumer.subscribe(
            self.config.topics,
            on_assign=self._on_assign,
            on_revoke=self._on_revoke,
        )

        # Start worker threads
        for i in range(self.config.worker_threads):
            worker = threading.Thread(
                target=self._worker_loop,
                args=(i,),
                daemon=True,
                name=f"worker-{i}"
            )
            worker.start()
            self._workers.append(worker)

        logger.info(
            f"Consumer running: {len(self._workers)} workers, "
            f"queue capacity={self.config.work_queue_size}"
        )

        # Offset tracking for periodic commits
        last_commit_time = time.time()
        commit_interval = 5.0  # Commit every 5 seconds
        pending_offsets: dict[str, TopicPartition] = {}

        try:
            while self._running:
                # ── Step 1: Adaptive poll adjustment ──
                self._adaptive_poller.adjust()
                effective_batch = self._adaptive_poller.current_poll_records

                # ── Step 2: Evaluate flow control (pause/resume) ──
                # Update queue fill for pressure calculation
                queue_fill = self._work_queue.qsize() / max(1, self.config.work_queue_size)
                self._pressure_monitor.update_queue_fill(queue_fill)

                # Update per-partition queue fill estimates
                # (Simplified: use global queue fill for all partitions)
                for status in self._flow_controller.get_partition_status():
                    tp_parts = status["partition"].split(":")
                    if len(tp_parts) == 2:
                        tp = TopicPartition(tp_parts[0], int(tp_parts[1]))
                        self._flow_controller.update_queue_fill(tp, queue_fill)

                flow_actions = self._flow_controller.evaluate_and_apply()

                # ── Step 3: Poll for messages ──
                msg = self._consumer.poll(timeout=1.0)

                with self._metrics_lock:
                    self._metrics["poll_cycles"] += 1

                if msg is None:
                    with self._metrics_lock:
                        self._metrics["empty_polls"] += 1
                    # Still need to commit periodically
                    if time.time() - last_commit_time > commit_interval:
                        self._commit_offsets(pending_offsets)
                        pending_offsets.clear()
                        last_commit_time = time.time()
                    continue

                if msg.error():
                    if msg.error().code() == KafkaError._PARTITION_EOF:
                        continue
                    logger.warning(f"Consumer error: {msg.error()}")
                    continue

                # ── Step 4: Enqueue message to work queue ──
                tp = TopicPartition(msg.topic(), msg.partition())
                tp_key = f"{msg.topic()}:{msg.partition()}"

                try:
                    # Non-blocking put with timeout
                    self._work_queue.put(
                        (msg, tp),
                        block=True,
                        timeout=0.5
                    )

                    with self._metrics_lock:
                        self._metrics["messages_polled"] += 1

                    # Track highest offset per partition for commit
                    pending_offsets[tp_key] = TopicPartition(
                        msg.topic(),
                        msg.partition(),
                        msg.offset() + 1  # Commit offset = last processed + 1
                    )

                except queue.Full:
                    # Queue is full — this partition should be paused
                    logger.debug(
                        f"Work queue full — cannot enqueue "
                        f"{msg.topic()}:{msg.partition()}:{msg.offset()}"
                    )
                    # Force pause this partition
                    self._flow_controller.update_queue_fill(tp, 1.0)

                    # Spillover if available
                    if self._spillover:
                        self._spillover.spill(
                            source_topic=msg.topic(),
                            source_partition=msg.partition(),
                            source_offset=msg.offset(),
                            key=msg.key(),
                            value=msg.value(),
                            headers=msg.headers(),
                            reason="queue_full",
                        )
                        with self._metrics_lock:
                            self._metrics["messages_spilled"] += 1

                        # Still track offset for commit
                        pending_offsets[tp_key] = TopicPartition(
                            msg.topic(),
                            msg.partition(),
                            msg.offset() + 1
                        )

                # ── Step 5: Periodic offset commit ──
                if time.time() - last_commit_time > commit_interval:
                    self._commit_offsets(pending_offsets)
                    pending_offsets.clear()
                    last_commit_time = time.time()

                # ── Step 6: Update consumer lag for pressure monitor ──
                self._update_consumer_lag()

        except KeyboardInterrupt:
            logger.info("Shutdown signal received")
        finally:
            self.shutdown()

    def _commit_offsets(self, offsets: dict[str, TopicPartition]):
        """Commit tracked offsets."""
        if not offsets:
            return

        try:
            offset_list = list(offsets.values())
            self._consumer.commit(offsets=offset_list, asynchronous=True)
            logger.debug(f"Committed offsets for {len(offset_list)} partitions")
        except Exception as e:
            logger.warning(f"Offset commit failed: {e}")

    def _update_consumer_lag(self):
        """Calculate and update total consumer lag."""
        try:
            assignment = self._consumer.assignment()
            if not assignment:
                return

            total_lag = 0
            for tp in assignment:
                try:
                    low, high = self._consumer.get_watermark_offsets(tp, timeout=1)
                    position = self._consumer.position([tp])
                    if position and position[0].offset >= 0:
                        partition_lag = max(0, high - position[0].offset)
                        total_lag += partition_lag
                except Exception:
                    pass

            self._pressure_monitor.update_consumer_lag(total_lag)

        except Exception as e:
            logger.debug(f"Lag calculation error: {e}")

    def shutdown(self):
        """Graceful shutdown."""
        logger.info("Shutting down backpressure consumer...")
        self._running = False

        # Wait for workers to finish current work
        for worker in self._workers:
            worker.join(timeout=10)

        # Final commit
        try:
            self._consumer.commit(asynchronous=False)
        except Exception:
            pass

        # Close consumer
        try:
            self._consumer.close()
        except Exception:
            pass

        # Close spillover handler
        if self._spillover:
            self._spillover.close()

        logger.info("Backpressure consumer shutdown complete")

    def get_status(self) -> dict:
        """Get comprehensive consumer status for monitoring."""
        with self._metrics_lock:
            metrics = dict(self._metrics)

        pressure = self._pressure_monitor.get_reading()
        trend = self._pressure_monitor.get_trend()

        avg_processing_ms = 0.0
        if metrics["messages_processed"] > 0:
            avg_processing_ms = (
                metrics["total_processing_time_ms"] / metrics["messages_processed"]
            )

        return {
            "metrics": metrics,
            "pressure": pressure.to_dict(),
            "pressure_level": self._pressure_monitor.get_pressure_level(),
            "pressure_trend": trend,
            "flow_control": {
                "paused_partitions": self._flow_controller.paused_count,
                "total_partitions": self._flow_controller.total_count,
                "active_ratio": round(self._flow_controller.active_ratio, 3),
                "partition_details": self._flow_controller.get_partition_status(),
            },
            "adaptive_poller": self._adaptive_poller.get_stats(),
            "circuit_breaker": self._circuit_breaker.get_stats(),
            "spillover": self._spillover.get_stats() if self._spillover else {},
            "work_queue": {
                "size": self._work_queue.qsize(),
                "capacity": self.config.work_queue_size,
                "fill_pct": round(
                    self._work_queue.qsize() / max(1, self.config.work_queue_size), 3

        Send a message with rate limiting.

        Args:
            topic: Target topic
            value: Message value (bytes)
            key: Optional message key (bytes)
            headers: Optional message headers
            partition: Optional specific partition
            on_delivery: Optional delivery callback
            block_on_rate_limit: If True, blocks until rate allows.
                                 If False, returns False immediately.

        Returns:
            True if message was queued, False if rate limited/buffer full
        """
        # Rate limiting
        if block_on_rate_limit:
            acquired = self._rate_limiter.acquire(timeout=5.0)
        else:
            if hasattr(self._rate_limiter, 'limiter'):
                acquired = self._rate_limiter.try_acquire()
            else:
                acquired = self._rate_limiter.try_acquire()

        if not acquired:
            with self._lock:
                self._metrics["rate_limited"] += 1
            return False

        # Check buffer pressure before sending
        buffer_pressure = self._get_buffer_pressure()
        if buffer_pressure > self.config.buffer_critical_threshold:
            logger.warning(
                f"Buffer pressure critical ({buffer_pressure:.1%}) — "
                f"triggering flush before send"
            )
            self._producer.flush(timeout=1.0)
            with self._lock:
                self._metrics["buffer_full_events"] += 1

        # Send message
        try:
            kwargs = {
                "topic": topic,
                "value": value,
                "on_delivery": on_delivery or self._default_delivery_callback,
            }
            if key is not None:
                kwargs["key"] = key
            if headers:
                kwargs["headers"] = headers
            if partition is not None:
                kwargs["partition"] = partition

            self._producer.produce(**kwargs)

            with self._lock:
                self._metrics["sent"] += 1

            # Trigger any ready callbacks
            self._producer.poll(0)

            return True

        except BufferError:
            with self._lock:
                self._metrics["buffer_full_events"] += 1
            logger.warning("Producer buffer full — message not queued")
            return False

        except Exception as e:
            with self._lock:
                self._metrics["failed"] += 1
            logger.error(f"Producer send error: {e}")
            return False

    def _default_delivery_callback(self, err, msg):
        """Default delivery callback for tracking metrics."""
        if err:
            with self._lock:
                self._metrics["failed"] += 1
            logger.debug(f"Delivery failed: {err}")
        else:
            with self._lock:
                self._metrics["delivered"] += 1

    def _stats_callback(self, stats_json_str):
        """
        Kafka producer statistics callback.
        Called every statistics.interval.ms with internal metrics.
        Used to track buffer usage and throttle time.
        """
        try:
            stats = json.loads(stats_json_str)

            # Buffer metrics
            # msg_cnt and msg_size give us buffer usage
            with self._lock:
                self._buffer_used_bytes = stats.get("msg_size", 0)
                throttle_time = stats.get("txmsgs_throttle_time", 0)
                self._metrics["throttle_time_ms"] += throttle_time

            # Update pressure monitor with buffer usage
            self._pressure_monitor.update_producer_buffer(
                self._buffer_used_bytes,
                self._buffer_total_bytes
            )

            buffer_pct = self._buffer_used_bytes / max(1, self._buffer_total_bytes)
            if buffer_pct > self.config.buffer_warn_threshold:
                logger.warning(
                    f"Producer buffer: {buffer_pct:.1%} "
                    f"({self._buffer_used_bytes / 1048576:.1f}MB / "
                    f"{self._buffer_total_bytes / 1048576:.0f}MB)"
                )

        except (json.JSONDecodeError, Exception) as e:
            logger.debug(f"Stats callback error: {e}")

    def _get_buffer_pressure(self) -> float:
        """Get current buffer fill percentage (0.0 to 1.0)."""
        with self._lock:
            if self._buffer_total_bytes <= 0:
                return 0.0
            return min(1.0, self._buffer_used_bytes / self._buffer_total_bytes)

    def _flush_loop(self):
        """Background thread to periodically flush producer."""
        while self._running:
            try:
                remaining = self._producer.flush(timeout=1.0)
                if remaining > 0:
                    logger.debug(f"Producer flush: {remaining} messages pending")
            except Exception as e:
                logger.warning(f"Producer flush error: {e}")
            time.sleep(0.5)

    def flush(self, timeout: float = 10.0) -> int:
        """Flush all pending messages."""
        return self._producer.flush(timeout=timeout)

    def get_status(self) -> dict:
        """Get producer status for monitoring."""
        with self._lock:
            metrics = dict(self._metrics)
            buffer_pct = self._get_buffer_pressure()

        rate_stats = self._rate_limiter.stats
        if hasattr(self._rate_limiter, 'limiter'):
            rate_stats = self._rate_limiter.limiter.stats

        return {
            "metrics": metrics,
            "buffer": {
                "used_bytes": self._buffer_used_bytes,
                "total_bytes": self._buffer_total_bytes,
                "fill_pct": round(buffer_pct, 3),
            },
            "rate_limiter": rate_stats.to_dict(),
            "adaptive": self.config.adaptive,
        }

    def close(self):
        """Graceful shutdown."""
        self._running = False
        remaining = self._producer.flush(timeout=30)
        if remaining > 0:
            logger.warning(f"Producer closed with {remaining} unflushed messages")
        else:
            logger.info("Producer closed — all messages flushed")

        if hasattr(self._rate_limiter, 'stop'):
            self._rate_limiter.stop()

class SimulatedDatabase:
    """
    Simulates a downstream database that slows down under load.

    Normal: 2ms per insert
    Under load: 50-500ms per insert
    Overloaded: throws exceptions (simulates connection timeout)
    """

    def __init__(self):
        self._lock = threading.Lock()
        self._insert_count = 0
        self._concurrent_inserts = 0
        self._max_concurrent = 50     # Connection pool size
        self._overload_threshold = 40  # Start slowing at 40 concurrent

    def insert(self, record: dict) -> bool:
        """Simulate database insert with realistic latency."""
        with self._lock:
            self._concurrent_inserts += 1
            concurrent = self._concurrent_inserts

        try:
            # Simulate latency based on load
            if concurrent > self._max_concurrent:
                # Connection pool exhausted — throw error
                raise ConnectionError(
                    f"Connection pool exhausted "
                    f"({concurrent}/{self._max_concurrent})"
                )
            elif concurrent > self._overload_threshold:
                # Slow — database under pressure
                latency = random.uniform(0.05, 0.5)  # 50-500ms
                time.sleep(latency)
            else:
                # Normal — fast insert
                latency = random.uniform(0.001, 0.005)  # 1-5ms
                time.sleep(latency)

            with self._lock:
                self._insert_count += 1

            return True

        finally:
            with self._lock:
                self._concurrent_inserts -= 1

    @property
    def stats(self) -> dict:
        with self._lock:
            return {
                "total_inserts": self._insert_count,
                "concurrent_inserts": self._concurrent_inserts,
                "max_concurrent": self._max_concurrent,
            }

class TrafficGenerator:
    """
    Generates traffic following Black Friday pattern.

    Phase 1 (0-30s):   Normal — 1,000 msg/s
    Phase 2 (30-60s):  Ramp — linear increase to 10,000 msg/s
    Phase 3 (60-120s): Peak — 10,000 msg/s
    Phase 4 (120-150s): Ramp down — linear decrease to 1,000 msg/s
    Phase 5 (150-180s): Normal — 1,000 msg/s
    """

    def __init__(self, bootstrap_servers: str):
        self._producer = Producer({
            "bootstrap.servers": bootstrap_servers,
            "acks": "1",
            "batch.size": 65536,
            "linger.ms": 10,
            "compression.type": "lz4",
            "buffer.memory": 67108864,
        })

        self._rate_limiter = TokenBucketRateLimiter(
            rate=1000,
            burst=5000,
            name="traffic-generator"
        )

        self._running = False
        self._start_time = 0
        self._messages_sent = 0
        self._lock = threading.Lock()

    def _get_target_rate(self, elapsed_seconds: float) -> float:
        """Calculate target rate based on simulation phase."""
        if elapsed_seconds < 30:
            return 1000.0                                      # Phase 1: Normal
        elif elapsed_seconds < 60:
            progress = (elapsed_seconds - 30) / 30.0
            return 1000.0 + (9000.0 * progress)               # Phase 2: Ramp up
        elif elapsed_seconds < 120:
            return 10000.0                                     # Phase 3: Peak
        elif elapsed_seconds < 150:
            progress = (elapsed_seconds - 120) / 30.0
            return 10000.0 - (9000.0 * progress)              # Phase 4: Ramp down
        else:
            return 1000.0                                      # Phase 5: Recovery

    def _get_phase_name(self, elapsed_seconds: float) -> str:
        if elapsed_seconds < 30:
            return "NORMAL"
        elif elapsed_seconds < 60:
            return "RAMP_UP"
        elif elapsed_seconds < 120:
            return "PEAK"
        elif elapsed_seconds < 150:
            return "RAMP_DOWN"
        else:
            return "RECOVERY"

    def run(self, duration_seconds: int = 180):
        """Run the traffic generator for specified duration."""
        self._running = True
        self._start_time = time.time()

        logger.info("=" * 60)
        logger.info("  BLACK FRIDAY TRAFFIC GENERATOR STARTED")

        logger.info("=" * 60)

            ("payments-events", 0.25),
            ("analytics-events", 0.15),

        last_rate_update = 0
        last_status_log = 0

        while self._running and (time.time() - self._start_time) < duration_seconds:
            elapsed = time.time() - self._start_time
            target_rate = self._get_target_rate(elapsed)

            # Update rate limiter every second
            if int(elapsed) > last_rate_update:
                self._rate_limiter.set_rate(target_rate)
                last_rate_update = int(elapsed)

            # Log status every 10 seconds
            if int(elapsed) % 10 == 0 and int(elapsed) > last_status_log:
                last_status_log = int(elapsed)
                phase = self._get_phase_name(elapsed)
                with self._lock:
                    sent = self._messages_sent

                    f"Target: {target_rate:.0f}/s, "
                    f"Total sent: {sent:,}"

            cumulative = 0

                self._producer.poll(0.1)

        remaining = self._producer.flush(timeout=30)
        if remaining > 0:
            logger.warning(f"Generator stopped with {remaining} unflushed")

        with self._lock:
            total = self._messages_sent

        logger.info("=" * 60)
        logger.info(f"  TRAFFIC GENERATOR COMPLETE: {total:,} messages sent")
        logger.info("=" * 60)

class TestTokenBucketRateLimiter(unittest.TestCase):

    def test_basic_rate_limiting(self):
        """Tokens should be consumed and refilled at correct rate."""
        limiter = TokenBucketRateLimiter(rate=100, burst=100, name="test")

        # Should be able to acquire burst immediately
        acquired = 0
        for _ in range(100):
            if limiter.try_acquire():
                acquired += 1
        self.assertEqual(acquired, 100)

        # Bucket should be empty now
        self.assertFalse(limiter.try_acquire())

        # Wait for refill
        time.sleep(0.1)  # 100/s × 0.1s = 10 tokens
        acquired = 0
        for _ in range(20):
            if limiter.try_acquire():
                acquired += 1
        # Should get approximately 10 tokens (timing-dependent)
        self.assertGreater(acquired, 5)
        self.assertLessEqual(acquired, 15)

    def test_blocking_acquire(self):
        """Blocking acquire should wait until token available."""
        limiter = TokenBucketRateLimiter(rate=1000, burst=1, name="test-block")

        # Consume the single burst token
        self.assertTrue(limiter.try_acquire())
        self.assertFalse(limiter.try_acquire())

        # Blocking acquire should succeed after refill
        start = time.time()
        result = limiter.acquire(timeout=2.0)
        elapsed = time.time() - start

        self.assertTrue(result)
        self.assertGreater(elapsed, 0.0005)  # Should have waited some time

    def test_acquire_timeout(self):
        """Acquire should return False on timeout."""
        limiter = TokenBucketRateLimiter(rate=1, burst=1, name="test-timeout")

        # Consume the token
        limiter.try_acquire()

        # Try to acquire with very short timeout
        start = time.time()
        result = limiter.acquire(timeout=0.05)
        elapsed = time.time() - start

        # At rate=1/s, need 1s for new token — 0.05s timeout should fail
        self.assertFalse(result)
        self.assertLess(elapsed, 0.2)

    def test_dynamic_rate_change(self):
        """Rate should be adjustable at runtime."""
        limiter = TokenBucketRateLimiter(rate=100, burst=100, name="test-dynamic")

        # Drain bucket
        for _ in range(100):
            limiter.try_acquire()

        limiter.set_rate(10000)
        time.sleep(0.01)  # 10000/s × 0.01s = 100 tokens

        acquired = 0
        for _ in range(200):
