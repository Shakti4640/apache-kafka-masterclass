PROJECT 26: THE RETRY STORM SHELTER — RETRY PATTERNS & BACKOFF

🏗️ HYPOTHETICAL PROBLEM

SCENARIO: Payment Processing Microservice

→ An online marketplace processes 50,000 payment events per hour
→ Each payment event requires:
  1. Read from Kafka topic "payment_events"
  2. Call external Payment Gateway API (Stripe/PayPal)
  3. Update order status in PostgreSQL database
  4. Produce confirmation event to "payment_confirmations"

→ THE REALITY OF DISTRIBUTED SYSTEMS:

  Monday 14:00 — Payment Gateway returns HTTP 503 for 45 seconds
    → 625 payment events fail during that window
    → Project 25's DLQ captures them all
    → But these payments are PERFECTLY VALID
    → Gateway was just temporarily overloaded
    → 45 seconds later, gateway is healthy again
    → Those 625 payments should have been RETRIED, not dead-lettered

  Tuesday 09:30 — PostgreSQL connection pool exhausted for 10 seconds
    → 140 payment events fail on DB write
    → Again, perfectly valid events
    → DB recovers in 10 seconds
    → Events should have been retried

  Wednesday 16:00 — Network blip causes DNS resolution failure
    → 30 events fail
    → Network recovers in 2 seconds
    → Events should have been retried

→ CURRENT BEHAVIOR (Project 25 DLQ only):
  
  TRY once → FAIL → DLQ immediately
  
  Result: 795 valid payments in DLQ over 3 days
  → Manual investigation for each
  → Manual replay after confirming systems are healthy
  → 795 customers waiting for payment confirmation
  → Customer support overwhelmed with "where's my order" calls

→ DESIRED BEHAVIOR:
  
  TRY → FAIL → RETRY after 1s → FAIL → RETRY after 5s →
  FAIL → RETRY after 30s → FAIL → RETRY after 2min →
  FAIL → DLQ (truly unprocessable)
  
  Result: 99% of transient failures auto-recover
  → Only truly broken records reach DLQ
  → Zero manual intervention for temporary outages
  → Customers never notice the 45-second gateway blip

→ BUT THERE'S A TRAP:

  Naive retry = RETRY STORM
  
  → 625 events fail at 14:00
  → All 625 retry at 14:00:01 (1 second later)
  → Gateway STILL recovering → all 625 fail AGAIN
  → All 625 retry at 14:00:02
  → Gateway now has 625 retries + new incoming traffic
  → Gateway collapses under load
  → MORE events start failing
  → CASCADING FAILURE

→ REQUIREMENT:
  1. Retry transient failures automatically
  2. Exponential backoff to prevent retry storms
  3. Maximum retry limit to prevent infinite loops
  4. Distinguish retriable vs non-retriable errors
  5. Track retry count per record
  6. After max retries exhausted → DLQ (Project 25)
  7. Zero manual intervention for transient failures

🔍 HOW TO APPROACH IT

THINKING THROUGH THE PROBLEM:

1. From Project 25 → We have DLQ as the final safety net
   → But DLQ is for PERMANENT failures
   → TRANSIENT failures need a RETRY layer first

2. From Project 6 → We understand offsets and consumer position
   → Cannot "re-read" a specific offset easily
   → Kafka consumers move FORWARD, not backward

3. From Project 4 → We understand message keys and routing
   → Same key = same partition = ordering preserved

4. THE CORE CHALLENGE:
   → Kafka has no built-in "retry after delay" mechanism
   → Once a consumer reads and commits an offset, it's done
   → We need to BUILD retry semantics on top of Kafka

THREE ARCHITECTURAL APPROACHES:

APPROACH A: In-Memory Retry (Simple but Dangerous)
  → Catch error → sleep(backoff) → retry in same thread
  → PROBLEM: Blocks consumer → stops processing ALL records
  → PROBLEM: If consumer crashes, retry state is lost
  → NOT SUITABLE for production

APPROACH B: Retry Topics (Kafka-Native Pattern)
  → Catch error → produce to retry topic with delay metadata
  → Separate consumer reads retry topic after delay
  → PROBLEM: Kafka doesn't support delayed consumption natively
  → SOLUTION: Use multiple retry topics with increasing delays

APPROACH C: Retry Topics with Timestamp-Based Delay ← CHOSEN
  → Catch error → produce to retry topic with:
    - Original record
    - Retry count header
    - "Not before" timestamp header
  → Retry consumer checks timestamp before processing
  → If too early → pause partition, wait, resume
  → Multiple retry levels: retry-1 (1s), retry-2 (10s), retry-3 (60s)
  → Exhausted retries → DLQ (Project 25)

SOLUTION: MULTI-LEVEL RETRY TOPIC PATTERN WITH EXPONENTIAL BACKOFF

✅ WHAT COULD BE DONE

OPTION A: Simple sleep-based retry in consumer thread
  → catch error → Thread.sleep(backoff) → retry
  → PROBLEM: Blocks entire consumer → all partitions stall
  → PROBLEM: Consumer may exceed max.poll.interval.ms → rebalance
  → PROBLEM: No persistence — crash = lost retry state
  → VERDICT: Only for toy/demo systems

OPTION B: Single retry topic with counter
  → One topic: payment_events_retry
  → Header tracks retry count
  → Consumer reads, checks count, waits if needed
  → PROBLEM: All retry levels mixed in one topic
  → PROBLEM: Short-delay retries blocked behind long-delay retries
  → PROBLEM: No partition-level delay control
  → VERDICT: Simpler but creates head-of-line blocking

OPTION C: Multiple retry topics with dedicated consumers ← CHOSEN
  → Separate topic per retry level
  → Each topic has its own consumer with appropriate delay
  → No head-of-line blocking
  → Clear visibility into retry distribution
  → Easy to tune delays per level independently
  → VERDICT: Production standard

OPTION D: External delay queue (Redis, database)
  → Write failed record to Redis with TTL
  → Separate process polls Redis and re-produces to Kafka
  → PROBLEM: Adds external dependency
  → PROBLEM: Redis failure = retry system failure
  → VERDICT: Valid but adds complexity — use when delay precision matters

OPTION E: Kafka Streams with punctuate() for delayed processing
  → Use Kafka Streams processor API with wall-clock punctuation
  → Store failed records in state store
  → Punctuator checks periodically and retries
  → VERDICT: Elegant but complex — Project 19-21 approach

✅ WHAT SHOULD DO

1. CLASSIFY ERRORS AS RETRIABLE VS NON-RETRIABLE
   → RETRIABLE: HTTP 503, connection timeout, pool exhausted,
     DNS failure, lock contention, rate limited (429)
   → NON-RETRIABLE: HTTP 400 (bad request), schema mismatch,
     null pointer, business rule violation, authentication failure
   → Non-retriable errors go DIRECTLY to DLQ — no retry
   → Retrying non-retriable errors wastes resources

2. USE EXPONENTIAL BACKOFF (NOT LINEAR)
   → Linear: 1s, 2s, 3s, 4s → too aggressive for long outages
   → Exponential: 1s, 10s, 60s, 300s → backs off quickly
   → Gives downstream systems time to recover
   → Prevents retry storms

3. ADD JITTER TO BACKOFF
   → Without jitter: 625 records all retry at exactly T+1s
   → With jitter: 625 records retry between T+0.5s and T+1.5s
   → Spreads retry load across time window
   → Prevents thundering herd problem

4. TRACK RETRY COUNT IN HEADERS (NOT IN VALUE)
   → Headers are metadata — don't pollute business payload
   → Downstream consumers can ignore retry headers
   → Easy to inspect without parsing record value

5. PRESERVE ORIGINAL RECORD EXACTLY
   → Retry topic value = EXACT same bytes as original
   → Only headers change (retry count, timestamps)
   → When retry succeeds, downstream sees original record

6. SET APPROPRIATE RETENTION ON RETRY TOPICS
   → Retry topics need retention > max cumulative backoff
   → If max backoff = 6 minutes → retention = 1 hour minimum
   → If records expire before retry → silent data loss

7. MONITOR RETRY TOPIC DEPTHS
   → Growing retry queues = downstream system struggling
   → Alert on: retry_topic_records > threshold
   → Connects to monitoring from Project 28

8. IMPLEMENT CIRCUIT BREAKER ALONGSIDE RETRY
   → If retry rate > 50% → downstream is probably DOWN
   → Pause main consumer → stop sending new requests
   → Resume when health check passes
   → Prevents overwhelming a struggling system

❌ WHAT SHOULD NOT DO

1. DO NOT RETRY INDEFINITELY
   → Infinite retries = infinite resource consumption
   → Set max_retries (4-5 is typical)
   → After max → DLQ (Project 25)
   → Some records are truly unprocessable

2. DO NOT USE FIXED DELAY FOR ALL RETRY LEVELS
   → retry_1: 10s, retry_2: 10s, retry_3: 10s
   → This is LINEAR, not exponential
   → Doesn't give downstream enough recovery time
   → Gateway needs 45s to recover but you're hammering it every 10s

3. DO NOT RETRY NON-RETRIABLE ERRORS
   → HTTP 400 Bad Request → will ALWAYS fail
   → Schema validation error → data is wrong, not transient
   → Null pointer → code bug, not infrastructure issue
   → Retrying these wastes resources and delays DLQ investigation

4. DO NOT BLOCK THE MAIN CONSUMER WHILE RETRYING
   → Thread.sleep(60000) in consumer loop = 60s of no processing
   → max.poll.interval.ms default = 300s
   → Multiple sleeps = rebalance trigger (Project 10)
   → Use separate retry topics/consumers instead

5. DO NOT FORGET TO COMMIT OFFSETS AFTER ROUTING TO RETRY
   → Main consumer reads record → fails → produces to retry_1
   → If you don't commit the original offset
   → Consumer restarts → reads same record again
   → Produces ANOTHER copy to retry_1
   → DUPLICATE retries

6. DO NOT CREATE RETRY TOPICS WITH TOO MANY PARTITIONS
   → Retry topics have LOW throughput (only failed records)
   → 1-3 partitions per retry topic is sufficient
   → More partitions = wasted broker resources

7. DO NOT MIX RETRY LOGIC WITH BUSINESS LOGIC
   → Retry routing should be a SEPARATE layer
   → Business processor should not know about retry topics
   → Clean separation: processor → error handler → retry router

8. DO NOT RETRY ACROSS DIFFERENT RECORD KEYS WITHOUT CAUTION
   → Key "user_123" fails → retried
   → Meanwhile new record for "user_123" arrives on main topic
   → Retry succeeds AFTER the new record is processed
   → OUT-OF-ORDER processing for that key
   → If ordering matters: pause partition for that key during retry

🚫 WHEN NOT TO USE

1. WHEN ERROR IS DETERMINISTIC
   → Same input ALWAYS produces same error
   → Retrying is guaranteed to fail
   → Example: "amount" field is string "fifty dollars"
   → No amount of waiting will make that parseable
   → Send to DLQ immediately

2. WHEN ORDERING IS STRICTLY REQUIRED
   → Record A (offset 100) fails → goes to retry
   → Record B (offset 101) succeeds → processed
   → Record A retries and succeeds → processed AFTER B
   → If A must be before B → retry pattern breaks ordering
   → Use errors.tolerance=none (Project 25) + manual handling

3. WHEN DOWNSTREAM IS PERMANENTLY DOWN
   → Service decommissioned but nobody updated config
   → Retrying for 6 minutes won't help
   → All records exhaust retries → all go to DLQ
   → Circuit breaker (WHAT SHOULD DO #8) is better here

4. WHEN RETRY VOLUME EXCEEDS MAIN VOLUME
   → If 80% of records need retry
   → The SYSTEM is broken, not individual records
   → Stop the pipeline → fix the system → resume
   → Retry pattern is for OCCASIONAL transient failures

5. WHEN COST OF RETRY > COST OF FAILURE
   → Sending marketing emails: failed email → just skip
   → Analytics event: one missed event → acceptable
   → Retry infrastructure cost > business impact of loss
   → Use simple DLQ without retry

  METHOD 1: Partition Pause/Resume (CHOSEN)
  ──────────────────────────────────────────
  
  consumer.poll() returns record
      │
      ▼
  Extract X-Retry-Not-Before header
      │
      ▼
  Is current_time >= not_before?
      │
    ┌─┴──┐
    │    │
   YES   NO
    │    │
    ▼    ▼
  Process  PAUSE this partition
  record   │
    │      ├→ consumer.pause([TopicPartition])
    │      ├→ sleep(remaining_delay)
    │      ├→ consumer.resume([TopicPartition])
    │      └→ Next poll() returns same record
    │
    ▼
  SUCCESS → commit → done
  FAILURE → route to retry_2 (or DLQ if exhausted)
  
  
  WHY PAUSE/RESUME:
  → consumer.pause() stops fetching from specific partition
  → Other partitions continue processing (no blocking)
  → consumer.resume() re-enables fetching
  → Record is NOT committed → will be re-delivered
  → Precise delay enforcement per-partition
  
  
  METHOD 2: Poll-Check-Skip Loop (Alternative)
  ─────────────────────────────────────────────
  
  while True:
      records = consumer.poll()
      for record in records:
          if current_time < record.not_before:
              # Don't commit — will see again
              # But this creates busy-wait loop
              sleep(100ms)
              break
          else:
              process(record)
              commit(record)
  
  PROBLEM: Busy-wait wastes CPU
  PROBLEM: If many delayed records, loop spins uselessly
  METHOD 1 (pause/resume) is more efficient

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: RETRY CONSUMER FASTER THAN DELAY
════════════════════════════════════════════
  → Retry consumer polls record
  → X-Retry-Not-Before is 10 seconds from now
  → Consumer processes immediately (ignores delay)
  → Record fails AGAIN (downstream still recovering)
  → Sent to retry_3 → also processed immediately
  → Entire retry chain exhausted in milliseconds
  → DLQ receives record that should have waited 6 minutes
  
  FIX: ALWAYS enforce delay before processing
       Use pause/resume mechanism described in internals


PITFALL 2: RETRY TOPIC PARTITION MISMATCH
═════════════════════════════════════════
  → Main topic: 6 partitions
  → Retry topic: 3 partitions (different count)
  → Record with key "pay_123" → partition 4 on main topic
  → Same key "pay_123" → partition 1 on retry topic
  → Different partition = different ordering guarantees
  
  FIX: Retry topics should have SAME partition count as source
       OR accept that ordering may differ in retry path


PITFALL 3: CONSUMER GROUP CONFLICT
═══════════════════════════════════
  → Main consumer group: "payment-processor"
  → Retry consumer group: also "payment-processor" (copy-paste error)
  → Kafka assigns retry topic partitions to main consumer
  → Main consumer tries to process retry records as fresh records
  → Retry headers confuse main processing logic
  
  FIX: UNIQUE group ID per consumer:
       "payment-processor-main"
       "payment-processor-retry-1"
       "payment-processor-retry-2"


PITFALL 4: OFFSET COMMIT BEFORE RETRY PRODUCE
══════════════════════════════════════════════
  → Main consumer reads record, processing fails
  → Commits offset FIRST
  → Then tries to produce to retry topic
  → Retry produce FAILS (broker down?)
  → Offset committed → record won't be re-read
  → Record LOST — not in main flow, not in retry, not in DLQ
  
  FIX: Produce to retry FIRST → wait for ack → THEN commit offset
       (Same pattern as Project 25 DLQ write)


PITFALL 5: RETRY HEADER ACCUMULATION
═════════════════════════════════════
  → Each retry level adds headers
  → After 4 retries: 40+ headers per record
  → Kafka default max.header.size = 1MB (usually fine)
  → But: header parsing overhead increases
  → And: error messages in headers can be LONG
  
  FIX: Truncate error messages in headers (max 500 chars)
       Only keep latest error, not full history
       Or use a separate error log keyed by original offset


PITFALL 6: THUNDERING HERD ON RETRY CONSUMER RESTART
═════════════════════════════════════════════════════
  → Retry consumer was down for 30 minutes
  → 5000 records accumulated in retry topics
  → Consumer restarts → reads all 5000 at once
  → All 5000 have X-Retry-Not-Before in the past
  → All 5000 processed simultaneously
  → Downstream overwhelmed
  
  FIX: Rate limit retry consumer processing
       max.poll.records=10 for retry consumers
       Add artificial throttle: process N per second


PITFALL 7: RETRY RECORDS OUTLIVE SOURCE CONTEXT
════════════════════════════════════════════════
  → Order #789 fails at 14:00 → enters retry chain
  → Customer CANCELS order at 14:02
  → Retry succeeds at 14:05 → processes cancelled order
  → Payment charged for cancelled order
  
  FIX: Before processing retry record, validate current state
       Check: "is this order still active?"
       This requires a state lookup — adds complexity
       But prevents processing stale/outdated events

📈 SCALABILITY CONSIDERATIONS

TOPIC COUNT:
════════════
  → 4 retry levels × N pipelines = 4N additional topics
  → 10 pipelines = 40 retry topics + 10 DLQ topics = 50 extra topics
  → Each topic has partitions, replicas, metadata
  → At 100+ pipelines: consider shared retry infrastructure
  
RETRY VOLUME:
═════════════
  → Healthy system: < 0.1% of records enter retry
  → Under stress: 5-20% may enter retry
  → Retry topics sized for PEAK, not average
  → Partition count: 1-3 per retry topic (low volume)
  → During outage: retry topics may accumulate thousands
  → Must handle burst when downstream recovers

CONSUMER COUNT:
═══════════════
  → Main consumer: scaled for full throughput (multiple instances)
  → Retry consumers: 1 instance per level is usually sufficient
  → Retry processing is LOW throughput by design
  → Over-scaling retry consumers wastes resources
  
  → EXCEPTION: During recovery from long outage
    → Retry topics may have thousands of records
    → Temporarily scale retry consumers to drain backlog
    → Scale back down after backlog cleared

STORAGE:
════════
  → Retry records = same size as original records
  → 50K records/hour × 0.1% retry rate = 50 records/hour in retry
  → At 1KB per record: 50KB/hour → negligible
  → During outage: 50K × 20% = 10K records/hour in retry
  → At 1KB: 10MB/hour → still manageable
  → Set retention: 1 hour for retry topics (records either succeed or DLQ)

BACKOFF TUNING BY SYSTEM:
═════════════════════════
  → Fast-recovering systems (network blips):
    Levels: 100ms, 1s, 5s, 30s
  → Medium-recovering systems (service restarts):
    Levels: 1s, 10s, 60s, 300s
  → Slow-recovering systems (database maintenance):
    Levels: 10s, 60s, 300s, 900s
  → Tune based on observed recovery patterns

class RetriableError(Exception):
    """Errors that should trigger retry"""
    pass

class NonRetriableError(Exception):
    """Errors that should go directly to DLQ"""
    pass

RETRIABLE_ERROR_PATTERNS = [
    'ConnectionError',
    'TimeoutError',
    'HTTP 503',
    'HTTP 429',
    'HTTP 502',
    'HTTP 504',
    'pool exhausted',
    'connection refused',
    'DNS resolution',
    'temporarily unavailable',
    'circuit breaker open',
    'lock contention',
    'deadlock',
]

NON_RETRIABLE_ERROR_PATTERNS = [
    'HTTP 400',
    'HTTP 401',
    'HTTP 403',
    'HTTP 404',
    'HTTP 422',
    'schema validation',
    'null pointer',
    'type error',
    'parse error',

]

def classify_error(error):
    """
    Determine if an error is retriable or not
    Returns: ('RETRIABLE', reason) or ('NON_RETRIABLE', reason)
    """
    error_str = str(error).lower()
    error_class = type(error).__name__
    
    # Check explicit types first
    if isinstance(error, RetriableError):
        return 'RETRIABLE', f'Explicit RetriableError: {error}'
    
    if isinstance(error, NonRetriableError):
        return 'NON_RETRIABLE', f'Explicit NonRetriableError: {error}'
    
    # Pattern matching
    for pattern in RETRIABLE_ERROR_PATTERNS:
        if pattern.lower() in error_str:
            return 'RETRIABLE', f'Matched retriable pattern: {pattern}'
    
    for pattern in NON_RETRIABLE_ERROR_PATTERNS:
        if pattern.lower() in error_str:
            return 'NON_RETRIABLE', f'Matched non-retriable pattern: {pattern}'
    
    # Default: retry once for unknown errors
    return 'RETRIABLE', f'Unknown error type ({error_class}) — defaulting to retriable'

def calculate_delay_ms(retry_level, config=RETRY_CONFIG):
    """
    Calculate delay with exponential backoff + jitter
    
    Formula: base_delay * jitter_random
    Where jitter_random = uniform(1 - jitter_factor, 1 + jitter_factor)
    """
    if retry_level not in config['levels']:
        return config['max_delay_ms']
    
    base_delay = config['levels'][retry_level]['base_delay_ms']
    
    # Apply jitter
    jitter_min = 1.0 - config['jitter_factor']
    jitter_max = 1.0 + config['jitter_factor']
    jitter_multiplier = random.uniform(jitter_min, jitter_max)
    
    delay = int(base_delay * jitter_multiplier)
    
    # Cap at maximum
    delay = min(delay, config['max_delay_ms'])
    
    return delay

def get_retry_topic(source_topic, retry_level, config=RETRY_CONFIG):
    """Get the retry topic name for a given level"""
    if retry_level > config['max_retries']:
        return config['dlq_topic']
    
    suffix = config['levels'][retry_level]['topic_suffix']
    return f"{source_topic}{suffix}"

def extract_retry_headers(msg):
    """Extract retry metadata from message headers"""
    headers = {}
    for key, value in (msg.headers() or []):
        try:
            headers[key] = value.decode('utf-8') if value else ''
        except UnicodeDecodeError:
            headers[key] = value.hex() if value else ''
    return headers

def get_retry_count(msg):
    """Get current retry count from headers (0 if first attempt)"""
    headers = extract_retry_headers(msg)
    try:
        return int(headers.get('X-Retry-Count', '0'))
    except ValueError:
        return 0

def build_retry_headers(original_msg, retry_count, delay_ms, error_message, error_class):
    """Build complete retry header set"""
    now_ms = int(time.time() * 1000)
    not_before = now_ms + delay_ms
    
    # Preserve original metadata from first attempt
    existing_headers = extract_retry_headers(original_msg)
    
    original_topic = existing_headers.get(
        'X-Original-Topic', 
        original_msg.topic()
    )
    original_partition = existing_headers.get(
        'X-Original-Partition', 
        str(original_msg.partition())
    )
    original_offset = existing_headers.get(
        'X-Original-Offset', 
        str(original_msg.offset())
    )
    original_timestamp = existing_headers.get(
        'X-Original-Timestamp',
        str(original_msg.timestamp()[1] or now_ms)
    )
    first_error_time = existing_headers.get(
        'X-First-Error-Time', 
        str(now_ms)
    )
    
    headers = [
        ('X-Retry-Count', str(retry_count).encode('utf-8')),
        ('X-Retry-Max', str(RETRY_CONFIG['max_retries']).encode('utf-8')),
        ('X-Retry-Not-Before', str(not_before).encode('utf-8')),
        ('X-Retry-Delay-Ms', str(delay_ms).encode('utf-8')),
        ('X-Original-Topic', original_topic.encode('utf-8')),
        ('X-Original-Partition', original_partition.encode('utf-8')),
        ('X-Original-Offset', original_offset.encode('utf-8')),
        ('X-Original-Timestamp', original_timestamp.encode('utf-8')),
        ('X-First-Error-Time', first_error_time.encode('utf-8')),
        ('X-Last-Error-Time', str(now_ms).encode('utf-8')),
        ('X-Last-Error-Message', error_message[:500].encode('utf-8')),
        ('X-Last-Error-Class', error_class.encode('utf-8')),
    ]
    
    return headers

def build_dlq_headers(original_msg, retry_count, error_message):
    """Build DLQ headers when retries are exhausted"""
    now_ms = int(time.time() * 1000)
    existing_headers = extract_retry_headers(original_msg)
    
    first_error_time = int(existing_headers.get('X-First-Error-Time', str(now_ms)))
    total_duration = now_ms - first_error_time
    
    headers = [
        ('X-Retry-Count', str(retry_count).encode('utf-8')),
        ('X-Retry-Max', str(RETRY_CONFIG['max_retries']).encode('utf-8')),
        ('X-Retry-Exhausted', b'true'),
        ('X-Total-Retry-Duration-Ms', str(total_duration).encode('utf-8')),
        ('X-Original-Topic', existing_headers.get(
            'X-Original-Topic', original_msg.topic()
        ).encode('utf-8')),
        ('X-Original-Offset', existing_headers.get(
            'X-Original-Offset', str(original_msg.offset())
        ).encode('utf-8')),
        ('X-First-Error-Time', str(first_error_time).encode('utf-8')),
        ('X-Last-Error-Time', str(now_ms).encode('utf-8')),
        ('X-Last-Error-Message', error_message[:500].encode('utf-8')),
        ('X-DLQ-Reason', b'RETRY_EXHAUSTED'),
    ]
    
    return headers

def format_duration(ms):
    """Format milliseconds into human-readable string"""
    if ms < 1000:
        return f"{ms}ms"
    elif ms < 60000:
        return f"{ms/1000:.1f}s"
    elif ms < 3600000:
        return f"{ms/60000:.1f}min"
    else:
        return f"{ms/3600000:.1f}hr"

import time
import random
import threading

class PaymentGateway:
    """
    Simulates external payment API with configurable failure modes
    
    Failure scenarios:
    1. Periodic outages (HTTP 503 for N seconds)
    2. Rate limiting (HTTP 429 when too many requests)
    3. Random transient errors (connection timeouts)
    4. Permanent failures for specific amounts (HTTP 400)
    """
    
    def __init__(self):
        self.request_count = 0
        self.success_count = 0
        self.failure_count = 0
        self._lock = threading.Lock()
        
        # Outage simulation
        self.outage_active = False
        self.outage_start = 0
        self.outage_duration = 0
        
        # Rate limiting
        self.requests_this_second = 0
        self.rate_limit_per_second = 50
        self.last_rate_reset = time.time()
        
        # Random failure probability (simulates network issues)
        self.random_failure_rate = 0.02  # 2% random failures
        
        # Schedule periodic outages
        self._schedule_outage()
    
    def _schedule_outage(self):
        """Schedule a simulated outage after random delay"""
        def trigger_outage():
            time.sleep(random.uniform(10, 30))  # Outage after 10-30 seconds
            self.outage_active = True
            self.outage_start = time.time()
            self.outage_duration = random.uniform(3, 8)  # 3-8 second outage
        
        t = threading.Thread(target=trigger_outage, daemon=True)
        t.start()
    
    def process_payment(self, order_id, amount, currency='USD'):
        """
        Simulate payment processing
        
        Returns: (success: bool, response: dict)
        Raises: RetriableError or NonRetriableError
        """
        with self._lock:
            self.request_count += 1
            
            # Reset rate counter every second
            now = time.time()
            if now - self.last_rate_reset >= 1.0:
                self.requests_this_second = 0
                self.last_rate_reset = now
            self.requests_this_second += 1
        
        # --- Check outage ---
        if self.outage_active:
            elapsed = time.time() - self.outage_start
            if elapsed < self.outage_duration:
                self.failure_count += 1
                from retry_config import RetriableError
                raise RetriableError(
                    f"HTTP 503 Service Unavailable — "
                    f"gateway outage ({elapsed:.1f}s / {self.outage_duration:.1f}s)"
                )
            else:
                self.outage_active = False
                self._schedule_outage()  # Schedule next outage
        
        # --- Check rate limit ---
        if self.requests_this_second > self.rate_limit_per_second:
            self.failure_count += 1
            from retry_config import RetriableError
            raise RetriableError(
                f"HTTP 429 Too Many Requests — "
                f"{self.requests_this_second} requests/sec exceeds limit "
                f"of {self.rate_limit_per_second}/sec"
            )
        
        # --- Check for non-retriable errors ---
        from retry_config import NonRetriableError
        
        if amount is None or not isinstance(amount, (int, float)):
            self.failure_count += 1
            raise NonRetriableError(
                f"HTTP 400 Bad Request — invalid amount: {repr(amount)}"
            )
        
        if amount <= 0:
            self.failure_count += 1
            raise NonRetriableError(
                f"HTTP 422 Unprocessable — amount must be positive: {amount}"
            )
        
        if amount > 50000:
            self.failure_count += 1
            raise NonRetriableError(
                f"HTTP 400 Bad Request — amount exceeds maximum: ${amount}"
            )
        
        # --- Random transient failure ---
        if random.random() < self.random_failure_rate:
            self.failure_count += 1
            from retry_config import RetriableError
            raise RetriableError(
                "ConnectionError — connection reset by peer (transient)"
            )
        
        # --- Simulate processing time ---
        time.sleep(random.uniform(0.001, 0.005))
        
        # --- SUCCESS ---
        self.success_count += 1
        return True, {
            'transaction_id': f"txn_{order_id}_{int(time.time()*1000)}",
            'status': 'COMPLETED',
            'amount': amount,
            'currency': currency,
            'processed_at': int(time.time() * 1000)
        }
    
    def get_stats(self):
        return {
            'total_requests': self.request_count,
            'successes': self.success_count,
            'failures': self.failure_count,
            'outage_active': self.outage_active,
            'success_rate': (
                f"{self.success_count/self.request_count*100:.1f}%"
                if self.request_count > 0 else "N/A"
            )
        }

    """
    
    def __init__(self, producer=None):
        self.producer = producer or Producer({
            'bootstrap.servers': RETRY_CONFIG['bootstrap_servers'],
            'acks': 'all',
            'retries': 3,
            'retry.backoff.ms': 500
        })
        
        self.stats = {
            'routed_to_retry': 0,
            'routed_to_dlq_exhausted': 0,
            'routed_to_dlq_non_retriable': 0,
            'route_failures': 0,
        }
    
    def route_failed_record(self, original_msg, error, source_consumer_type='main'):
        """
        Route a failed record to the appropriate destination
        
        Logic:
        1. Classify error → retriable or not
        2. If non-retriable → DLQ immediately
        3. If retriable → check retry count
        4. If retries exhausted → DLQ
        5. If retries available → next retry topic with delay
        """
        
        error_class, classification_reason = classify_error(error)
        error_message = str(error)
        current_retry_count = get_retry_count(original_msg)
        
        # ============================================
        # PATH 1: Non-retriable → DLQ immediately
        # ============================================
        if error_class == 'NON_RETRIABLE':
            dlq_headers = build_dlq_headers(
                original_msg, current_retry_count, error_message
            )
            # Override DLQ reason
            dlq_headers.append(('X-DLQ-Reason', b'NON_RETRIABLE'))
            
            success = self._produce_with_ack(
                topic=RETRY_CONFIG['dlq_topic'],
                key=original_msg.key(),
                value=original_msg.value(),
                headers=dlq_headers
            )
            
            if success:
                self.stats['routed_to_dlq_non_retriable'] += 1
                key_str = original_msg.key().decode('utf-8') if original_msg.key() else '?'
                print(f"    ❌ NON-RETRIABLE → DLQ: key={key_str}")
                print(f"       Reason: {classification_reason}")
            else:
                self.stats['route_failures'] += 1
                print(f"    🚨 FAILED TO WRITE TO DLQ — RECORD MAY BE LOST")
            
            return 'DLQ_NON_RETRIABLE', success
        
        # ============================================
        # PATH 2: Retriable but exhausted → DLQ
        # ============================================
        next_retry_count = current_retry_count + 1
        
        if next_retry_count > RETRY_CONFIG['max_retries']:
            dlq_headers = build_dlq_headers(
                original_msg, current_retry_count, error_message
            )
            
            success = self._produce_with_ack(
                topic=RETRY_CONFIG['dlq_topic'],
                key=original_msg.key(),
                value=original_msg.value(),
                headers=dlq_headers
            )
            
            if success:
                self.stats['routed_to_dlq_exhausted'] += 1
                key_str = original_msg.key().decode('utf-8') if original_msg.key() else '?'
                existing = extract_retry_headers(original_msg)
                first_error = int(existing.get('X-First-Error-Time', '0'))
                total_time = int(time.time() * 1000) - first_error if first_error else 0
                
                print(f"    💀 RETRIES EXHAUSTED → DLQ: key={key_str}")
                print(f"       Attempts: {current_retry_count}/{RETRY_CONFIG['max_retries']}")
                print(f"       Total retry duration: {format_duration(total_time)}")
                print(f"       Last error: {error_message[:80]}")
            else:
                self.stats['route_failures'] += 1
                print(f"    🚨 FAILED TO WRITE TO DLQ — RECORD MAY BE LOST")
            
            return 'DLQ_EXHAUSTED', success
        
        # ============================================
        # PATH 3: Retriable with retries remaining → Retry topic
        # ============================================
        delay_ms = calculate_delay_ms(next_retry_count)
        retry_topic = get_retry_topic(
            RETRY_CONFIG['source_topic'], next_retry_count
        )
        
        retry_headers = build_retry_headers(
            original_msg, next_retry_count, delay_ms, error_message, error_class
        )
        
        success = self._produce_with_ack(
            topic=retry_topic,
            key=original_msg.key(),
            value=original_msg.value(),
            headers=retry_headers
        )
        
        if success:
            self.stats['routed_to_retry'] += 1
            key_str = original_msg.key().decode('utf-8') if original_msg.key() else '?'
            level_desc = RETRY_CONFIG['levels'][next_retry_count]['description']
            
            print(f"    🔄 RETRY {next_retry_count}/{RETRY_CONFIG['max_retries']} → "
                  f"{retry_topic}: key={key_str}")
            print(f"       Delay: {format_duration(delay_ms)} ({level_desc})")
            print(f"       Error: {error_message[:60]}")
        else:
            self.stats['route_failures'] += 1
            print(f"    🚨 FAILED TO WRITE TO RETRY TOPIC — RECORD MAY BE LOST")
        
        return 'RETRY', success
    
    def _produce_with_ack(self, topic, key, value, headers):
        """
        Synchronous produce with delivery confirmation
        CRITICAL: Must confirm write before caller commits offset
        (Lesson from Project 25 — Pitfall 6)
        """
        delivery_result = {'success': False, 'error': None}
        
        def on_delivery(err, msg):
            if err:
                delivery_result['error'] = err
            else:
                delivery_result['success'] = True
        
        try:
            self.producer.produce(
                topic=topic,
                key=key,
                value=value,
                headers=headers,
                callback=on_delivery
            )
            # SYNCHRONOUS FLUSH — blocks until delivery confirmed
            self.producer.flush(timeout=10)
            
            return delivery_result['success']
        
        except Exception as e:
            print(f"    Producer exception: {e}")
            return False
    
    def get_stats(self):
        return dict(self.stats)


class DelayEnforcer:
    """
    Enforces delay on retry consumers using pause/resume mechanism
    
    When a record arrives with X-Retry-Not-Before in the future:
    1. Pause the partition
    2. Wait until the delay expires
    3. Resume the partition
    4. Re-poll to get the same record
    """
    
    def __init__(self, consumer):
        self.consumer = consumer
        self.paused_partitions = {}  # tp → resume_time
        self.stats = {
            'delays_enforced': 0,
            'total_delay_ms': 0,
            'records_ready': 0,
        }
    
    def check_and_enforce_delay(self, msg):
        """
        Check if record is ready for processing
        
        Returns:
            True  → record is ready, process it
            False → record is delayed, partition paused, skip it
        """
        headers = extract_retry_headers(msg)
        not_before_str = headers.get('X-Retry-Not-Before', '0')
        
        try:
            not_before_ms = int(not_before_str)
        except ValueError:
            # No valid delay header → process immediately
            self.stats['records_ready'] += 1
            return True
        
        now_ms = int(time.time() * 1000)
        
        if now_ms >= not_before_ms:
            # Delay has passed → ready to process
            self.stats['records_ready'] += 1
            return True
        
        # Delay not met → pause partition and wait
        remaining_ms = not_before_ms - now_ms
        tp = TopicPartition(msg.topic(), msg.partition())
        
        self.stats['delays_enforced'] += 1
        self.stats['total_delay_ms'] += remaining_ms
        
        key_str = msg.key().decode('utf-8') if msg.key() else '?'
        retry_count = headers.get('X-Retry-Count', '?')
        
        print(f"    ⏳ DELAY ENFORCED: key={key_str}, "
              f"retry={retry_count}, "
              f"waiting {format_duration(remaining_ms)}")
        
        # Pause this specific partition
        self.consumer.pause([tp])
        self.paused_partitions[tp] = not_before_ms
        
        return False
    
    def resume_ready_partitions(self):
        """
        Check all paused partitions and resume those whose delay expired
        Call this in the main poll loop
        """
        if not self.paused_partitions:
            return
        
        now_ms = int(time.time() * 1000)
        to_resume = []
        
        for tp, resume_time in list(self.paused_partitions.items()):
            if now_ms >= resume_time:
                to_resume.append(tp)
        
        if to_resume:
            self.consumer.resume(to_resume)
            for tp in to_resume:
                del self.paused_partitions[tp]
    
    def get_paused_count(self):
        return len(self.paused_partitions)
    
    def get_stats(self):
        stats = dict(self.stats)
        stats['currently_paused'] = len(self.paused_partitions)
        if stats['delays_enforced'] > 0:
            stats['avg_delay_ms'] = stats['total_delay_ms'] / stats['delays_enforced']
        return stats

from confluent_kafka import Consumer, Producer, KafkaError
import json
import time

from retry_config import RETRY_CONFIG, format_duration

BOOTSTRAP = RETRY_CONFIG['bootstrap_servers']

consumer = Consumer({

    'enable.auto.commit': False,

def process_payment(record_value):

    Process a single payment event through the gateway
    Returns (success, result_or_error)
    """
    data = json.loads(record_value.decode('utf-8'))
    
    order_id = data.get('order_id')
    amount = data.get('amount')
    currency = data.get('currency', 'USD')
    
    # Call the (simulated) payment gateway
    success, response = gateway.process_payment(order_id, amount, currency)
    
    return success, response, data

def produce_confirmation(original_data, gateway_response):

    confirmation = {
        'order_id': original_data['order_id'],
        'customer_id': original_data.get('customer_id'),
        'transaction_id': gateway_response['transaction_id'],
        'amount': gateway_response['amount'],
        'status': 'PAYMENT_CONFIRMED',
        'processed_at': gateway_response['processed_at']
    }
    
    key = str(original_data['order_id']).encode('utf-8')
    value = json.dumps(confirmation).encode('utf-8')
    
    success_producer.produce(
        topic=SUCCESS_TOPIC,
        key=key,
        value=value
    )

# Main Consumer Loop
# ============================================================

def main():
    consumer.subscribe([SOURCE_TOPIC])
    
    stats = {
        'total': 0,
        'success': 0,
        'sent_to_retry': 0,
        'sent_to_dlq': 0,
        'deser_errors': 0,
    }
    
    print("=" * 70)
    print("  💳 PAYMENT PROCESSOR — MAIN CONSUMER")
    print("=" * 70)
    print(f"  Source:  {SOURCE_TOPIC}")
    print(f"  Output:  {SUCCESS_TOPIC}")
    print(f"  Retry:   {SOURCE_TOPIC}_retry_1..4")
    print(f"  DLQ:     {RETRY_CONFIG['dlq_topic']}")
    print(f"  Max retries: {RETRY_CONFIG['max_retries']}")
    print("=" * 70)
    
    start_time = time.time()
    empty_polls = 0
    
    try:
        while empty_polls < 20:
            msg = consumer.poll(1.0)
            
            if msg is None:
                empty_polls += 1
                continue
            
            if msg.error():
                if msg.error().code() == KafkaError._PARTITION_EOF:
                    empty_polls += 1
                    continue
                print(f"  Consumer error: {msg.error()}")
                continue
            
            empty_polls = 0
            stats['total'] += 1
            
            record_id = f"P{msg.partition()}:O{msg.offset()}"
            key_str = msg.key().decode('utf-8') if msg.key() else '?'
            
            # ============================================
            # STEP 1: Deserialize
            # ============================================
            try:
                data = json.loads(msg.value().decode('utf-8'))
            except (json.JSONDecodeError, UnicodeDecodeError) as e:
                stats['deser_errors'] += 1
                # Deserialization errors are non-retriable (Project 25 pattern)
                from retry_config import NonRetriableError
                retry_router.route_failed_record(
                    msg, NonRetriableError(f"Deserialization failed: {e}")
                )
                consumer.commit(message=msg)
                continue
            
            # ============================================
            # STEP 2: Process through gateway
            # ============================================
            try:
                success, response, original_data = process_payment(msg.value())
                
                # SUCCESS PATH
                produce_confirmation(original_data, response)
                success_producer.flush(timeout=5)
                
                stats['success'] += 1
                consumer.commit(message=msg)
                
                if stats['success'] % 25 == 0:
                    elapsed = time.time() - start_time
                    rate = stats['success'] / elapsed if elapsed > 0 else 0
                    print(f"\n  [{record_id}] ✅ Payment confirmed: "
                          f"order={data.get('order_id')}, "
                          f"amount=${data.get('amount', 0):.2f} "
                          f"({rate:.1f} rec/s)")
            
            except Exception as proc_error:
                # ============================================
                # STEP 3: Route failure through retry engine
                # ============================================
                print(f"\n  [{record_id}] ⚠️  Processing failed: "
                      f"order={data.get('order_id', '?')}")
                
                route_result, route_success = retry_router.route_failed_record(
                    msg, proc_error, source_consumer_type='main'
                )
                
                if route_result.startswith('RETRY'):
                    stats['sent_to_retry'] += 1
                elif route_result.startswith('DLQ'):
                    stats['sent_to_dlq'] += 1
                
                # CRITICAL: Commit offset AFTER successful route
                # (Pattern from Project 25)
                if route_success:
                    consumer.commit(message=msg)
                else:
                    # Route failed — don't commit — will reprocess
                    print(f"    ⚠️  Not committing offset — will reprocess")
    
    except KeyboardInterrupt:
        print("\n  Shutting down...")
    
    finally:
        consumer.close()
        success_producer.flush()
    
    # ============================================
    # Summary
    # ============================================
    elapsed = time.time() - start_time
    
    print(f"\n{'=' * 70}")
    print(f"  MAIN PROCESSOR SUMMARY")
    print(f"{'=' * 70}")
    print(f"  Duration:           {format_duration(int(elapsed * 1000))}")
    print(f"  Total consumed:     {stats['total']}")
    print(f"  ✅ Successful:      {stats['success']}")
    print(f"  🔄 Sent to retry:   {stats['sent_to_retry']}")
    print(f"  ❌ Sent to DLQ:     {stats['sent_to_dlq']}")
    print(f"  💀 Deser errors:    {stats['deser_errors']}")
    
    router_stats = retry_router.get_stats()
    print(f"\n  Retry Router:")
    for k, v in router_stats.items():
        print(f"    {k}: {v}")
    
    gw_stats = gateway.get_stats()
    print(f"\n  Gateway:")
    for k, v in gw_stats.items():
        print(f"    {k}: {v}")
    
    print(f"{'=' * 70}")


if __name__ == '__main__':
    main()

from confluent_kafka import Consumer, Producer, KafkaError
import json

)

class RetryLevelConsumer:
    """
    Consumes from a specific retry level topic
    Enforces delay, retries processing, routes further failures
    """
    
    def __init__(self, retry_level):
        if retry_level not in RETRY_CONFIG['levels']:
            raise ValueError(f"Invalid retry level: {retry_level}")
        
        self.retry_level = retry_level
        self.level_config = RETRY_CONFIG['levels'][retry_level]
        self.retry_topic = f"{RETRY_CONFIG['source_topic']}{self.level_config['topic_suffix']}"
        
        self.consumer = Consumer({
            'bootstrap.servers': RETRY_CONFIG['bootstrap_servers'],
            'group.id': f'payment-processor-retry-{retry_level}',
            'auto.offset.reset': 'earliest',
            'enable.auto.commit': False,
            'max.poll.records': 10,  # Low batch size for retry consumers
            'max.poll.interval.ms': 600000  # Extended for long delays
        })
        
        self.success_producer = Producer({
            'bootstrap.servers': RETRY_CONFIG['bootstrap_servers'],
            'acks': 'all'
        })
        
        self.retry_router = RetryRouter()
        self.delay_enforcer = DelayEnforcer(self.consumer)
        self.gateway = PaymentGateway()
        
        # Lower random failure rate for retry consumers
        # (simulate that transient issues are more likely resolved)
        self.gateway.random_failure_rate = 0.01
        
        self.stats = {
            'total': 0,
            'success': 0,
            'delayed': 0,
            'sent_to_next_retry': 0,
            'sent_to_dlq': 0,
        }
    
    def process_payment(self, msg_value):
        """Process payment through gateway (same as main consumer)"""
        data = json.loads(msg_value.decode('utf-8'))
        
        order_id = data.get('order_id')
        amount = data.get('amount')
        currency = data.get('currency', 'USD')
        
        success, response = self.gateway.process_payment(order_id, amount, currency)
        return success, response, data
    
    def produce_confirmation(self, original_data, gateway_response):
        """Produce success confirmation"""
        confirmation = {
            'order_id': original_data['order_id'],
            'customer_id': original_data.get('customer_id'),
            'transaction_id': gateway_response['transaction_id'],
            'amount': gateway_response['amount'],
            'status': 'PAYMENT_CONFIRMED',
            'retry_level_succeeded': self.retry_level,
            'processed_at': gateway_response['processed_at']
        }
        
        key = str(original_data['order_id']).encode('utf-8')
        value = json.dumps(confirmation).encode('utf-8')
        
        self.success_producer.produce(
            topic=RETRY_CONFIG['success_topic'],
            key=key,
            value=value
        )
        self.success_producer.flush(timeout=5)
    
    def run(self, max_empty_polls=30):
        """Main processing loop for this retry level"""
        
        self.consumer.subscribe([self.retry_topic])
        
        print("=" * 70)
        print(f"  🔄 RETRY CONSUMER — LEVEL {self.retry_level}")
        print("=" * 70)
        print(f"  Topic:       {self.retry_topic}")
        print(f"  Base delay:  {format_duration(self.level_config['base_delay_ms'])}")
        print(f"  Description: {self.level_config['description']}")
        print(f"  Next level:  {'DLQ' if self.retry_level >= RETRY_CONFIG['max_retries'] else f'retry_{self.retry_level + 1}'}")
        print("=" * 70)
        
        start_time = time.time()
        empty_polls = 0
        
        try:
            while empty_polls < max_empty_polls:
                # CRITICAL: Resume any partitions whose delay has expired
                self.delay_enforcer.resume_ready_partitions()
                
                msg = self.consumer.poll(1.0)
                
                if msg is None:
                    empty_polls += 1
                    
                    # Show waiting status if partitions are paused
                    paused = self.delay_enforcer.get_paused_count()
                    if paused > 0 and empty_polls % 5 == 0:
                        print(f"    ⏳ Waiting... ({paused} partition(s) paused for delay)")
                    
                    continue
                
                if msg.error():
                    if msg.error().code() == KafkaError._PARTITION_EOF:
                        empty_polls += 1
                        continue
                    print(f"  Consumer error: {msg.error()}")
                    continue
                
                empty_polls = 0
                self.stats['total'] += 1
                
                key_str = msg.key().decode('utf-8') if msg.key() else '?'
                retry_count = get_retry_count(msg)
                headers = extract_retry_headers(msg)
                
                # ============================================
                # STEP 1: Enforce delay
                # ============================================
                is_ready = self.delay_enforcer.check_and_enforce_delay(msg)
                
                if not is_ready:
                    self.stats['delayed'] += 1
                    # Don't commit — partition is paused
                    # Record will be re-delivered after resume
                    continue
                
                # ============================================
                # STEP 2: Process (record is ready)
                # ============================================
                print(f"\n  [{self.retry_topic}] Processing: key={key_str}, "
                      f"retry={retry_count}/{RETRY_CONFIG['max_retries']}")
                
                try:
                    success, response, original_data = self.process_payment(msg.value())
                    
                    # SUCCESS — record recovered!
                    self.produce_confirmation(original_data, response)
                    self.stats['success'] += 1
                    
                    # Calculate recovery metrics
                    first_error = int(headers.get('X-First-Error-Time', '0'))
                    recovery_time = int(time.time() * 1000) - first_error if first_error else 0
                    
                    print(f"    ✅ RETRY SUCCESS: order={original_data.get('order_id')}, "
                          f"amount=${original_data.get('amount', 0):.2f}")
                    print(f"       Recovered after {retry_count} retries, "
                          f"total wait: {format_duration(recovery_time)}")
                    
                    consumer_commit = self.consumer.commit(message=msg)
                
                except Exception as proc_error:
                    # ============================================
                    # STEP 3: Route to next retry level or DLQ
                    # ============================================
                    print(f"    ⚠️  Retry {retry_count} FAILED: {str(proc_error)[:60]}")
                    
                    route_result, route_success = self.retry_router.route_failed_record(
                        msg, proc_error, source_consumer_type=f'retry_{self.retry_level}'
                    )
                    
                    if route_result.startswith('RETRY'):
                        self.stats['sent_to_next_retry'] += 1
                    elif route_result.startswith('DLQ'):
                        self.stats['sent_to_dlq'] += 1
                    
                    if route_success:
                        self.consumer.commit(message=msg)
        
        except KeyboardInterrupt:
            print("\n  Shutting down retry consumer...")
        
        finally:
            self.consumer.close()
            self.success_producer.flush()
        
        # Summary
        elapsed = time.time() - start_time
        delay_stats = self.delay_enforcer.get_stats()
        
        print(f"\n{'=' * 70}")
        print(f"  RETRY LEVEL {self.retry_level} SUMMARY")
        print(f"{'=' * 70}")
        print(f"  Duration:            {format_duration(int(elapsed * 1000))}")
        print(f"  Total consumed:      {self.stats['total']}")
        print(f"  ✅ Recovered:        {self.stats['success']}")
        print(f"  ⏳ Delays enforced:  {self.stats['delayed']}")
        print(f"  🔄 → Next retry:    {self.stats['sent_to_next_retry']}")
        print(f"  ❌ → DLQ:           {self.stats['sent_to_dlq']}")
        
        if self.stats['total'] > 0:
            recovery_rate = self.stats['success'] / self.stats['total'] * 100
            print(f"  Recovery rate:       {recovery_rate:.1f}%")
        
        if delay_stats.get('delays_enforced', 0) > 0:
            print(f"  Avg delay enforced:  "
                  f"{format_duration(int(delay_stats.get('avg_delay_ms', 0)))}")
        
        print(f"{'=' * 70}")
        
        return self.stats

import threading
import time
from retry_consumer import RetryLevelConsumer
from retry_config import RETRY_CONFIG, format_duration

class RetryOrchestrator:
    """
    Manages all retry level consumers as a single unit
    Each level runs in its own thread
    """
    
    def __init__(self):
        self.consumers = {}
        self.threads = {}
        self.results = {}
        self.running = True
    
    def start_all_levels(self, max_empty_polls=40):
        """Start a retry consumer for each configured level"""
        
        print("=" * 70)
        print("  🎛️  RETRY ORCHESTRATOR — STARTING ALL LEVELS")
        print("=" * 70)
        
        for level in sorted(RETRY_CONFIG['levels'].keys()):
            level_config = RETRY_CONFIG['levels'][level]
            topic = f"{RETRY_CONFIG['source_topic']}{level_config['topic_suffix']}"
            
            print(f"  Level {level}: {topic} "
                  f"(delay: {format_duration(level_config['base_delay_ms'])})")
            
            consumer_instance = RetryLevelConsumer(retry_level=level)
            self.consumers[level] = consumer_instance
            
            def run_level(consumer, lvl, polls):
                try:
                    stats = consumer.run(max_empty_polls=polls)
                    self.results[lvl] = stats
                except Exception as e:
                    print(f"  ❌ Level {lvl} crashed: {e}")
                    self.results[lvl] = {'error': str(e)}
            
            thread = threading.Thread(
                target=run_level,
                args=(consumer_instance, level, max_empty_polls),
                daemon=True,
                name=f"retry-level-{level}"
            )
            self.threads[level] = thread
        
        print(f"\n  Starting {len(self.threads)} retry consumers...")
        print("=" * 70)
        
        start_time = time.time()
        
        # Start all threads
        for level, thread in sorted(self.threads.items()):
            thread.start()
            time.sleep(0.5)  # Stagger starts slightly
        
        # Wait for all to complete
        for level, thread in sorted(self.threads.items()):
            thread.join()
        
        elapsed = time.time() - start_time
        
        # Print unified summary
        self._print_summary(elapsed)
    
    def _print_summary(self, elapsed):
        """Print unified summary across all retry levels"""
        
        print(f"\n{'=' * 70}")
        print(f"  🎛️  RETRY ORCHESTRATOR — UNIFIED SUMMARY")
        print(f"{'=' * 70}")
        print(f"  Total orchestrator duration: {format_duration(int(elapsed * 1000))}")
        
        total_consumed = 0
        total_recovered = 0
        total_to_next = 0
        total_to_dlq = 0
        
        print(f"\n  {'Level':<8} {'Consumed':<10} {'Recovered':<11} {'→ Next':<9} {'→ DLQ':<8} {'Recovery%':<10}")
        print(f"  {'─'*8} {'─'*10} {'─'*11} {'─'*9} {'─'*8} {'─'*10}")
        
        for level in sorted(RETRY_CONFIG['levels'].keys()):
            stats = self.results.get(level, {})
            
            if 'error' in stats:
                print(f"  {level:<8} ERROR: {stats['error']}")
                continue
            
            consumed = stats.get('total', 0)
            recovered = stats.get('success', 0)
            to_next = stats.get('sent_to_next_retry', 0)
            to_dlq = stats.get('sent_to_dlq', 0)
            recovery_pct = f"{recovered/consumed*100:.1f}%" if consumed > 0 else "N/A"
            
            total_consumed += consumed
            total_recovered += recovered
            total_to_next += to_next
            total_to_dlq += to_dlq
            
            print(f"  {level:<8} {consumed:<10} {recovered:<11} {to_next:<9} {to_dlq:<8} {recovery_pct:<10}")
        
        print(f"  {'─'*8} {'─'*10} {'─'*11} {'─'*9} {'─'*8} {'─'*10}")
        
        overall_recovery = f"{total_recovered/total_consumed*100:.1f}%" if total_consumed > 0 else "N/A"
        print(f"  {'TOTAL':<8} {total_consumed:<10} {total_recovered:<11} "
              f"{total_to_next:<9} {total_to_dlq:<8} {overall_recovery:<10}")
        
        print(f"\n  KEY INSIGHT:")
        if total_recovered > 0:
            print(f"  → {total_recovered} records RECOVERED through retry")
            print(f"  → Without retry, these would ALL be in DLQ")
            print(f"  → Retry saved {total_recovered} records from manual intervention")
        
        if total_to_dlq > 0:
            print(f"  → {total_to_dlq} records exhausted all retries → DLQ")
            print(f"  → These need investigation (Project 25 DLQ tools)")
        
        print(f"{'=' * 70}")


if __name__ == '__main__':
    orchestrator = RetryOrchestrator()
    orchestrator.start_all_levels(max_empty_polls=40)

from confluent_kafka import Consumer, TopicPartition
import time
from retry_config import RETRY_CONFIG, format_duration

BOOTSTRAP = RETRY_CONFIG['bootstrap_servers']

def get_topic_depth(topic):
    """Get total unprocessed records in a topic"""
    c = Consumer({
        'bootstrap.servers': BOOTSTRAP,
        'group.id': f'monitor-probe-{int(time.time()*1000)}',
        'auto.offset.reset': 'latest'
    })
    
    try:
        metadata = c.list_topics(topic, timeout=5)
        if topic not in metadata.topics:
            return -1
        
        total = 0
        for pid in metadata.topics[topic].partitions:
            tp = TopicPartition(topic, pid)
            low, high = c.get_watermark_offsets(tp, timeout=5)
            total += (high - low)
        return total
    except Exception:
        return -1
    finally:
        c.close()

def monitor_pipeline(duration_seconds=120, interval_seconds=10):
    """Monitor all retry pipeline topics"""
    
    topics = {
        'INPUT': RETRY_CONFIG['source_topic'],
        'RETRY_1': f"{RETRY_CONFIG['source_topic']}_retry_1",
        'RETRY_2': f"{RETRY_CONFIG['source_topic']}_retry_2",
        'RETRY_3': f"{RETRY_CONFIG['source_topic']}_retry_3",
        'RETRY_4': f"{RETRY_CONFIG['source_topic']}_retry_4",
        'DLQ': RETRY_CONFIG['dlq_topic'],
        'OUTPUT': RETRY_CONFIG['success_topic'],
    }
    
    print("=" * 70)
    print("  📊 RETRY PIPELINE MONITOR")
    print("=" * 70)
    
    # Print topic map
    print(f"\n  Pipeline topology:")
    print(f"  INPUT → [Processor] → OUTPUT (success)")
    print(f"              ↓ fail")
    print(f"           RETRY_1 (1s) → RETRY_2 (10s) → RETRY_3 (60s) → RETRY_4 (5min) → DLQ")
    print(f"")
    
    start = time.time()
    history = []
    
    try:
        while time.time() - start < duration_seconds:
            timestamp = time.strftime('%H:%M:%S')
            snapshot = {'time': timestamp}
            
            print(f"\n  ─── {timestamp} {'─' * 48}")
            print(f"  {'Stage':<12} {'Records':<10} {'Visual'}")
            print(f"  {'─'*12} {'─'*10} {'─'*35}")
            
            for label, topic in topics.items():
                depth = get_topic_depth(topic)
                snapshot[label] = depth
                
                if depth < 0:
                    bar = "  (topic not found)"
                elif depth == 0:
                    bar = "  ·"
                else:
                    bar_len = min(depth, 40)
                    if label == 'OUTPUT':
                        bar = "  " + "█" * bar_len + f" ✅"
                    elif label == 'DLQ':
                        bar = "  " + "█" * bar_len + f" ❌"
                    elif label.startswith('RETRY'):
                        bar = "  " + "▓" * bar_len + f" 🔄"
                    else:
                        bar = "  " + "█" * bar_len
                
                depth_str = str(depth) if depth >= 0 else "N/A"
                print(f"  {label:<12} {depth_str:<10} {bar}")
            
            history.append(snapshot)
            
            # Calculate flow rate if enough history
            if len(history) >= 2:
                prev = history[-2]
                curr = history[-1]
                
                output_delta = (curr.get('OUTPUT', 0) - prev.get('OUTPUT', 0))
                dlq_delta = (curr.get('DLQ', 0) - prev.get('DLQ', 0))
                
                if output_delta > 0 or dlq_delta > 0:
                    print(f"\n  Flow rate (last {interval_seconds}s): "
                          f"✅ +{output_delta} processed, "
                          f"❌ +{dlq_delta} to DLQ")
            
            time.sleep(interval_seconds)
    
    except KeyboardInterrupt:
        print("\n  Monitor stopped.")
    
    # Final summary
    if history:
        first = history[0]
        last = history[-1]
        
        print(f"\n{'=' * 70}")
        print(f"  MONITORING SESSION SUMMARY")
        print(f"{'=' * 70}")
        print(f"  Duration: {len(history) * interval_seconds}s")
        print(f"  Snapshots: {len(history)}")
        
        for label in topics:
            start_val = first.get(label, 0)
            end_val = last.get(label, 0)
            delta = end_val - start_val
            direction = "↑" if delta > 0 else "↓" if delta < 0 else "→"
            print(f"  {label:<12}: {start_val} → {end_val} ({direction}{abs(delta)})")
        
        print(f"{'=' * 70}")


if __name__ == '__main__':
    monitor_pipeline(duration_seconds=120, interval_seconds=10)
