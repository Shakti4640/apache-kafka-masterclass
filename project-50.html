├── Expensive at scale ($2M+ for our volume)

OPTION B: BUILD EVERYTHING CUSTOM
├── Maximum control and customization
├── Requires dedicated platform team (5-8 engineers)
├── Slow to build (12-18 months for full platform)
├── Maintenance burden is ongoing
└── Verdict: Too slow, too expensive in people

OPTION C: COMPOSE FROM OPEN-SOURCE + CUSTOM GLUE (CHOSEN)
├── Kafka + Strimzi (Project 46) for infrastructure
├── Schema Registry (Project 16) for schema governance
├── Custom control plane for topic registry + governance
├── Custom self-service portal (thin layer)
├── Open-source observability (Project 49)
├── Build time: 3-6 months for MVP
└── Verdict: Best balance of control, cost, and time

WE CHOOSE OPTION C:
→ Kafka infrastructure already built (Projects 1-49)
→ Need only the governance + self-service layers
→ Custom control plane is the NEW component
→ Everything else is integration of existing work

✅ WHAT SHOULD DO

1. IMPLEMENT DOMAIN-DRIVEN TOPIC DESIGN
   → Topics organized by business domain, not by team
   → Naming convention: {domain}.{entity}.{event-type}.{version}
   → Example: payments.transaction.completed.v1
   → Prevents proliferation of ad-hoc topic names

2. BUILD A TOPIC REGISTRY (CATALOG)
   → Every topic registered with: owner, schema, SLA, description
   → Discoverable by all teams
   → Changes require approval workflow
   → Orphan detection: topics with no registered owner

3. ENFORCE SCHEMA GOVERNANCE
   → All topics MUST have registered Avro/Protobuf schema
   → Compatibility mode enforced per topic (Project 38)
   → Breaking changes blocked at Schema Registry level
   → Schema review process for cross-team topics

4. IMPLEMENT AUTOMATED ACCESS CONTROL
   → Teams request access to topics via portal
   → Auto-provision KafkaUser + ACLs (Project 30, 46)
   → Producer access: requires schema compatibility check
   → Consumer access: auto-approved with notification to owner

5. BUILD COST ATTRIBUTION MODEL
   → Meter per-topic: storage bytes, produce rate, consume rate
   → Attribute to owning team
   → Monthly cost report per team
   → Alerts when team exceeds budget

6. CREATE SELF-SERVICE ONBOARDING
   → New team signs up → namespace created → credentials issued
   → Topic creation: fill form → governance check → auto-provision
   → Schema registration: upload → compatibility check → register
   → Target: 0 → producing in < 1 hour

7. INTEGRATE ALL OBSERVABILITY
   → Topic catalog links to Grafana dashboards (Project 28)
   → Schema changes linked to trace impact (Project 49)
   → Lineage graph (Project 49) visible in portal
   → SLA compliance per topic visible to owners

8. DESIGN BRIDGE TO NEXT TECHNOLOGIES
   → Flink SQL interface for stream processing
   → Spark Structured Streaming connector config
   → Cloud-native eventing bridge (EventBridge, Event Grid)
   → Migration path documentation for each technology

❌ WHAT SHOULD NOT DO

1. DO NOT MAKE GOVERNANCE SO STRICT THAT TEAMS CAN'T MOVE
   → Governance enables, not blocks
   → Fast-path for development/staging topics
   → Strict governance only for production cross-team topics
   → "Paved road" approach: easy to do the right thing

2. DO NOT BUILD A CUSTOM SCHEMA REGISTRY
   → Confluent Schema Registry already handles this (Project 16)
   → Custom governance layer ON TOP of Schema Registry
   → Don't reinvent compatibility checking

3. DO NOT CENTRALIZE ALL TOPIC MANAGEMENT
   → Platform team becomes bottleneck
   → Self-service with guardrails, not central approval
   → Platform team sets policies, automation enforces them

4. DO NOT IGNORE EXISTING TOPICS
   → 673 existing topics need migration to new naming
   → Gradual migration with aliasing (not big-bang rename)
   → Tag existing topics with owners and domains

5. DO NOT OVER-ENGINEER THE PORTAL
   → Start with CLI + simple web UI
   → API-first design: portal is just a frontend
   → Teams can use API directly in CI/CD pipelines

6. DO NOT FORGET DEVELOPER EXPERIENCE
   → Client libraries with built-in tracing (Project 49)
   → Code generators from schemas
   → Local development environment (docker-compose)
   → Clear documentation with examples for every pattern

7. DO NOT BUILD MONOLITHIC CONTROL PLANE
   → Control plane is a set of APIs, not one big service
   → Topic Registry API, Governance API, Cost API, Lineage API
   → Each can evolve independently

8. DO NOT ASSUME ONE TECHNOLOGY FOREVER
   → Kafka is excellent but not eternal
   → Design abstractions that allow technology migration
   → Bridge APIs that decouple teams from Kafka specifics

GOVERNANCE ENGINE RULES:

Rule 1: NAMING VALIDATION
  → Must match pattern: {domain}.{entity}.{event-type}.{version}
  → Domain must be from approved domain list
  → Version must be v{N} format
  → REJECT if pattern doesn't match

Rule 2: SCHEMA REQUIREMENT
  → All production topics MUST have registered schema
  → Schema format must be Avro or Protobuf (no raw JSON in prod)
  → Compatibility mode must be set (BACKWARD recommended)
  → REJECT topic creation without schema

Rule 3: OWNERSHIP REQUIREMENT
  → Every topic must have owner_team and owner_email
  → Owner team must exist in team registry
  → REJECT topic creation without ownership

Rule 4: CROSS-TEAM ACCESS CONTROL
  → Producing to another team's topic: requires owner approval
  → Consuming from another team's topic: auto-approved + notification
  → Schema changes to cross-team topics: requires consumer notification
  → BLOCK breaking changes if active consumers exist

Rule 5: COST GUARDRAILS
  → Partition count capped per team (default: 100)
  → Retention capped per team (default: 7 days for non-compacted)
  → Storage quota per team (default: 1TB)
  → WARN when approaching 80%, BLOCK at 100%


COST ATTRIBUTION MODEL:

Cost per topic = Storage Cost + Compute Cost + Network Cost

Storage Cost:
  = (avg_message_size × messages_per_day × retention_days × replication_factor)
    × price_per_GB_per_month

Compute Cost:
  = (produce_rate + consume_rate) × price_per_million_messages

Network Cost:
  = cross_region_bytes × price_per_GB_transfer
  + inter_az_bytes × price_per_GB_transfer

Example for payments.transaction.completed.v1:
  Storage: 1KB × 5M/day × 7 days × 3 replicas = 105GB → $2.10/month
  Compute: (5M + 15M) × $0.05/M = $1.00/month
  Network: 10GB cross-region × $0.02/GB = $0.20/month
  Total: $3.30/month

Attributed to: team-payments (owner)
Consumer cost: split proportionally by consume rate

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: GOVERNANCE BECOMES BUREAUCRACY
├── Problem: Too many rules, too many approval steps
├── Impact: Teams bypass the platform ("shadow Kafka")
├── Symptom: Teams spinning up their own Kafka clusters
├── Solution: Fast-path for low-risk operations
└── Rule: If it can be automated, don't require human approval

PITFALL 2: TOPIC CATALOG BECOMES STALE
├── Problem: Teams don't update catalog when things change
├── Impact: Catalog shows wrong owners, wrong schemas
├── Symptom: "The catalog says team-alpha owns this but they disbanded"
├── Solution: Auto-update from runtime data (interceptors + lineage)
└── Periodic ownership verification emails

PITFALL 3: NAMING CONVENTION EXCEPTIONS PROLIFERATE
├── Problem: "Just this once, we need a different name format"
├── Impact: 30% of topics are "exceptions" — convention meaningless
├── Symptom: New engineers can't find topics by convention
├── Solution: Very few exceptions, require VP-level approval
└── Migration path for legacy topics to new naming

PITFALL 4: COST ATTRIBUTION CAUSES TEAM CONFLICTS
├── Problem: "Why am I paying for Team Beta's consumption?"
├── Impact: Teams argue about cost allocation
├── Symptom: Monthly cost review meetings become hostile
├── Solution: Clear model: owner pays storage, consumers pay compute
└── Publish model before implementing, get buy-in

PITFALL 5: SELF-SERVICE CREATES TOPIC SPRAWL
├── Problem: Easy to create topics → too many topics
├── Impact: 2000 topics, most with < 1 msg/day
├── Symptom: Broker metadata overhead, slow topic listings
├── Solution: Lifecycle management — archive unused topics
└── Auto-detect: no messages in 30 days → warning → archive in 60

PITFALL 6: SCHEMA GOVERNANCE BLOCKS FAST ITERATION
├── Problem: Schema compatibility check rejects valid evolution
├── Impact: Teams stuck, can't deploy needed changes
├── Symptom: "Schema Registry won't let me add this field"
├── Solution: Understand compatibility modes (Project 38)
└── Use BACKWARD compatibility (add optional fields freely)

PITFALL 7: BRIDGE TO NEW TECHNOLOGY BECOMES MIGRATION
├── Problem: "Let's move everything to Flink" without strategy
├── Impact: 6-month migration project, half the team learning Flink
├── Symptom: Two systems running in parallel, neither well-maintained
├── Solution: Bridge pattern — Flink READS from Kafka, doesn't replace it
└── Kafka remains the event backbone, processing moves to best tool

class TopicRegistry:
    """
    Central topic registry for the event mesh platform.

    Provides:
    - CRUD operations for topic registrations
    - Search by domain, team, tag
    - Dependency queries (who produces/consumes)
    - Orphan detection (topics without active producers/consumers)
    - Health status aggregation
    """

    def __init__(self):
        self._topics: dict[str, TopicRegistration] = {}
        self._lock = threading.Lock()

    def register(self, registration: TopicRegistration) -> tuple[bool, str]:
        """
        Register a new topic in the catalog.

        Returns:
            (success, message)
        """
        with self._lock:
            if registration.topic_name in self._topics:
                return False, f"Topic '{registration.topic_name}' already registered"

            self._topics[registration.topic_name] = registration
            logger.info(
                f"Topic registered: {registration.topic_name} "
                f"(owner={registration.owner_team})"
            )
            return True, "Topic registered successfully"

    def update(self, topic_name: str, updates: dict) -> tuple[bool, str]:
        """Update an existing topic registration."""
        with self._lock:
            if topic_name not in self._topics:
                return False, f"Topic '{topic_name}' not found"

            reg = self._topics[topic_name]
            for key, value in updates.items():
                if hasattr(reg, key):
                    setattr(reg, key, value)
            reg.last_modified = time.time()

            return True, "Topic updated successfully"

    def get(self, topic_name: str) -> Optional[TopicRegistration]:
        """Get a topic registration by name."""
        with self._lock:
            return self._topics.get(topic_name)

    def search(
        self,
        domain: Optional[str] = None,
        owner_team: Optional[str] = None,
        tag: Optional[str] = None,
        status: Optional[str] = None
    ) -> list[TopicRegistration]:
        """Search topics by criteria."""
        with self._lock:
            results = list(self._topics.values())

            if domain:
                results = [t for t in results if t.domain == domain]
            if owner_team:
                results = [t for t in results if t.owner_team == owner_team]
            if tag:
                results = [t for t in results if tag in t.tags]
            if status:
                results = [t for t in results if t.status == status]

            return results

    def get_all(self) -> list[TopicRegistration]:
        """Get all registered topics."""
        with self._lock:
            return list(self._topics.values())

    def get_consumers(self, topic_name: str) -> list[AccessEntry]:
        """Get all consumers of a topic."""
        with self._lock:
            reg = self._topics.get(topic_name)
            return reg.consumers if reg else []

    def get_producers(self, topic_name: str) -> list[AccessEntry]:
        """Get all producers to a topic."""
        with self._lock:
            reg = self._topics.get(topic_name)
            return reg.producers if reg else []

    def add_access(self, topic_name: str, entry: AccessEntry) -> tuple[bool, str]:
        """Add a producer or consumer access entry."""
        with self._lock:
            reg = self._topics.get(topic_name)
            if not reg:
                return False, f"Topic '{topic_name}' not found"

            target_list = reg.producers if entry.access_type == "produce" else reg.consumers
            target_list.append(entry)
            reg.last_modified = time.time()

            logger.info(
                f"Access granted: {entry.team}/{entry.principal} → "
                f"{entry.access_type} on {topic_name}"
            )
            return True, "Access granted"

    def get_team_topics(self, team: str) -> dict:
        """Get all topics a team owns, produces to, or consumes from."""
        with self._lock:
            owned = [t for t in self._topics.values() if t.owner_team == team]
            producing = []
            consuming = []

            for t in self._topics.values():
                for p in t.producers:
                    entry = p if isinstance(p, dict) else asdict(p)
                    if entry.get("team") == team:
                        producing.append(t.topic_name)
                for c in t.consumers:
                    entry = c if isinstance(c, dict) else asdict(c)
                    if entry.get("team") == team:
                        consuming.append(t.topic_name)

            return {
                "owned": [t.topic_name for t in owned],
                "producing": list(set(producing)),
                "consuming": list(set(consuming)),
            }

    def detect_orphans(self, inactive_days: int = 30) -> list[str]:
        """Find topics with no recent activity."""
        cutoff = time.time() - (inactive_days * 86400)
        orphans = []

        with self._lock:
            for name, reg in self._topics.items():
                if reg.last_message_at < cutoff and reg.status == "active":
                    orphans.append(name)

        return orphans

    def get_domain_summary(self) -> dict:
        """Get summary of topics grouped by domain."""
        with self._lock:
            domains = {}
            for reg in self._topics.values():
                if reg.domain not in domains:
                    domains[reg.domain] = {
                        "topic_count": 0,
                        "teams": set(),
                        "total_throughput": 0.0,
                    }
                domains[reg.domain]["topic_count"] += 1
                domains[reg.domain]["teams"].add(reg.owner_team)
                domains[reg.domain]["total_throughput"] += reg.throughput_msg_s

            # Convert sets to lists for serialization
            for d in domains.values():
                d["teams"] = list(d["teams"])
                d["total_throughput"] = round(d["total_throughput"], 1)

            return domains

    def get_platform_stats(self) -> dict:
        """Get overall platform statistics."""
        with self._lock:
            total = len(self._topics)
            active = sum(1 for t in self._topics.values() if t.status == "active")
            domains = len(set(t.domain for t in self._topics.values()))
            teams = len(set(t.owner_team for t in self._topics.values()))
            total_throughput = sum(t.throughput_msg_s for t in self._topics.values())

            healthy = sum(1 for t in self._topics.values() if t.health_status == "HEALTHY")

            return {
                "total_topics": total,
                "active_topics": active,
                "domains": domains,
                "teams": teams,
                "total_throughput_msg_s": round(total_throughput, 1),
                "healthy_topics": healthy,
                "health_pct": round(healthy / max(1, active) * 100, 1),
            }

import re

@dataclass
class ValidationResult:
    """Result of a governance validation check."""
    valid: bool
    errors: list = field(default_factory=list)
    warnings: list = field(default_factory=list)

    def add_error(self, message: str):
        self.errors.append(message)
        self.valid = False

    def add_warning(self, message: str):
        self.warnings.append(message)

    def to_dict(self) -> dict:
        return {
            "valid": self.valid,
            "errors": self.errors,
            "warnings": self.warnings,
        }


class GovernanceEngine:
    """
    Validates topic registrations against governance rules.

    Rules:
    1. Naming convention compliance
    2. Schema requirement and compatibility
    3. Ownership requirement
    4. Configuration guardrails
    5. Cross-team access policies
    6. Cost quota compliance
    """

    # Topic name pattern: {domain}.{entity}.{event-type}.{version}
    TOPIC_NAME_PATTERN = re.compile(
        r'^[a-z][a-z0-9-]{0,19}\.'          # domain: lowercase, max 20
        r'[a-z][a-z0-9-]{0,29}\.'            # entity: lowercase, max 30
        r'[a-z][a-z0-9-]{0,29}\.'            # event-type: lowercase, max 30
        r'v[1-9][0-9]*$'                      # version: v1, v2, etc.
    )

    # Private topic pattern: _private.{team}.{purpose}
    PRIVATE_TOPIC_PATTERN = re.compile(
        r'^_private\.[a-z][a-z0-9-]{0,29}\.[a-z][a-z0-9-]{0,49}$'
    )

    # Platform topic pattern: _platform.{component}.{purpose}
    PLATFORM_TOPIC_PATTERN = re.compile(
        r'^_platform\.[a-z][a-z0-9-]{0,29}\.[a-z][a-z0-9-]{0,49}$'
    )

    def __init__(
        self,
        approved_domains: Optional[list] = None,
        team_registry: Optional[dict] = None,
        max_partitions_per_team: int = 200,
        max_retention_days: int = 30,
        max_topics_per_team: int = 50,
        require_schema: bool = True
    ):
        self.approved_domains = set(approved_domains or [
            "payments", "orders", "users", "inventory",
            "notifications", "analytics", "compliance",
            "shipping", "catalog", "search", "auth",
        ])
        self.team_registry = team_registry or {}
        self.max_partitions_per_team = max_partitions_per_team
        self.max_retention_days = max_retention_days
        self.max_topics_per_team = max_topics_per_team
        self.require_schema = require_schema

    def validate_topic_creation(
        self,
        registration: TopicRegistration,
        existing_registry: TopicRegistry
    ) -> ValidationResult:
        """
        Validate a topic creation request against all governance rules.

        Returns:
            ValidationResult with errors and warnings
        """
        result = ValidationResult(valid=True)

        # Rule 1: Naming convention
        self._validate_naming(registration, result)

        # Rule 2: Domain approval
        self._validate_domain(registration, result)

        # Rule 3: Ownership
        self._validate_ownership(registration, result)

        # Rule 4: Schema requirement
        self._validate_schema(registration, result)

        # Rule 5: Configuration guardrails
        self._validate_configuration(registration, result)

        # Rule 6: Team quotas
        self._validate_quotas(registration, existing_registry, result)

        # Rule 7: Uniqueness
        self._validate_uniqueness(registration, existing_registry, result)

        if result.valid:
            logger.info(
                f"Governance APPROVED: {registration.topic_name} "
                f"(warnings={len(result.warnings)})"
            )
        else:
            logger.warning(
                f"Governance REJECTED: {registration.topic_name} "
                f"(errors={result.errors})"
            )

        return result

    def _validate_naming(self, reg: TopicRegistration, result: ValidationResult):
        """Validate topic name follows convention."""
        name = reg.topic_name

        # Check standard pattern
        if self.TOPIC_NAME_PATTERN.match(name):
            return

        # Check private pattern
        if self.PRIVATE_TOPIC_PATTERN.match(name):
            result.add_warning("Private topic — not discoverable by other teams")
            return

        # Check platform pattern
        if self.PLATFORM_TOPIC_PATTERN.match(name):
            return

        result.add_error(
            f"Topic name '{name}' does not match naming convention. "
            f"Expected format: {{domain}}.{{entity}}.{{event-type}}.v{{N}} "
            f"Example: payments.transaction.completed.v1"
        )

    def _validate_domain(self, reg: TopicRegistration, result: ValidationResult):
        """Validate domain is approved."""
        if reg.topic_name.startswith("_private.") or reg.topic_name.startswith("_platform."):
            return

        if reg.domain not in self.approved_domains:
            result.add_error(
                f"Domain '{reg.domain}' is not approved. "
                f"Approved domains: {sorted(self.approved_domains)}"
            )

    def _validate_ownership(self, reg: TopicRegistration, result: ValidationResult):
        """Validate topic has proper ownership."""
        if not reg.owner_team:
            result.add_error("owner_team is required")

        if not reg.owner_email:
            result.add_error("owner_email is required")

        if reg.owner_email and "@" not in reg.owner_email:
            result.add_error("owner_email must be a valid email address")

        if not reg.description or len(reg.description) < 10:
            result.add_warning(
                "Description should be at least 10 characters "
                "to help other teams understand this topic"
            )

    def _validate_schema(self, reg: TopicRegistration, result: ValidationResult):
        """Validate schema requirements."""
        if not self.require_schema:
            return

        if reg.topic_name.startswith("_private."):
            result.add_warning("Private topics should still have schemas for documentation")
            return

        if not reg.schema.format or reg.schema.format not in ("avro", "protobuf", "json"):
            result.add_error(
                "Schema format must be 'avro', 'protobuf', or 'json'. "
                "Avro is recommended for production topics."
            )

        if reg.schema.format == "json":
            result.add_warning(
                "JSON schema has limited compatibility checking. "
                "Consider Avro or Protobuf for production topics."
            )

        if reg.schema.compatibility not in ("BACKWARD", "FORWARD", "FULL", "NONE"):
            result.add_error(
                f"Invalid compatibility mode: {reg.schema.compatibility}. "
                f"Use BACKWARD (recommended), FORWARD, FULL, or NONE"
            )

        if reg.schema.compatibility == "NONE":
            result.add_warning(
                "Compatibility mode NONE allows breaking changes. "
                "Use BACKWARD for safer schema evolution."
            )

    def _validate_configuration(self, reg: TopicRegistration, result: ValidationResult):
        """Validate topic configuration within guardrails."""
        config = reg.config

        if config.partitions < 1:
            result.add_error("Partitions must be >= 1")
        elif config.partitions > 128:
            result.add_error("Partitions exceeds maximum (128)")
        elif config.partitions > 24:
            result.add_warning(
                f"High partition count ({config.partitions}). "
                f"Consider if this is necessary."
            )

        if config.replication_factor < 3:
            result.add_warning(
                "Replication factor < 3 risks data loss. "
                "Use 3 for production topics."
            )

        retention_days = config.retention_ms / 86400000
        if retention_days > self.max_retention_days:
            result.add_error(
                f"Retention ({retention_days:.0f} days) exceeds maximum "
                f"({self.max_retention_days} days). Request exception if needed."
            )

        if config.compression_type not in ("none", "gzip", "snappy", "lz4", "zstd"):
            result.add_error(f"Invalid compression type: {config.compression_type}")

    def _validate_quotas(
        self,
        reg: TopicRegistration,
        registry: TopicRegistry,
        result: ValidationResult
    ):
        """Validate team hasn't exceeded quotas."""
        team = reg.owner_team
        team_topics = registry.search(owner_team=team, status="active")

        # Topic count quota
        if len(team_topics) >= self.max_topics_per_team:
            result.add_error(
                f"Team '{team}' has reached topic limit "
                f"({len(team_topics)}/{self.max_topics_per_team})"
            )

        # Partition count quota
        total_partitions = sum(t.config.partitions for t in team_topics)
        new_total = total_partitions + reg.config.partitions
        if new_total > self.max_partitions_per_team:
            result.add_error(
                f"Team '{team}' partition quota exceeded: "
                f"{new_total}/{self.max_partitions_per_team}"
            )

    def _validate_uniqueness(
        self,
        reg: TopicRegistration,
        registry: TopicRegistry,
        result: ValidationResult
    ):
        """Validate topic doesn't already exist."""
        existing = registry.get(reg.topic_name)
        if existing:
            result.add_error(
                f"Topic '{reg.topic_name}' already exists "
                f"(owner: {existing.owner_team})"
            )

    def validate_schema_change(
        self,
        topic_name: str,
        registry: TopicRegistry
    ) -> ValidationResult:
        """
        Validate a schema change for a topic.
        Checks if consumers will be affected.
        """
        result = ValidationResult(valid=True)
        reg = registry.get(topic_name)

        if not reg:
            result.add_error(f"Topic '{topic_name}' not found in registry")
            return result

        # Check for active consumers
        consumers = reg.consumers
        if consumers:
            consumer_teams = set()
            for c in consumers:
                entry = c if isinstance(c, dict) else {"team": c.team}
                consumer_teams.add(entry.get("team", "unknown"))

            if len(consumer_teams) > 0:
                result.add_warning(
                    f"Schema change will affect {len(consumer_teams)} consumer teams: "
                    f"{sorted(consumer_teams)}. "
                    f"Ensure compatibility mode ({reg.schema.compatibility}) allows this change."
                )

        return result

class CostAttributionEngine:
    """
    Calculates per-team costs for Kafka usage.

    Cost allocation rules:
    - Storage: Charged to topic OWNER
    - Produce compute: Charged to topic OWNER
    - Consume compute: Charged to CONSUMER team
    - Network: Charged to topic OWNER (cross-region replication)
    - Platform base cost: Split equally across all active teams
    """

    def __init__(self, cost_model: Optional[CostModel] = None):
        self.model = cost_model or CostModel()

    def calculate_topic_cost(
        self,
        topic_name: str,
        avg_message_size_bytes: int,
        messages_per_day: int,
        retention_days: int,
        replication_factor: int,
        consumer_count: int,
        cross_region_replicated: bool = False
    ) -> TopicCost:
        """Calculate cost for a single topic."""
        cost = TopicCost(topic_name=topic_name, owner_team="")

        # Storage cost
        daily_bytes = avg_message_size_bytes * messages_per_day
        total_bytes = daily_bytes * retention_days * replication_factor
        storage_gb = total_bytes / (1024 ** 3)
        cost.storage_cost = storage_gb * self.model.storage_per_gb_month

        # Compute cost (produce + consume)
        total_messages = messages_per_day * 30  # Monthly
        produce_cost = (total_messages / 1_000_000) * self.model.compute_per_million_msgs
        consume_cost = (total_messages * consumer_count / 1_000_000) * self.model.compute_per_million_msgs
        cost.compute_cost = produce_cost + consume_cost

        # Network cost (cross-region replication)
        if cross_region_replicated:
            monthly_bytes = daily_bytes * 30
            network_gb = monthly_bytes / (1024 ** 3)
            cost.network_cost = network_gb * self.model.network_cross_region_per_gb

        cost.calculate_total()
        return cost

    def calculate_all_costs(self, registry: TopicRegistry) -> dict[str, TeamCost]:
        """
        Calculate costs for all teams based on topic registry.

        Returns:
            Dict mapping team name → TeamCost
        """
        team_costs: dict[str, TeamCost] = {}
        all_topics = registry.get_all()
        active_teams = set()

        for reg in all_topics:
            if reg.status != "active":
                continue

            active_teams.add(reg.owner_team)

            # Estimate message size and rate from registry
            avg_msg_size = 1024       # Default 1KB
            msgs_per_day = int(reg.throughput_msg_s * 86400) if reg.throughput_msg_s > 0 else 10000
            retention_days = reg.config.retention_ms // 86400000
            consumer_count = len(reg.consumers)
            cross_region = any(
                t in reg.tags for t in ["dr-replicated", "cross-region"]
            )

            topic_cost = self.calculate_topic_cost(
                topic_name=reg.topic_name,
                avg_message_size_bytes=avg_msg_size,
                messages_per_day=msgs_per_day,
                retention_days=max(1, retention_days),
                replication_factor=reg.config.replication_factor,
                consumer_count=max(1, consumer_count),
                cross_region_replicated=cross_region,
            )
            topic_cost.owner_team = reg.owner_team

            # Attribute to owner team
            if reg.owner_team not in team_costs:
                team_costs[reg.owner_team] = TeamCost(team=reg.owner_team)
            team_costs[reg.owner_team].owned_topic_costs.append(topic_cost)

            # Attribute consume cost to consumer teams
            for consumer in reg.consumers:
                consumer_team = consumer.team if hasattr(consumer, 'team') else consumer.get("team", "unknown")
                active_teams.add(consumer_team)

                if consumer_team not in team_costs:
                    team_costs[consumer_team] = TeamCost(team=consumer_team)

                # Consumer compute cost (proportional)
                consumer_topic_cost = TopicCost(
                    topic_name=reg.topic_name,
                    owner_team=consumer_team,
                )
                per_consumer_msgs = msgs_per_day * 30 / max(1, consumer_count)
                consumer_topic_cost.compute_cost = (
                    (per_consumer_msgs / 1_000_000) * self.model.compute_per_million_msgs
                )
                consumer_topic_cost.calculate_total()
                team_costs[consumer_team].consumed_topic_costs.append(consumer_topic_cost)

        # Split platform base cost across active teams
        if active_teams:
            per_team_platform = self.model.base_cluster_cost_month / len(active_teams)
            for team in active_teams:
                if team in team_costs:
                    team_costs[team].platform_share = per_team_platform

        # Calculate totals
        for tc in team_costs.values():
            tc.total_cost = (
                sum(t.total_cost for t in tc.owned_topic_costs)
                + sum(t.total_cost for t in tc.consumed_topic_costs)
                + tc.platform_share
            )

        return team_costs

    def generate_report(self, registry: TopicRegistry) -> dict:
        """Generate a complete cost report."""
        team_costs = self.calculate_all_costs(registry)

        total_platform_cost = sum(tc.total_cost for tc in team_costs.values())

        report = {
            "generated_at": time.time(),
            "total_platform_cost": round(total_platform_cost, 2),
            "team_count": len(team_costs),
            "teams": {
                team: tc.to_dict()
                for team, tc in sorted(
                    team_costs.items(),
                    key=lambda x: x[1].total_cost,
                    reverse=True,
                )
            },
            "top_cost_topics": self._get_top_cost_topics(team_costs, limit=10),
        }

        return report

    def _get_top_cost_topics(self, team_costs: dict, limit: int = 10) -> list:
        """Get the most expensive topics across all teams."""
        all_topic_costs = []
        for tc in team_costs.values():
            for topic_cost in tc.owned_topic_costs:
                all_topic_costs.append({
                    "topic": topic_cost.topic_name,
                    "owner": topic_cost.owner_team,
                    "monthly_cost": round(topic_cost.total_cost, 2),
                    "storage": round(topic_cost.storage_cost, 2),
                    "compute": round(topic_cost.compute_cost, 2),
                    "network": round(topic_cost.network_cost, 2),
                })

        all_topic_costs.sort(key=lambda x: x["monthly_cost"], reverse=True)
        return all_topic_costs[:limit]

@dataclass
class ProvisioningResult:

    acls_created: bool = False
    monitoring_configured: bool = False
    errors: list = None
    duration_seconds: float = 0.0

        if self.errors is None:
            self.errors = []

    def to_dict(self) -> dict:
        return {
            "success": self.success,
            "topic_created": self.topic_created,
            "schema_registered": self.schema_registered,
            "acls_created": self.acls_created,
            "monitoring_configured": self.monitoring_configured,
            "errors": self.errors,
            "duration_seconds": round(self.duration_seconds, 2),
        }


class TopicProvisioner:
    """
    Provisions Kafka resources from topic registrations.

    Provisioning flow:
    1. Generate KafkaTopic CRD YAML (Strimzi — Project 46)
    2. Generate KafkaUser CRD YAML with ACLs (Project 46)
    3. Register schema in Schema Registry (Project 16)
    4. Create monitoring alerts (Project 28)
    5. Update topic registry with provisioning status

    In production: applies CRDs via Kubernetes API.
    Here: generates YAML for GitOps deployment.
    """

    def __init__(
        self,
        cluster_name: str = "production-cluster",
        namespace: str = "kafka-platform",
        schema_registry_url: str = "http://schema-registry:8081"
    ):
        self.cluster_name = cluster_name
        self.namespace = namespace
        self.schema_registry_url = schema_registry_url

    def provision(self, registration: TopicRegistration) -> ProvisioningResult:
        """
        Provision all resources for a topic registration.
        Returns result with generated YAML and status.
        """
        start_time = time.time()
        result = ProvisioningResult(success=True)

        try:
            # Step 1: Generate and apply KafkaTopic CRD
            topic_yaml = self._generate_topic_crd(registration)
            result.topic_created = self._apply_resource(topic_yaml, "KafkaTopic")

            # Step 2: Generate and apply KafkaUser CRDs for all producers/consumers
            for producer in registration.producers:
                user_yaml = self._generate_user_crd(
                    registration, producer, "produce"
                )
                self._apply_resource(user_yaml, "KafkaUser")

            for consumer in registration.consumers:
                user_yaml = self._generate_user_crd(
                    registration, consumer, "consume"
                )
                self._apply_resource(user_yaml, "KafkaUser")

            result.acls_created = True

            # Step 3: Register schema
            if registration.schema.format in ("avro", "protobuf"):
                result.schema_registered = self._register_schema(registration)

            # Step 4: Configure monitoring
            result.monitoring_configured = self._configure_monitoring(registration)

            logger.info(
                f"Provisioning COMPLETE for {registration.topic_name} "
                f"in {time.time() - start_time:.1f}s"
            )

        except Exception as e:
            result.success = False
            result.errors.append(str(e))
            logger.error(f"Provisioning FAILED for {registration.topic_name}: {e}")

        result.duration_seconds = time.time() - start_time
        return result

    def _generate_topic_crd(self, reg: TopicRegistration) -> dict:
        """Generate Strimzi KafkaTopic CRD (Project 46 pattern)."""
        crd = {
            "apiVersion": "kafka.strimzi.io/v1beta2",
            "kind": "KafkaTopic",
            "metadata": {
                "name": reg.topic_name.replace(".", "-"),
                "namespace": self.namespace,
                "labels": {
                    "strimzi.io/cluster": self.cluster_name,
                    "platform.domain": reg.domain,
                    "platform.owner": reg.owner_team,
                    "platform.classification": reg.classification,
                },
                "annotations": {
                    "platform.description": reg.description[:256],
                    "platform.owner-email": reg.owner_email,
                    "platform.cost-center": reg.cost_center,
                },
            },
            "spec": {
                "topicName": reg.topic_name,
                "partitions": reg.config.partitions,
                "replicas": reg.config.replication_factor,
                "config": {
                    "retention.ms": str(reg.config.retention_ms),
                    "cleanup.policy": reg.config.cleanup_policy,
                    "compression.type": reg.config.compression_type,
                    "min.insync.replicas": str(reg.config.min_insync_replicas),
                    "max.message.bytes": str(reg.config.max_message_bytes),
                },
            },
        }

        logger.info(f"Generated KafkaTopic CRD: {reg.topic_name}")
        return crd

    def _generate_user_crd(
        self,
        reg: TopicRegistration,
        access: AccessEntry,
        access_type: str
    ) -> dict:
        """Generate Strimzi KafkaUser CRD with ACLs (Project 46 pattern)."""
        principal = access.principal if hasattr(access, 'principal') else access.get("principal", "unknown")
        team = access.team if hasattr(access, 'team') else access.get("team", "unknown")
        group = None
        if hasattr(access, 'consumer_group'):
            group = access.consumer_group
        elif isinstance(access, dict):
            group = access.get("consumer_group")

        acls = []

        if access_type == "produce":
            acls.append({
                "resource": {
                    "type": "topic",
                    "name": reg.topic_name,
                    "patternType": "literal",
                },
                "operations": ["Write", "Describe"],
            })
        elif access_type == "consume":
            acls.append({
                "resource": {
                    "type": "topic",
                    "name": reg.topic_name,
                    "patternType": "literal",
                },
                "operations": ["Read", "Describe"],
            })
            if group:
                acls.append({
                    "resource": {
                        "type": "group",
                        "name": group,
                        "patternType": "literal",
                    },
                    "operations": ["Read"],
                })

        crd = {
            "apiVersion": "kafka.strimzi.io/v1beta2",
            "kind": "KafkaUser",
            "metadata": {
                "name": principal,
                "namespace": self.namespace,
                "labels": {
                    "strimzi.io/cluster": self.cluster_name,
                    "platform.team": team,
                },
            },
            "spec": {
                "authentication": {"type": "scram-sha-512"},
                "authorization": {
                    "type": "simple",
                    "acls": acls,
                },
            },
        }

        logger.info(f"Generated KafkaUser CRD: {principal} ({access_type})")
        return crd

    def _apply_resource(self, resource: dict, kind: str) -> bool:
        """
        Apply a Kubernetes resource.

        try:

        except Exception as e:

    def _configure_monitoring(self, reg: TopicRegistration) -> bool:

    def create_topic(

    # ══════════════════════════════════════════════════════════
    # ACCESS MANAGEMENT
    # ══════════════════════════════════════════════════════════

    def request_access(
        self,
        topic_name: str,
        team: str,
        principal: str,
        access_type: str = "consume",
        consumer_group: Optional[str] = None,
        requested_by: str = "unknown"
    ) -> dict:
        """
        Request access to a topic.

        For consumers: auto-approved with owner notification.
        For producers: requires governance check.
        """
        reg = self.registry.get(topic_name)
        if not reg:
            return {
                "success": False,
                "message": f"Topic '{topic_name}' not found in catalog",
            }

        # Build access entry
        entry = AccessEntry(
            team=team,
            principal=principal,
            consumer_group=consumer_group,
            access_type=access_type,
            granted_by=requested_by,
        )

        # Consumer access: auto-approved
        if access_type == "consume":
            success, msg = self.registry.add_access(topic_name, entry)

            self._audit("access_granted", requested_by, {
                "topic": topic_name,
                "team": team,
                "principal": principal,
                "type": "consume",
                "auto_approved": True,
            })

            logger.info(
                f"Consumer access auto-approved: {team}/{principal} → {topic_name}"
            )

            return {
                "success": success,
                "message": msg,
                "auto_approved": True,
                "notification": f"Owner ({reg.owner_email}) notified of new consumer",
            }

        # Producer access: governance check
        elif access_type == "produce":
            # Check if team is producing to another team's topic
            if team != reg.owner_team:
                logger.warning(
                    f"Cross-team produce request: {team} → {topic_name} "
                    f"(owned by {reg.owner_team})"
                )

                # Schema compatibility is mandatory for cross-team produce
                schema_validation = self.governance.validate_schema_change(
                    topic_name, self.registry
                )

                self._audit("access_requested", requested_by, {
                    "topic": topic_name,
                    "team": team,
                    "type": "produce",
                    "cross_team": True,
                    "requires_approval": True,
                })

                return {
                    "success": True,
                    "message": "Producer access request submitted for approval",
                    "auto_approved": False,

import logging

class FlinkBridge:
    """
    Generates Apache Flink configurations from Kafka topic registrations.

    Bridges Kafka → Flink by generating:
    1. Flink SQL CREATE TABLE statements
    2. Flink job configurations
    3. Connector properties

    WHY FLINK:
    → Kafka Streams (Projects 19-22) handles simple transformations
    → Flink handles complex event processing:
      - Multi-stream temporal joins
      - Complex windowing with late data handling
      - ML model inference in stream
      - Exactly-once across multiple sinks
      - SQL interface for analysts (not just developers)
    """

    def __init__(self, kafka_bootstrap: str = "localhost:9092"):
        self.kafka_bootstrap = kafka_bootstrap

    def generate_flink_sql_ddl(self, registration: TopicRegistration) -> str:
        """
        Generate Flink SQL CREATE TABLE for a Kafka topic.

        This allows Flink SQL queries like:
          SELECT customer_id, SUM(amount)
          FROM payments_transaction_completed
          WHERE event_time > CURRENT_TIMESTAMP - INTERVAL '1' HOUR
          GROUP BY customer_id
        """
        table_name = registration.topic_name.replace(".", "_").replace("-", "_")

        # Map schema format to Flink format
        format_map = {
            "avro": "avro-confluent",
            "json": "json",
            "protobuf": "protobuf",
        }
        flink_format = format_map.get(registration.schema.format, "json")

        ddl = f"""
-- Flink SQL DDL for topic: {registration.topic_name}
-- Owner: {registration.owner_team}
-- Description: {registration.description}
-- Generated by Event Mesh Platform

CREATE TABLE {table_name} (
    -- Schema fields would be derived from Avro/Protobuf schema
    -- Placeholder structure based on topic domain
    `event_id` STRING,
    `event_time` TIMESTAMP(3),
    `correlation_id` STRING,
    -- Domain-specific fields go here
    -- (Auto-generated from Schema Registry in production)

    -- Kafka metadata columns
    `kafka_partition` INT METADATA FROM 'partition' VIRTUAL,
    `kafka_offset` BIGINT METADATA FROM 'offset' VIRTUAL,
    `kafka_timestamp` TIMESTAMP(3) METADATA FROM 'timestamp' VIRTUAL,

    -- Watermark for event-time processing
    WATERMARK FOR `event_time` AS `event_time` - INTERVAL '5' SECOND
) WITH (
    'connector' = 'kafka',
    'topic' = '{registration.topic_name}',
    'properties.bootstrap.servers' = '{self.kafka_bootstrap}',
    'properties.group.id' = 'flink-{table_name}',
    'scan.startup.mode' = 'latest-offset',
    'format' = '{flink_format}',
    'avro-confluent.url' = 'http://schema-registry:8081'
);

-- Example query: real-time aggregation
-- SELECT
--     TUMBLE_START(event_time, INTERVAL '1' MINUTE) AS window_start,
--     COUNT(*) AS event_count
-- FROM {table_name}
-- GROUP BY TUMBLE(event_time, INTERVAL '1' MINUTE);
"""
        logger.info(f"Generated Flink SQL DDL for {registration.topic_name}")
        return ddl.strip()

    def generate_flink_job_config(self, registration: TopicRegistration) -> dict:
        """Generate Flink job configuration properties."""
        return {
            "job.name": f"flink-{registration.topic_name.replace('.', '-')}",
            "execution.checkpointing.interval": "60000",
            "execution.checkpointing.mode": "EXACTLY_ONCE",
            "state.backend": "rocksdb",
            "state.checkpoints.dir": f"s3://flink-checkpoints/{registration.topic_name}",
            "kafka.bootstrap.servers": self.kafka_bootstrap,
            "kafka.group.id": f"flink-{registration.topic_name.replace('.', '-')}",
        }

    @staticmethod
    def get_migration_guide() -> str:
        """Guide for migrating from Kafka Streams to Flink."""
        return """
KAFKA STREAMS → FLINK MIGRATION GUIDE:

WHEN TO STAY ON KAFKA STREAMS (Projects 19-22):
→ Simple filter/map/aggregate operations
→ Single-topic or simple join processing
→ Library embedded in your microservice
→ No separate cluster needed

WHEN TO MOVE TO FLINK:
→ Complex multi-stream joins with temporal conditions
→ CEP (Complex Event Processing) patterns
→ SQL interface needed for data analysts
→ Need to sink to multiple destinations atomically
→ Processing requires ML model inference
→ Late data handling beyond Kafka Streams capabilities

MIGRATION STEPS:
1. Define Flink SQL tables for your Kafka topics (use this bridge)
2. Rewrite Kafka Streams topology as Flink SQL or DataStream API
3. Deploy Flink job alongside existing Kafka Streams app
4. Validate output parity (shadow mode)
5. Switch consumers from Kafka Streams output to Flink output
6. Decommission Kafka Streams app

COEXISTENCE PATTERN:
→ Kafka remains the event backbone
→ Kafka Streams for simple per-service processing
→ Flink for cross-service complex analytics
→ Both read from same Kafka topics
→ Both write results back to Kafka topics
"""

class SparkBridge:
    """
    Generates Spark Structured Streaming configurations.

    WHY SPARK:
    → Unified batch + streaming API
    → Rich ML library (MLlib)
    → Best for analytics workloads that combine
      historical (batch) and real-time (streaming) data
    → Kafka Streams can't do batch; Spark does both
    """

    def __init__(self, kafka_bootstrap: str = "localhost:9092"):
        self.kafka_bootstrap = kafka_bootstrap

    def generate_spark_reader_config(self, registration: TopicRegistration) -> str:
        """Generate PySpark code for reading from a Kafka topic."""
        code = f'''
# Spark Structured Streaming reader for: {registration.topic_name}
# Owner: {registration.owner_team}
# Generated by Event Mesh Platform

from pyspark.sql import SparkSession
from pyspark.sql.functions import from_json, col
from pyspark.sql.types import StructType, StringType, DoubleType, TimestampType

spark = SparkSession.builder \\
    .appName("spark-{registration.topic_name.replace('.', '-')}") \\
    .getOrCreate()

# Read from Kafka topic
kafka_df = spark.readStream \\
    .format("kafka") \\
    .option("kafka.bootstrap.servers", "{self.kafka_bootstrap}") \\
    .option("subscribe", "{registration.topic_name}") \\
    .option("startingOffsets", "latest") \\
    .option("kafka.group.id", "spark-{registration.topic_name.replace('.', '-')}") \\
    .load()

# Parse value (adjust schema based on actual topic schema)
# For Avro: use spark-avro package with Schema Registry
# For JSON:
schema = StructType() \\
    .add("event_id", StringType()) \\
    .add("timestamp", TimestampType()) \\
    .add("correlation_id", StringType())
    # Add domain-specific fields from Schema Registry

parsed_df = kafka_df \\
    .select(
        col("key").cast("string").alias("message_key"),
        from_json(col("value").cast("string"), schema).alias("data"),
        col("topic"),
        col("partition"),
        col("offset"),
        col("timestamp").alias("kafka_timestamp"),
    ) \\
    .select("message_key", "data.*", "topic", "partition", "offset", "kafka_timestamp")

# Process and write results
# Example: write aggregation back to Kafka
# query = parsed_df \\
#     .groupBy(window("timestamp", "1 minute")) \\
#     .count() \\
#     .writeStream \\
#     .format("kafka") \\
#     .option("kafka.bootstrap.servers", "{self.kafka_bootstrap}") \\
#     .option("topic", "analytics.{registration.entity}.aggregated.v1") \\
#     .option("checkpointLocation", "/tmp/spark-checkpoints/{registration.topic_name}") \\
#     .start()
'''
        logger.info(f"Generated Spark reader config for {registration.topic_name}")
        return code.strip()

    @staticmethod
    def get_migration_guide() -> str:
        return """
KAFKA → SPARK STRUCTURED STREAMING BRIDGE GUIDE:

WHEN TO USE SPARK:
→ Combining real-time Kafka data with historical batch data
→ ML model training on streaming data (MLlib)
→ Complex analytics requiring DataFrame/SQL operations
→ ETL pipelines landing Kafka data into data lake
→ Team already has Spark expertise

WHEN NOT TO USE SPARK:
→ Low-latency processing (< 100ms) — use Kafka Streams
→ Simple filtering/routing — use Kafka Streams
→ Per-record processing — Spark's micro-batch adds latency

COEXISTENCE:
→ Kafka = event backbone (real-time transport)
→ Spark = analytics engine (batch + streaming processing)
→ Spark reads from Kafka, writes to Kafka or data lake
→ Not a replacement for Kafka — a complement
"""
