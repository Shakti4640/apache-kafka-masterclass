PROJECT 24: THE DATA MASKING PIPELINE — SINGLE MESSAGE TRANSFORMS (SMTs)

🏗️ HYPOTHETICAL PROBLEM

SCENARIO: Healthcare Data Compliance Platform

→ A hospital system uses the JDBC Source Connector (Project 23)
  to stream patient records from a PostgreSQL database into Kafka

→ The raw database rows contain:
  - patient_id
  - full_name
  - ssn (Social Security Number)
  - diagnosis
  - treatment_notes
  - created_at (timestamp)
  - internal_db_sequence_id

→ PROBLEM 1: SSN must NEVER appear in Kafka topics in plaintext
   (HIPAA compliance — violation = $50,000+ fine per record)

→ PROBLEM 2: The internal_db_sequence_id column is meaningless
   downstream — it wastes storage and confuses consumers

→ PROBLEM 3: Downstream analytics needs a field called "source_system"
   with value "hospital_east" — but this column does NOT exist in the DB

→ PROBLEM 4: Records must land in date-partitioned topics like
   "patient_records_2024_01_15" instead of a single flat topic
   (for retention management and cost optimization)

→ PROBLEM 5: All of this must happen WITHOUT writing custom code
   — the team has no Java developers, only infrastructure engineers

→ CONSTRAINT: You cannot modify the source database schema
→ CONSTRAINT: You cannot add processing applications between
   Connect and downstream consumers
→ CONSTRAINT: Must happen at the connector level, inline

🔍 HOW TO APPROACH IT

THINKING THROUGH THE PROBLEM:

1. From Project 15 → We know Kafka Connect moves data via connectors
2. From Project 23 → We know JDBC Source pulls rows into Kafka topics
3. The gap → Data arrives AS-IS from the database
   → No transformation layer exists between source and topic

WHAT WE NEED:
→ A lightweight transformation layer INSIDE the connector pipeline
→ That can mask fields (SSN)
→ That can remove fields (internal_db_sequence_id)
→ That can add fields (source_system)
→ That can reroute to dynamic topic names (date-based)
→ All without writing a single line of Java/Python application code

THIS IS EXACTLY WHAT SMTs (Single Message Transforms) SOLVE:
→ Built into Kafka Connect framework
→ Configured purely via JSON/properties
→ Applied per-record as data flows through the connector
→ Chainable — multiple transforms in sequence
→ Works on both Source and Sink connectors

✅ WHAT COULD BE DONE

OPTION A: Write a Kafka Streams application (Project 19-21)
  → Consumes raw topic, transforms, produces to clean topic
  → Works but adds infrastructure, latency, failure points
  → Overkill for simple field-level transforms

OPTION B: Use a consumer application with custom logic
  → Full control but requires coding, deployment, monitoring
  → Defeats purpose of Kafka Connect's no-code philosophy

OPTION C: Transform at database level (views, triggers)
  → Violates constraint of not modifying source DB
  → Creates coupling between DB schema and streaming needs

OPTION D: Use SMTs within Kafka Connect ← CHOSEN
  → Zero custom code
  → Configuration-only
  → Runs inside the connector process
  → Maintained by Apache Kafka community
  → Battle-tested in production

✅ WHAT SHOULD DO

1. CHAIN SMTs IN CORRECT ORDER
   → Order matters — each SMT sees the output of the previous one
   → Mask BEFORE removing fields (don't remove what you need to mask)
   → Add fields BEFORE routing (if routing depends on added fields)

2. USE DESCRIPTIVE TRANSFORM ALIASES
   → "transforms=maskSSN,dropInternal,addSource,routeByDate"
   → NOT "transforms=t1,t2,t3,t4"

3. SPECIFY KEY VS VALUE TARGETING
   → Most SMTs can operate on key OR value
   → Always be explicit: use .value. or .key. in config

4. TEST SMT CHAIN ON SMALL DATASET FIRST
   → Run connector with a test topic
   → Verify each transform independently before chaining

5. VERSION CONTROL YOUR CONNECTOR CONFIGS
   → SMT configurations ARE your transformation logic
   → Treat them as code — Git-tracked, reviewed, auditable

6. USE PREDICATES (Kafka 2.6+) FOR CONDITIONAL APPLICATION
   → Apply SMT only when a condition matches
   → Example: mask SSN only if record has ssn field

7. DOCUMENT THE TRANSFORMATION CHAIN
   → For compliance audits, you must prove SSN masking exists
   → Config file = audit evidence

❌ WHAT SHOULD NOT DO

1. DO NOT USE SMTs FOR COMPLEX BUSINESS LOGIC
   → SMTs are for field-level operations: add, remove, mask, rename, route
   → NOT for: joins, aggregations, lookups, enrichment from external APIs
   → For those → use Kafka Streams (Projects 19-21) or ksqlDB

2. DO NOT CHAIN MORE THAN 5-7 SMTs
   → Each SMT adds latency per record
   → Beyond 7, you're building a processing pipeline in config
   → That belongs in a proper stream processor

3. DO NOT ASSUME SMTs ARE ATOMIC
   → If connector crashes mid-chain, partial transforms may occur
   → SMTs don't have transaction semantics within the chain

4. DO NOT USE MaskField AND THINK DATA IS "ENCRYPTED"
   → MaskField replaces with a fixed mask value
   → It is NOT encryption, NOT hashing, NOT tokenization
   → For true encryption → custom SMT or external service

5. DO NOT FORGET SCHEMA IMPACT
   → InsertField adds a field → schema changes downstream
   → ReplaceField removes a field → consumers expecting it will break
   → Always coordinate with downstream teams

6. DO NOT MIX UP "transforms" AND "transforms.NAME.type"
   → The alias list and the type declarations must match exactly
   → Typo in alias = silently ignored transform

🚫 WHEN NOT TO USE

1. WHEN TRANSFORMATION REQUIRES EXTERNAL STATE
   → SMTs are stateless — they see ONE record at a time
   → Cannot look up data from a database, cache, or API
   → Use Kafka Streams KTable join (Project 32) instead

2. WHEN YOU NEED CONDITIONAL ROUTING BASED ON COMPLEX RULES
   → SMTs have basic predicates (field exists, field equals)
   → Cannot do: "if field A > 100 AND field B contains 'urgent'"
   → Use Kafka Streams branch() (Project 20) instead

3. WHEN MASKING MUST BE REVERSIBLE
   → MaskField is destructive — original value is lost
   → For reversible masking → use encryption (custom SMT)

4. WHEN MULTIPLE OUTPUT TOPICS NEEDED FROM ONE RECORD
   → SMTs produce ONE output record per input record
   → Cannot fan-out one record to multiple topics
   → Use Kafka Streams flatMap + to() for that

5. WHEN TRANSFORM LOGIC CHANGES FREQUENTLY
   → Every SMT config change requires connector restart
   → High-churn logic belongs in a stream processing app
   → Where you can do rolling deployments

SMT JAVA INTERFACE (what each SMT implements):
═══════════════════════════════════════════════════════════

  public interface Transformation<R extends ConnectRecord<R>> {

      // Called ONCE when connector starts
      void configure(Map<String, ?> configs);

      // Called for EVERY SINGLE RECORD passing through
      R apply(R record);

      // Called when connector shuts down
      void close();

      // Describes available configuration options
      ConfigDef config();
  }

  KEY INSIGHT:
  → apply(R record) receives a ConnectRecord
  → ConnectRecord contains:
      - topic (String)
      - kafkaPartition (Integer)
      - keySchema (Schema)
      - key (Object)
      - valueSchema (Schema)
      - value (Object)
      - timestamp (Long)
      - headers (Headers)
  → SMT can modify ANY of these fields
  → Returns a NEW ConnectRecord (immutable pattern)


═══════════════════════════════════════════════════════════
MASKFIELD INTERNAL MECHANICS:
═══════════════════════════════════════════════════════════

  Input: {"ssn": "[REDACTED:SSN]", "name": "Jane"}
  Config: fields=ssn, replacement=""

  STEP 1: Extract valueSchema → find field "ssn"
  STEP 2: Determine field type → STRING
  STEP 3: Apply type-specific mask:
           STRING  → empty string ""
           INT     → 0
           FLOAT   → 0.0
           BOOLEAN → false
  STEP 4: Build new Struct with masked value
  Output: {"ssn": "", "name": "Jane"}

  NOTE: MaskField does NOT use "XXX-XX-XXXX"
        → For STRING: replaces with empty string by default
        → For custom mask patterns: need custom SMT


═══════════════════════════════════════════════════════════
TIMESTAMPROUTER INTERNAL MECHANICS:
═══════════════════════════════════════════════════════════

  Input Topic: "patient_records"
  Record Timestamp: 1705276800000 (Jan 15, 2024)
  Config: topic.format=${topic}_${timestamp}
          timestamp.format=yyyy_MM_dd

  STEP 1: Extract record.timestamp() → 1705276800000
  STEP 2: Format with SimpleDateFormat → "2024_01_15"
  STEP 3: Apply topic.format template:
           ${topic} → "patient_records"
           ${timestamp} → "2024_01_15"
  STEP 4: Result → "patient_records_2024_01_15"
  STEP 5: Return new ConnectRecord with modified topic name

  CRITICAL: This creates NEW TOPICS dynamically
  → Kafka auto.create.topics.enable must be true
  → OR topics must be pre-created


═══════════════════════════════════════════════════════════
PREDICATE ENGINE (Kafka 2.6+):
═══════════════════════════════════════════════════════════

  Purpose: Apply SMT only when condition matches

  Config:
    transforms=maskSSN
    transforms.maskSSN.type=MaskField
    transforms.maskSSN.predicate=hasSSN
    transforms.maskSSN.negate=false

    predicates=hasSSN
    predicates.hasSSN.type=HasHeaderKey
    predicates.hasSSN.name=contains-ssn

  EXECUTION:
  For each record:
    IF predicate.test(record) == true:
      → Apply SMT
    ELSE:
      → Pass record through unchanged

  Available Predicates:
    → TopicNameMatches (regex on topic name)
    → HasHeaderKey (check if header exists)
    → RecordIsTombstone (value is null)

🚨 COMMON PITFALLS AND FAILURE MODES

PITFALL 1: TRANSFORM ALIAS MISMATCH
═══════════════════════════════════
  WRONG:
    transforms=maskSSN,dropField
    transforms.maskSsn.type=...     ← Case mismatch: maskSsn ≠ maskSSN
  
  RESULT: Transform silently ignored — no error, no masking
  FIX: Exact case match between alias list and config keys


PITFALL 2: ORDERING DESTROYS DATA YOU NEED
═══════════════════════════════════════════
  WRONG ORDER:
    transforms=dropSSN,maskSSN
    → dropSSN removes "ssn" field first
    → maskSSN tries to mask "ssn" → field doesn't exist → ERROR

  CORRECT ORDER:
    transforms=maskSSN,dropSSN
    → Mask first (if you want both), then drop


PITFALL 3: SCHEMA-AWARE VS SCHEMALESS MISMATCH
═══════════════════════════════════════════════
  → JDBC Source with Avro → records have SCHEMA
  → MaskField on schemaless JSON → different behavior
  → Some SMTs require schema, some work without
  → Error: "Only Struct objects supported for [MaskField]"
  FIX: Ensure converter matches SMT expectations
       Use JsonConverter with schemas.enable=true if needed


PITFALL 4: TIMESTAMPROUTER TOPIC EXPLOSION
══════════════════════════════════════════
  → Format: yyyy_MM_dd_HH → creates 24 topics PER DAY
  → After 1 year → 8,760 topics from ONE connector
  → Broker metadata overhead becomes massive
  FIX: Use coarse granularity (yyyy_MM or yyyy_MM_dd)
       Implement topic cleanup/deletion automation


PITFALL 5: INSERTFIELD WITH WRONG SCOPE
════════════════════════════════════════
  → InsertField can target KEY or VALUE
  → "transforms.addSource.type=InsertField$Value"  ← VALUE
  → "transforms.addSource.type=InsertField$Key"    ← KEY
  → Using wrong suffix → field appears in wrong place
  → Consumers reading value don't see it


PITFALL 6: REPLACEFIELD INCLUDE VS EXCLUDE CONFUSION
═════════════════════════════════════════════════════
  include = WHITELIST: keep ONLY these fields, drop everything else
  exclude = BLACKLIST: keep everything EXCEPT these fields

  WRONG:
    ReplaceField$Value with include=patient_id,name
    → REMOVES diagnosis, treatment_notes, ssn, everything else!
  
  USUALLY WANT:
    ReplaceField$Value with exclude=internal_db_sequence_id
    → Removes ONLY the specified field


PITFALL 7: MASKFIELD ON NON-STRING FIELDS
══════════════════════════════════════════
  → MaskField replaces INT with 0, BOOLEAN with false
  → If downstream expects valid ranges (e.g., age > 0)
  → Masked value 0 may pass validation but be semantically wrong
  → Document what "masked" means for each data type

📈 SCALABILITY CONSIDERATIONS

THROUGHPUT IMPACT:
══════════════════
  → Each SMT adds ~0.01-0.1ms per record (field-level ops)
  → Chain of 5 SMTs → ~0.05-0.5ms per record
  → At 100K records/sec → 5-50ms aggregate overhead
  → NEGLIGIBLE for most use cases

  → BUT: Complex regex in RegexRouter → higher CPU
  → BUT: Flatten on deeply nested structs → higher CPU
  → BENCHMARK: Always test with production-volume data

MEMORY IMPACT:
══════════════
  → SMTs create NEW ConnectRecord objects per transform
  → Chain of 5 → 5 intermediate objects per record
  → Garbage collection pressure increases
  → For high-throughput: monitor JVM GC pauses

CONNECTOR TASK SCALING:
═══════════════════════
  → SMTs run inside each connector task
  → More tasks (tasks.max) → more parallel SMT execution
  → Each task has its OWN SMT instances (not shared)
  → Scales linearly with task count

TOPIC CREATION SCALING (TimestampRouter):
═════════════════════════════════════════
  → Dynamic topics created on-the-fly
  → Broker default: auto.create.topics.enable=true
  → Each new topic → metadata update across all brokers
  → Hundreds of dynamic topics → controller bottleneck
  → MITIGATION: Pre-create topics, use coarse time granularity

SCHEMA REGISTRY IMPACT:
═══════════════════════
  → InsertField/ReplaceField → schema changes
  → New schema version registered per unique structure
  → If field values change schema → version explosion
  → Schema Registry caches help, but monitor subject count

# ============================================================
# FILE: verify_smt_pipeline.py
# Consumes from the SMT-transformed topic and validates
# that all transforms were applied correctly
# ============================================================

from confluent_kafka import Consumer, KafkaError
import json
import sys

def create_consumer():
    """Consumer setup — reuses patterns from Project 12"""
    return Consumer({
        'bootstrap.servers': 'localhost:9092',
        'group.id': 'smt-verification-group',
        'auto.offset.reset': 'earliest',
        'enable.auto.commit': True
    })

def find_smt_topics(consumer):
    """Find all date-routed topics from our connector"""
    metadata = consumer.list_topics(timeout=10)
    smt_topics = [
        topic for topic in metadata.topics 
        if topic.startswith('compliant_patient_records_')
    ]
    return sorted(smt_topics)

def validate_record(record_value, record_number):
    """
    Validates that ALL SMT transformations were applied correctly
    Returns list of validation results
    """
    results = []
    
    # --- Handle schema-wrapped records ---
    if 'payload' in record_value:
        payload = record_value['payload']
    else:
        payload = record_value
    
    # ====================================
    # VALIDATION 1: SSN is masked
    # ====================================
    ssn_value = payload.get('ssn', 'FIELD_MISSING')
    if ssn_value == '' or ssn_value == 'REDACTED' or ssn_value == 0:
        results.append(('PASS', 'MaskField:SSN', f'SSN masked → "{ssn_value}"'))
    elif ssn_value == 'FIELD_MISSING':
        results.append(('WARN', 'MaskField:SSN', 'SSN field not present'))
    else:
        results.append(('FAIL', 'MaskField:SSN', f'SSN NOT masked → "{ssn_value}"'))
    
    # ====================================
    # VALIDATION 2: date_of_birth is masked
    # ====================================
    dob_value = payload.get('date_of_birth', 'FIELD_MISSING')
    if dob_value == '' or dob_value == 0 or dob_value is None:
        results.append(('PASS', 'MaskField:DOB', f'DOB masked → "{dob_value}"'))
    elif dob_value == 'FIELD_MISSING':
        results.append(('WARN', 'MaskField:DOB', 'DOB field not present'))
    else:
        results.append(('FAIL', 'MaskField:DOB', f'DOB NOT masked → "{dob_value}"'))
    
    # ====================================
    # VALIDATION 3: Internal fields removed
    # ====================================
    if 'internal_db_seq_id' not in payload:
        results.append(('PASS', 'ReplaceField:Remove', 'internal_db_seq_id removed'))
    else:
        results.append(('FAIL', 'ReplaceField:Remove', 'internal_db_seq_id STILL PRESENT'))
    
    if 'debug_flags' not in payload:
        results.append(('PASS', 'ReplaceField:Remove', 'debug_flags removed'))
    else:
        results.append(('FAIL', 'ReplaceField:Remove', 'debug_flags STILL PRESENT'))
    
    # ====================================
    # VALIDATION 4: Field renamed
    # ====================================
    if 'doctor_name' in payload and 'attending_physician' not in payload:
        results.append(('PASS', 'ReplaceField:Rename', 
                        f'attending_physician → doctor_name = "{payload["doctor_name"]}"'))
    elif 'attending_physician' in payload:
        results.append(('FAIL', 'ReplaceField:Rename', 'Field NOT renamed'))
    else:
        results.append(('WARN', 'ReplaceField:Rename', 'Neither field found'))
    
    # ====================================
    # VALIDATION 5: Source metadata added
    # ====================================
    source = payload.get('source_system', 'MISSING')
    if source == 'hospital_east_pg':
        results.append(('PASS', 'InsertField:Static', 
                        f'source_system = "{source}"'))
    elif source == 'MISSING':
        results.append(('FAIL', 'InsertField:Static', 'source_system NOT added'))
    else:
        results.append(('FAIL', 'InsertField:Static', 
                        f'Wrong value: "{source}"'))
    
    # ====================================
    # VALIDATION 6: Ingestion timestamp added
    # ====================================
    ing_ts = payload.get('kafka_ingestion_ts', 'MISSING')
    if ing_ts != 'MISSING' and ing_ts is not None:
        results.append(('PASS', 'InsertField:Timestamp', 
                        f'kafka_ingestion_ts = {ing_ts}'))
    else:
        results.append(('FAIL', 'InsertField:Timestamp', 
                        'kafka_ingestion_ts NOT added'))
    
    # ====================================
    # VALIDATION 7: Essential fields preserved
    # ====================================
    essential_fields = ['patient_id', 'full_name', 'diagnosis', 'treatment_notes']
    for field in essential_fields:
        if field in payload and payload[field] is not None:
            results.append(('PASS', f'Preserved:{field}', 
                           f'{field} = "{str(payload[field])[:30]}..."'))
        else:
            results.append(('FAIL', f'Preserved:{field}', 
                           f'{field} MISSING or null'))
    
    return results

def main():
    consumer = create_consumer()
    
    # --- Find date-routed topics ---
    topics = find_smt_topics(consumer)
    
    if not topics:
        print("=" * 70)
        print("  NO SMT-TRANSFORMED TOPICS FOUND")
        print("  Expected topics matching: compliant_patient_records_*")
        print("=" * 70)
        
        # Fallback: check all topics
        metadata = consumer.list_topics(timeout=10)
        print("\n  Available topics:")
        for t in sorted(metadata.topics):
            if not t.startswith('_'):
                print(f"    → {t}")
        consumer.close()
        return
    
    # --- VALIDATION 7 (from above): TimestampRouter worked ---
    print("=" * 70)
    print("  SMT PIPELINE VALIDATION REPORT")
    print("=" * 70)
    print(f"\n  ✅ TimestampRouter VALIDATED — Found {len(topics)} date-routed topics:")
    for t in topics:
        print(f"     → {t}")
    
    # --- Subscribe and validate records ---
    consumer.subscribe(topics)
    
    print(f"\n{'=' * 70}")
    print("  RECORD-LEVEL VALIDATION")
    print(f"{'=' * 70}")
    
    records_checked = 0
    max_records = 5
    empty_polls = 0
    
    total_pass = 0
    total_fail = 0
    total_warn = 0
    
    while records_checked < max_records and empty_polls < 10:
        msg = consumer.poll(2.0)
        
        if msg is None:
            empty_polls += 1
            continue
        
        if msg.error():
            if msg.error().code() == KafkaError._PARTITION_EOF:
                continue
            print(f"  ERROR: {msg.error()}")
            continue
        
        empty_polls = 0
        records_checked += 1
        
        try:
            value = json.loads(msg.value().decode('utf-8'))
        except json.JSONDecodeError:
            print(f"\n  Record {records_checked}: CANNOT PARSE JSON")
            continue
        
        print(f"\n  ┌─ Record {records_checked} from topic: {msg.topic()}")
        print(f"  │  Partition: {msg.partition()}, Offset: {msg.offset()}")
        print(f"  │")
        
        results = validate_record(value, records_checked)
        
        for status, check_name, detail in results:
            if status == 'PASS':
                icon = '✅'
                total_pass += 1
            elif status == 'FAIL':
                icon = '❌'
                total_fail += 1
            else:
                icon = '⚠️'
                total_warn += 1
            
            print(f"  │  {icon} [{check_name}] {detail}")
        
        print(f"  └─{'─' * 50}")
    
    # --- Summary ---
    print(f"\n{'=' * 70}")
    print(f"  SUMMARY: {records_checked} records validated")
    print(f"  ✅ PASS: {total_pass}  |  ❌ FAIL: {total_fail}  |  ⚠️ WARN: {total_warn}")
    
    if total_fail == 0:
        print(f"\n  🎉 ALL SMT TRANSFORMS VERIFIED SUCCESSFULLY")
    else:
        print(f"\n  🚨 {total_fail} VALIDATION FAILURES — CHECK CONNECTOR CONFIG")
    
    print(f"{'=' * 70}")
    
    consumer.close()

if __name__ == '__main__':
    main()

def inspect_all_connectors():
    """List every connector and its SMT chain"""

    print("=" * 70)
    
    try:
        resp = requests.get(f"{CONNECT_URL}/connectors")
        resp.raise_for_status()
        connectors = resp.json()
    except requests.exceptions.ConnectionError:
        print("  ❌ Cannot connect to Kafka Connect at", CONNECT_URL)
        return []
    
    if not connectors:
        print("  No connectors found.")
        return []
    
    connector_details = []
    
    for name in connectors:
        config_resp = requests.get(f"{CONNECT_URL}/connectors/{name}/config")
        config = config_resp.json()
        
        status_resp = requests.get(f"{CONNECT_URL}/connectors/{name}/status")
        status = status_resp.json()
        
        connector_state = status.get('connector', {}).get('state', 'UNKNOWN')
        task_states = [
            t.get('state', 'UNKNOWN') 
            for t in status.get('tasks', [])
        ]
        
        transforms_str = config.get('transforms', '')
        transform_list = [
            t.strip() for t in transforms_str.split(',') 
            if t.strip()
        ]
        
        print(f"\n  ┌─ Connector: {name}")
        print(f"  │  State: {connector_state}")
        print(f"  │  Tasks: {', '.join(task_states) if task_states else 'None'}")

            print(f"  └─{'─' * 50}")
            continue
        
        print(f"  │  SMT Count: {len(transform_list)}")
        print(f"  │")
        print(f"  │  TRANSFORM CHAIN (execution order):")
        print(f"  │  {'─' * 45}")
        
        smt_details = []
        
        for idx, alias in enumerate(transform_list, 1):
            smt_type = config.get(f'transforms.{alias}.type', 'TYPE_MISSING')
            short_type = smt_type.split('.')[-1] if smt_type != 'TYPE_MISSING' else 'MISSING!'
            
            # Detect target (Key or Value)
            target = 'Value' if '$Value' in smt_type else (
                'Key' if '$Key' in smt_type else 'Both/Auto'
            )
            
            # Extract relevant sub-configs
            sub_configs = {
                k.replace(f'transforms.{alias}.', ''): v 
                for k, v in config.items() 
                if k.startswith(f'transforms.{alias}.') and k != f'transforms.{alias}.type'
            }
            
            # Check for predicate
            predicate = sub_configs.pop('predicate', None)
            negate = sub_configs.pop('negate', None)
            
            arrow = '→' if idx < len(transform_list) else '⏹'
            
            print(f"  │  {idx}. [{arrow}] {alias}")
            print(f"  │       Type: {short_type} (target: {target})")
            
            if sub_configs:
                for sk, sv in sub_configs.items():
                    print(f"  │       {sk}: {sv}")
            
            if predicate:
                negate_str = " (negated)" if negate == 'true' else ""
                print(f"  │       predicate: {predicate}{negate_str}")
            
            print(f"  │")
            
            smt_details.append({
                'alias': alias,
                'type': short_type,
                'target': target,
                'configs': sub_configs,
                'predicate': predicate
            })
        
        # --- Detect potential issues ---
        issues = detect_smt_issues(transform_list, config)

        connector_details.append({
            'name': name,
            'state': connector_state,
            'transforms': smt_details
        })
    
    return connector_details


def detect_smt_issues(transform_list, config):
    """
    Analyze SMT chain for common misconfigurations
    Based on pitfalls documented earlier in this project
    """
    issues = []
    
    # ISSUE 1: Alias defined but type missing
    for alias in transform_list:
        smt_type = config.get(f'transforms.{alias}.type', '')
        if not smt_type:
            issues.append(
                f'Transform "{alias}" has no .type defined — will be IGNORED'
            )
    
    # ISSUE 2: Check for case mismatches
    all_transform_keys = [
        k for k in config.keys() 
        if k.startswith('transforms.') and k != 'transforms'
    ]
    
    for key in all_transform_keys:
        parts = key.split('.')
        if len(parts) >= 2:
            alias_in_key = parts[1]
            if alias_in_key not in transform_list:
                # Check case-insensitive match
                lower_aliases = {a.lower(): a for a in transform_list}
                if alias_in_key.lower() in lower_aliases:
                    correct = lower_aliases[alias_in_key.lower()]
                    issues.append(
                        f'Case mismatch: config uses "{alias_in_key}" '
                        f'but alias list has "{correct}"'
                    )
    
    # ISSUE 3: ReplaceField with both include and exclude
    for alias in transform_list:
        smt_type = config.get(f'transforms.{alias}.type', '')
        if 'ReplaceField' in smt_type:
            has_include = f'transforms.{alias}.include' in config
            has_exclude = f'transforms.{alias}.exclude' in config
            if has_include and has_exclude:
                issues.append(
                    f'"{alias}": ReplaceField has BOTH include AND exclude '
                    f'— only one should be used'
                )
    
    # ISSUE 4: MaskField after ReplaceField that removes the same field
    masked_fields = set()
    removed_fields = set()
    
    for alias in transform_list:
        smt_type = config.get(f'transforms.{alias}.type', '')
        
        if 'MaskField' in smt_type:
            fields = config.get(f'transforms.{alias}.fields', '')
            masked_fields = set(f.strip() for f in fields.split(','))
        
        if 'ReplaceField' in smt_type:
            exclude = config.get(f'transforms.{alias}.exclude', '')
            if exclude:
                current_excluded = set(f.strip() for f in exclude.split(','))
                overlap = masked_fields & current_excluded
                if overlap:
                    # This is fine — mask then remove
                    pass
                removed_fields.update(current_excluded)
    
    # Check if masking happens AFTER removal
    for alias in transform_list:
        smt_type = config.get(f'transforms.{alias}.type', '')
        if 'MaskField' in smt_type:
            fields = config.get(f'transforms.{alias}.fields', '')
            mask_targets = set(f.strip() for f in fields.split(','))
            overlap = mask_targets & removed_fields
            if overlap:
                issues.append(
                    f'"{alias}": Tries to mask {overlap} but field(s) '
                    f'already removed by earlier ReplaceField'
                )
    
    # ISSUE 5: TimestampRouter with very fine granularity
    for alias in transform_list:
        smt_type = config.get(f'transforms.{alias}.type', '')
        if 'TimestampRouter' in smt_type:
            ts_format = config.get(
                f'transforms.{alias}.timestamp.format', ''
            )
            if 'HH' in ts_format or 'mm' in ts_format:
                issues.append(
                    f'"{alias}": TimestampRouter uses hour/minute granularity '
                    f'("{ts_format}") — risk of topic explosion'
                )
    
    # ISSUE 6: Too many transforms
    if len(transform_list) > 7:
        issues.append(
            f'Chain has {len(transform_list)} transforms — '
            f'consider moving logic to a stream processor'
        )
    
    return issues

def compare_raw_vs_transformed():
    """
    Compare records from raw topic vs SMT-transformed topic
    Shows exactly what each SMT did
    """
    from confluent_kafka import Consumer, KafkaError
    
    print("\n" + "=" * 70)
    print("  RAW vs TRANSFORMED RECORD COMPARISON")
    print("=" * 70)
    
    # --- Consume one raw record ---
    raw_consumer = Consumer({
        'bootstrap.servers': 'localhost:9092',
        'group.id': 'smt-debug-raw-' + str(hash("raw"))[-6:],
        'auto.offset.reset': 'earliest'
    })
    
    # Find raw and transformed topics
    metadata = raw_consumer.list_topics(timeout=10)
    all_topics = sorted(metadata.topics.keys())
    
    raw_topics = [t for t in all_topics if t.startswith('raw_')]
    transformed_topics = [t for t in all_topics if t.startswith('compliant_')]
    
    if not raw_topics:
        print("  No raw topics found (prefix 'raw_')")
        print("  Run the baseline connector first (Step 3)")
        raw_consumer.close()
        return
    
    if not transformed_topics:
        print("  No transformed topics found (prefix 'compliant_')")
        print("  Run the production connector first (Step 5)")
        raw_consumer.close()
        return
    
    # Get raw record
    raw_consumer.subscribe(raw_topics[:1])
    raw_record = None
    
    for _ in range(20):
        msg = raw_consumer.poll(1.0)
        if msg and not msg.error():
            raw_record = json.loads(msg.value().decode('utf-8'))
            break
    raw_consumer.close()
    
    # Get transformed record
    trans_consumer = Consumer({
        'bootstrap.servers': 'localhost:9092',
        'group.id': 'smt-debug-trans-' + str(hash("trans"))[-6:],
        'auto.offset.reset': 'earliest'
    })
    trans_consumer.subscribe(transformed_topics[:1])
    trans_record = None
    
    for _ in range(20):
        msg = trans_consumer.poll(1.0)
        if msg and not msg.error():
            trans_record = json.loads(msg.value().decode('utf-8'))
            break
    trans_consumer.close()
    
    if not raw_record or not trans_record:
        print("  Could not fetch records from both topics")
        return
    
    # Extract payloads
    raw_payload = raw_record.get('payload', raw_record)
    trans_payload = trans_record.get('payload', trans_record)
    
    raw_fields = set(raw_payload.keys())
    trans_fields = set(trans_payload.keys())
    
    removed = raw_fields - trans_fields
    added = trans_fields - raw_fields
    common = raw_fields & trans_fields
    
    print(f"\n  📥 RAW RECORD (from {raw_topics[0]}):")
    print(f"     Fields: {len(raw_fields)}")
    for k, v in raw_payload.items():
        print(f"     {k}: {repr(v)[:60]}")
    
    print(f"\n  📤 TRANSFORMED RECORD (from {transformed_topics[0]}):")
    print(f"     Fields: {len(trans_fields)}")
    for k, v in trans_payload.items():
        print(f"     {k}: {repr(v)[:60]}")
    
    print(f"\n  🔍 DIFF ANALYSIS:")
    print(f"  {'─' * 55}")
    
    if removed:
        print(f"  ❌ REMOVED FIELDS ({len(removed)}):")
        for f in removed:
            print(f"     - {f}: was {repr(raw_payload[f])[:40]}")
    
    if added:
        print(f"  ✅ ADDED FIELDS ({len(added)}):")
        for f in added:
            print(f"     + {f}: = {repr(trans_payload[f])[:40]}")
    
    modified = []
    for f in common:
        if raw_payload[f] != trans_payload[f]:
            modified.append(f)
    
    if modified:
        print(f"  🔄 MODIFIED FIELDS ({len(modified)}):")
        for f in modified:
            print(f"     ~ {f}:")
            print(f"       BEFORE: {repr(raw_payload[f])[:40]}")
            print(f"       AFTER:  {repr(trans_payload[f])[:40]}")
    
    unchanged = common - set(modified)
    if unchanged:
        print(f"  ➡️  UNCHANGED FIELDS ({len(unchanged)}):")
        for f in unchanged:
            print(f"     = {f}")
    
    print(f"  {'─' * 55}")
