PROJECT 13: THE E-COMMERCE CART — STRUCTURED MESSAGE DESIGN

🔴 HYPOTHETICAL PROBLEM

You are building an e-commerce platform — "MegaShop"

Multiple microservices communicate through Kafka:
  → Cart Service: adds/removes items, updates quantities
  → Order Service: creates orders from carts
  → Payment Service: processes payments
  → Inventory Service: reserves and releases stock
  → Notification Service: sends emails and push notifications
  → Analytics Service: tracks user behavior

The current state is CHAOS:
  → Cart service sends: {"item": "laptop", "qty": 1}
  → Order service sends: {"order_id": "123", "items": [...], "total": 999}
  → Payment service sends: {"payment_status": "success", "order": "123"}
  → Each team invented their own message format
  → No standard for timestamps (some use epoch, some use ISO strings)
  → No way to trace a message across services
  → No metadata — you can't tell WHO sent a message or WHEN
  → A new developer asks: "What fields does the cart event have?"
    Nobody knows for sure

The CTO demands:
  "Every message must be self-describing"
  "We need to trace an order from cart → payment → delivery"
  "Messages must have versioning — we'll evolve schemas over time"
  "Metadata must be SEPARATE from business data"
  "We need message size controls — someone sent a 50MB image in a message"

The questions:
  → What should a well-designed Kafka message look like?
  → Where does metadata go — in the value or in headers?
  → How do you handle timestamps correctly?
  → How do you design messages for schema evolution?
  → What are the message size implications?
  → How do you design for cross-service traceability?

🔵 HOW TO APPROACH IT

Step 1: Understand that a Kafka message has FOUR components
  → Key: determines partition (Project 4)
  → Value: the actual payload
  → Headers: metadata key-value pairs (NEW in this project)
  → Timestamp: when the message was created or ingested (NEW)

Step 2: Recognize the SEPARATION OF CONCERNS principle
  → Business data → goes in the VALUE
  → Routing/tracing metadata → goes in HEADERS
  → Partition routing → goes in the KEY
  → Time information → goes in TIMESTAMP + structured fields

Step 3: Design messages for CONSUMERS, not producers
  → Every consumer needs to know what fields to expect
  → Messages must be self-describing or schema-referenced
  → Backward compatibility matters from day one

Step 4: Connect to what we already know
  → Project 4: Key determines partition routing
  → Project 7: Serialization (JSON) — now we structure WHAT we serialize
  → Project 12: Python producer/consumer patterns — now we use headers, timestamps
  → Project 11: Delivery guarantees — message design affects deduplication

🟡 WHAT COULD BE DONE

OPTION A: Flat JSON — everything in the value
  {
    "event_type": "cart.item.added",
    "user_id": "user-123",
    "product_id": "PROD-001",
    "quantity": 1,
    "price": 1299.99,
    "timestamp": "2024-01-15T10:30:00Z",
    "source": "cart-service",
    "correlation_id": "uuid-abc"
  }
  
  → Simple but mixes metadata with business data
  → Every consumer must parse full JSON to check event_type
  → No separation of concerns
  → Hard to add routing metadata without polluting business payload

OPTION B: Envelope pattern — metadata wrapper + data payload
  {
    "event_id": "evt-456",
    "event_type": "cart.item.added",
    "event_time": "2024-01-15T10:30:00.000Z",
    "metadata": { "source": "cart-service", "version": "1.0" },
    "data": { "user_id": "user-123", "product_id": "PROD-001", ... }
  }
  
  → Clean separation of metadata and business data
  → Consistent envelope across ALL services
  → Still requires full JSON parse to read metadata
  → ✅ GOOD — this is what we'll use for the VALUE

OPTION C: Headers for metadata + clean value for business data
  Headers: {event_type: "cart.item.added", correlation_id: "uuid-abc"}
  Value: { "data": { ... business fields only ... } }
  
  → Headers are accessible WITHOUT parsing the value
  → Enables routing/filtering without deserialization
  → Headers have size limits (practical ~few KB)
  → ✅ BEST — combine with Option B for complete solution

OPTION D: Binary format (Avro/Protobuf) for value
  → Compact, schema-enforced, fast
  → Requires Schema Registry
  → ✅ COVERED IN PROJECT 16 — we build the JSON foundation here

✅ WHAT SHOULD DO

1. USE THE ENVELOPE PATTERN FOR MESSAGE VALUES
   → Consistent structure: event_id, event_type, event_time, data, metadata
   → Every service produces the same outer structure
   → Only the "data" field varies per event type
   → Consumers can process the envelope generically

2. USE HEADERS FOR ROUTING AND TRACING METADATA
   → event_type in headers → consumers can filter WITHOUT deserializing
   → correlation_id in headers → distributed tracing across services
   → source_service in headers → know who produced the message
   → content_type in headers → know how to deserialize the value
   → schema_version in headers → know which schema to use

3. USE MEANINGFUL KEYS FOR PARTITION ROUTING
   → user_id as key → all user events in same partition → ordered
   → order_id as key → all order events in same partition → ordered
   → Key choice depends on the ORDERING REQUIREMENT
   → (as learned in Project 4)

4. USE ISO-8601 FOR TIMESTAMPS IN THE VALUE
   → "2024-01-15T10:30:00.000Z" — human readable
   → Include timezone (always UTC with Z suffix)
   → Millisecond precision minimum
   → Also set the Kafka message timestamp (epoch ms)

5. INCLUDE A UNIQUE EVENT ID IN EVERY MESSAGE
   → UUID v4 for global uniqueness
   → Enables deduplication at consumer side
   → (as shown in Project 11 — exactly-once simulation)

6. KEEP MESSAGES SMALL — REFERENCE LARGE DATA
   → Kafka default max: 1MB per message
   → Ideal message size: 1-10KB
   → For large data: store in object storage, put REFERENCE in message
   → Never put images, files, or large blobs directly in messages

7. DESIGN FOR BACKWARD COMPATIBILITY FROM DAY ONE
   → New fields should be OPTIONAL with defaults
   → Never remove or rename existing fields
   → Use version field to signal schema changes
   → (Project 38 covers schema evolution in depth)

❌ WHAT SHOULD NOT DO

1. DO NOT MIX METADATA AND BUSINESS DATA IN A FLAT STRUCTURE
   → Makes it impossible to process generically
   → Every consumer must know every field
   → Adding metadata fields pollutes the business domain

2. DO NOT USE HEADERS FOR LARGE DATA
   → Headers are meant for small metadata (< 1KB each)
   → Not indexed, not queryable
   → Large headers waste network bandwidth on every message

3. DO NOT USE INCONSISTENT TIMESTAMP FORMATS
   → "2024-01-15" (no time? which timezone?)
   → 1705312200 (seconds or milliseconds? UTC?)
   → "Jan 15, 2024 10:30 AM EST" (locale-dependent, ambiguous)
   → ALWAYS: ISO-8601 with timezone in value, epoch ms for Kafka timestamp

4. DO NOT USE NULL KEYS WHEN YOU NEED ORDERING
   → Null key → round-robin across partitions → no ordering
   → If user events must be ordered, user_id MUST be the key
   → (as learned in Project 4)

5. DO NOT EMBED THE ENTIRE RELATED ENTITY
   → Cart event should NOT contain the full product catalog entry
   → Use product_id reference, not the full product object
   → Reduces message size, avoids stale data in messages

6. DO NOT USE SEQUENTIAL IDS AS EVENT IDS
   → "event-1", "event-2" → collision across services
   → UUIDs prevent cross-service collision
   → Sequential IDs also leak information about volume

7. DO NOT IGNORE MESSAGE SIZE
   → Kafka has message.max.bytes (default 1MB broker side)
   → max.request.size (default 1MB producer side)
   → Exceeding these causes silent drops or errors
   → Monitor average message size as a metric

8. DO NOT PUT SENSITIVE DATA IN HEADERS
   → Headers are not encrypted separately from the value
   → Headers are visible in many monitoring tools
   → Passwords, tokens, PII should be in the encrypted value only

🚫 WHEN NOT TO USE

DO NOT USE ENVELOPE PATTERN WHEN:
  → You're producing raw log lines (just use string value)
  → You're using Avro with Schema Registry (schema IS the envelope — Project 16)
  → Extreme latency sensitivity where every byte matters
  → Internal system metrics (overhead not worth it)

DO NOT USE HEADERS WHEN:
  → Your Kafka version is older than 0.11 (headers not supported)
  → You need to SEARCH/INDEX by header values (Kafka doesn't index headers)
  → The metadata exceeds a few KB (put it in the value instead)
  → You're using Kafka Connect that doesn't propagate headers

DO NOT USE JSON FOR HIGH-THROUGHPUT SCENARIOS:
  → JSON is verbose — "quantity": 1 vs 2 bytes in Avro
  → JSON parsing is slow compared to binary formats
  → For >100K msgs/sec, consider Avro (Project 16) or Protobuf
  → JSON is fine for <50K msgs/sec with reasonable message sizes

DO NOT USE COMPLEX NESTED STRUCTURES:
  → Deeply nested JSON (5+ levels) is hard to evolve
  → Consumers must navigate complex paths
  → Flatten where possible: data.item.product.category.name → data.product_category

KEY OBSERVATIONS:
  → Headers are stored PER RECORD (not per batch)
  → Headers add bytes to every message (size consideration)
  → Timestamps are stored as DELTAS within a batch (space efficient)
  → Key and value are just byte arrays — Kafka doesn't know JSON/Avro
  → Compression applies to the ENTIRE batch (not individual records)


═══════════════════════════════════════════════════════════════
              HEADER MECHANICS
═══════════════════════════════════════════════════════════════

Headers are:
  → List of (key, value) pairs
  → Key: string (UTF-8)
  → Value: bytes (or None)
  → DUPLICATE KEYS ARE ALLOWED (unlike HTTP headers)
  → Order is preserved
  → NOT indexed by Kafka — cannot query/filter by header on broker

Producer side:
  producer.produce(
      topic='orders',
      key=b'user-123',
      value=b'{"data": {...}}',
      headers={                        # Dict style
          'event_type': 'order.created',
          'correlation_id': 'abc-123',
      }
  )
  
  # OR list style (allows duplicate keys):
  headers=[
      ('event_type', b'order.created'),
      ('trace_id', b'abc-123'),
      ('trace_id', b'def-456'),        # Duplicate key OK
  ]

Consumer side:
  msg = consumer.poll(1.0)
  headers = msg.headers()
  # Returns: [('event_type', b'order.created'), ('trace_id', b'abc-123')]
  
  # Convert to dict (loses duplicate keys):
  header_dict = {k: v.decode('utf-8') for k, v in headers} if headers else {}


═══════════════════════════════════════════════════════════════
              TIMESTAMP MECHANICS
═══════════════════════════════════════════════════════════════

Two timestamp modes (configured per-topic):

1. CreateTime (default):
   → Producer sets the timestamp
   → producer.produce(..., timestamp=epoch_ms)
   → If not set: librdkafka uses current time automatically
   → Represents: "when did the event HAPPEN"
   
2. LogAppendTime:
   → Broker OVERWRITES the timestamp on write
   → Whatever the producer sends is REPLACED
   → Represents: "when did Kafka RECEIVE this"
   → Useful for: audit trails, compliance

Access on consumer side:
  msg = consumer.poll(1.0)
  ts_type, ts_value = msg.timestamp()
  # ts_type: 0=CREATE_TIME, 1=LOG_APPEND_TIME
  # ts_value: milliseconds since epoch


═══════════════════════════════════════════════════════════════
              MESSAGE SIZE CALCULATION
═══════════════════════════════════════════════════════════════

Total message size = key_bytes + value_bytes + headers_bytes + record_overhead

Record overhead ≈ 50-100 bytes (length fields, timestamps, attributes)

Example calculation:
  Key: "user-12345"           → 10 bytes
  Value: JSON envelope        → 500 bytes
  Headers: 3 headers          → ~100 bytes
  Overhead:                   → ~70 bytes
  ─────────────────────────────────────
  Total per record:           → ~680 bytes
  
  In a batch of 100 records:
    100 × 680 = 68,000 bytes
    + batch header ≈ 60 bytes
    + LZ4 compression ≈ 40% reduction
    ≈ ~41,000 bytes on wire

Size limits (defaults):
  message.max.bytes (broker): 1,048,576 (1MB) per BATCH
  max.request.size (producer): 1,048,576 (1MB) per request
  fetch.max.bytes (consumer): 52,428,800 (50MB) per fetch
  max.message.bytes (topic-level): 1,048,576 (1MB)

🕳️ COMMON PITFALLS AND FAILURE MODES

PITFALL 1: "Event type only in the value — consumer must deserialize to route"
  → Consumer receives 10 different event types on one topic
  → Must parse full JSON just to check event_type
  → Wastes CPU on irrelevant messages
  → FIX: Put event_type in HEADERS for pre-parse filtering

PITFALL 2: "Timestamps inconsistent across services"
  → Cart service uses epoch seconds
  → Order service uses epoch milliseconds
  → Payment service uses ISO-8601 with timezone
  → Consumer gets confused, time-based queries break
  → FIX: Standardize on ISO-8601 UTC in value, epoch ms for Kafka timestamp

PITFALL 3: "Changed a field name — broke all consumers"
  → Renamed "price" to "unit_price" in cart events
  → All downstream consumers that read "price" break
  → FIX: NEVER rename fields — add new field, deprecate old one
  → Include schema_version so consumers can handle both

PITFALL 4: "Messages getting rejected — too large"
  → Someone embedded a base64-encoded image in the message
  → Exceeded message.max.bytes → message silently dropped or errored
  → FIX: Store large data externally, put URL/reference in message
  → Monitor average and max message sizes

PITFALL 5: "Can't correlate events across services"
  → Cart event has no link to the resulting order event
  → Order event has no link back to the payment event
  → FIX: Use correlation_id — generate at the FIRST event
  → Pass it through headers to ALL subsequent events in the chain

PITFALL 6: "Duplicate events because no event_id"
  → Producer retried (Project 11), consumer sees message twice
  → Without event_id, consumer can't deduplicate
  → FIX: Every message gets a UUID event_id
  → Consumer checks "have I seen this event_id before?"

PITFALL 7: "Headers with None values cause crashes"
  → header value is bytes or None — not string
  → Calling .decode() on None → AttributeError
  → FIX: Always check for None before decoding headers

HEADER OVERHEAD:
  → Each header: key_length + key_bytes + value_length + value_bytes
  → 5 typical headers ≈ 100-200 bytes per message
  → At 100K msgs/sec: 10-20 MB/sec JUST for headers
  → Keep headers concise — short key names, compact values

COMPRESSION EFFECTIVENESS BY MESSAGE DESIGN:
  → JSON with envelope: 50-70% compression ratio (LZ4)
  → Flat JSON: 40-60% compression
  → Binary (Avro): 20-40% compression (already compact)
  → Headers are NOT compressed (only value is compressed in batch)

TOPIC-PER-EVENT-TYPE VS SINGLE-TOPIC:

class EventEnvelope:
    """
    Standard message envelope for all e-commerce events.
    
    Structure:
      {
        "event_id":    unique identifier (UUID)
        "event_type":  dot-notation event name
        "event_time":  ISO-8601 UTC timestamp
        "data":        business payload (varies per event type)
        "metadata":    cross-cutting concerns
      }
    
    WHY this structure:
      → event_id: deduplication (Project 11 exactly-once simulation)
      → event_type: routing and filtering
      → event_time: business timestamp (may differ from Kafka timestamp)
      → data: clean separation of business payload
      → metadata: tracing, versioning, source identification
    """
    
    @staticmethod
    def create(event_type, data, correlation_id=None, source_service=None,
               schema_version="1.0", event_time=None):
        """
        Create a standardized event envelope.
        
        Returns: (envelope_dict, headers_dict, key_bytes)
        """
        event_id = str(uuid.uuid4())
        
        if event_time is None:
            event_time = datetime.now(timezone.utc).isoformat()
        
        if correlation_id is None:
            correlation_id = str(uuid.uuid4())
        
        envelope = {
            'event_id': event_id,
            'event_type': event_type,
            'event_time': event_time,
            'data': data,
            'metadata': {
                'schema_version': schema_version,
                'source_service': source_service or 'unknown',
                'correlation_id': correlation_id,
                'produced_at': datetime.now(timezone.utc).isoformat(),
            }
        }
        
        # ── Headers for routing/filtering without deserialization ──
        headers = {
            'event_type': event_type,
            'event_version': schema_version,
            'correlation_id': correlation_id,
            'source_service': source_service or 'unknown',
            'content_type': 'application/json',
        }
        
        return envelope, headers
    
    @staticmethod
    def validate(envelope):
        """
        Validate that a message follows the envelope structure.
        Returns (is_valid, errors_list).
        """
        errors = []
        
        required_fields = ['event_id', 'event_type', 'event_time', 'data']
        for field in required_fields:
            if field not in envelope:
                errors.append(f"Missing required field: {field}")
        
        if 'metadata' in envelope:
            meta_fields = ['schema_version', 'source_service', 'correlation_id']
            for field in meta_fields:
                if field not in envelope.get('metadata', {}):
                    errors.append(f"Missing metadata field: {field}")
        else:
            errors.append("Missing metadata section")
        
        return len(errors) == 0, errors

class EcommerceProducer:
    """
    Structured event producer for e-commerce platform.
    
    Uses:
      → Envelope pattern from 01_message_schemas.py
      → Producer patterns from Project 12 (callbacks, poll, flush)
      → Delivery guarantees from Project 11 (acks, idempotence)
      → Key-based routing from Project 4 (partition by user/order)
    """
    
    def __init__(self, bootstrap_servers):
        self.running = True
        self.stats = defaultdict(int)
        
        self.producer = Producer({
            'bootstrap.servers': bootstrap_servers,
            'acks': 'all',
            'enable.idempotence': True,
            'batch.size': 32768,
            'linger.ms': 10,
            'compression.type': 'lz4',
        })
        
        signal.signal(signal.SIGINT, lambda s, f: setattr(self, 'running', False))
    
    def _delivery_callback(self, err, msg):
        """Delivery callback — pattern from Project 12."""
        if err:
            self.stats['delivery_errors'] += 1
            event_type = 'unknown'
            if msg.headers():
                for k, v in msg.headers():
                    if k == 'event_type':
                        event_type = v.decode('utf-8') if v else 'unknown'
                        break
            print(f"  ❌ Failed: {event_type} — {err}")
        else:
            self.stats['delivered'] += 1
    
    def publish_event(self, envelope, headers, key):
        """
        Publish a structured event to the correct topic.
        
        The topic is determined by the event_type in the envelope.
        This is the CENTRAL publishing method — all events go through here.
        """
        event_type = envelope['event_type']
        topic = TopicRegistry.get_topic(event_type)
        
        # ── Serialize ──
        key_bytes = key.encode('utf-8') if isinstance(key, str) else key
        value_bytes = json.dumps(envelope).encode('utf-8')
        
        # ── Convert headers to list of tuples with bytes values ──
        header_list = [
            (k, v.encode('utf-8') if isinstance(v, str) else v)
            for k, v in headers.items()
        ]
        
        # ── Size check ──
        total_size = len(key_bytes) + len(value_bytes)
        if total_size > 900000:  # 900KB warning threshold
            print(f"  ⚠️  Large message: {total_size} bytes for {event_type}")
            self.stats['oversized_warnings'] += 1
        
        # ── Produce with backpressure handling (Project 12 pattern) ──
        try:
            self.producer.produce(
                topic=topic,
                key=key_bytes,
                value=value_bytes,
                headers=header_list,
                callback=self._delivery_callback,
                timestamp=int(time.time() * 1000),  # Epoch ms
            )
            self.stats['produced'] += 1
            self.stats[f'produced_{event_type}'] += 1
            
        except BufferError:
            self.stats['buffer_errors'] += 1
            self.producer.poll(1.0)
            # Retry once
            self.producer.produce(
                topic=topic,
                key=key_bytes,
                value=value_bytes,
                headers=header_list,
                callback=self._delivery_callback,
            )
        
        # ── Serve callbacks periodically ──
        self.producer.poll(0)
    
    def shutdown(self):
        remaining = self.producer.flush(timeout=15)
        stats = dict(self.stats)
        print(f"\n  📊 Producer stats: {json.dumps(stats, indent=2)}")
        if remaining > 0:
            print(f"  ⚠️  {remaining} messages unflushed")
        return stats

      → Filters by event_type in headers
      → Only deserializes value for matching events
      → Validates envelope structure
      → Tracks correlation chains
    """
    
    def __init__(self, bootstrap_servers, group_id, topics,
                 event_type_filter=None):
        """
        event_type_filter: list of event types to process
                          None = process all events
        """
        self.running = True
        self.event_type_filter = set(event_type_filter) if event_type_filter else None
        self.stats = defaultdict(int)
        self.correlation_chains = defaultdict(list)
        
        self.consumer = Consumer({
            'bootstrap.servers': bootstrap_servers,
            'group.id': group_id,
            'auto.offset.reset': 'earliest',
            'enable.auto.commit': False,
        })
        
        self.consumer.subscribe(topics)
        signal.signal(signal.SIGINT, lambda s, f: setattr(self, 'running', False))
    
    def _parse_headers(self, msg):
        """
        Extract headers into a clean dictionary.
        
        IMPORTANT: Header values are bytes or None.
        Must handle None values to avoid AttributeError.
        (Pitfall from this project's pitfalls section)
        """
        raw_headers = msg.headers()
        if not raw_headers:
            return {}
        
        parsed = {}
        for key, value in raw_headers:
            if value is not None:
                try:
                    parsed[key] = value.decode('utf-8')
                except UnicodeDecodeError:
                    parsed[key] = value  # Keep as bytes
            else:
                parsed[key] = None
        
        return parsed
    
    def _should_process(self, headers):
        """
        Check if this message should be processed based on headers.
        
        THIS IS THE KEY OPTIMIZATION:
          → Read event_type from headers (fast, no JSON parsing)
          → Skip irrelevant events immediately
          → Only deserialize value for matching events
        """
        if self.event_type_filter is None:
            return True  # Process everything
        
        event_type = headers.get('event_type')
        if event_type is None:
            self.stats['missing_event_type_header'] += 1
            return True  # Process if no header (defensive)
        
        return event_type in self.event_type_filter
    
    def _process_message(self, msg):
        """
        Full message processing pipeline:
          1. Parse headers (fast)
          2. Filter by event type (fast)
          3. Deserialize value (only if needed)
          4. Validate envelope structure
          5. Process business data
          6. Track correlation chain
        """
        # ── Step 1: Parse headers ──
        headers = self._parse_headers(msg)
        self.stats['total_received'] += 1
        
        # ── Step 2: Filter by event type ──
        if not self._should_process(headers):
            self.stats['filtered_out'] += 1
            return None
        
        # ── Step 3: Deserialize value ──
        try:
            value = json.loads(msg.value().decode('utf-8'))
        except (json.JSONDecodeError, UnicodeDecodeError) as e:
            self.stats['deserialize_errors'] += 1
            return None
        
        # ── Step 4: Validate envelope ──
        is_valid, errors = EventEnvelope.validate(value)
        if not is_valid:
            self.stats['invalid_envelope'] += 1
            print(f"  ⚠️  Invalid envelope: {errors}")
            return None
        
        self.stats['valid_processed'] += 1
        event_type = value['event_type']
        self.stats[f'processed_{event_type}'] += 1
        
        # ── Step 5: Track correlation chain ──
        correlation_id = value.get('metadata', {}).get('correlation_id')
        if correlation_id:
            self.correlation_chains[correlation_id].append({
                'event_type': event_type,
                'event_id': value['event_id'],
                'event_time': value['event_time'],
                'topic': msg.topic(),
                'partition': msg.partition(),
                'offset': msg.offset(),
            })
        
        # ── Step 6: Extract key metadata ──
        key = msg.key().decode('utf-8') if msg.key() else None
        ts_type, ts_value = msg.timestamp()
        
        result = {
            'key': key,
            'event_type': event_type,
            'event_id': value['event_id'],
            'event_time': value['event_time'],
            'data': value['data'],
            'metadata': value.get('metadata', {}),
            'headers': headers,
            'kafka_metadata': {
                'topic': msg.topic(),
                'partition': msg.partition(),
                'offset': msg.offset(),
                'timestamp_type': 'CreateTime' if ts_type == 0 else 'LogAppendTime',
                'timestamp_ms': ts_value,
            }
        }
        
        return result
    
    def consume_loop(self, max_messages=500, commit_interval=50):
        """
        Main consumption loop with header filtering and batch commits.
        """
        print(f"  📡 Consuming (filter={self.event_type_filter or 'ALL'})...\n")
        
        messages_since_commit = 0
        processed_results = []
        start_time = time.time()
        
        try:
            while self.running and self.stats['total_received'] < max_messages:
                msg = self.consumer.poll(timeout=1.0)
                
                if msg is None:
                    if messages_since_commit > 0:
                        self.consumer.commit(asynchronous=False)
                        messages_since_commit = 0
                    continue
                
                if msg.error():
                    if msg.error().code() == KafkaError._PARTITION_EOF:
                        continue
                    self.stats['consume_errors'] += 1
                    continue
                
                # ── Process structured message ──
                result = self._process_message(msg)
                
                if result:
                    processed_results.append(result)
                    
                    # ── Display processing result ──
                    event_type = result['event_type']
                    event_id = result['event_id'][:8]
                    corr_id = result['metadata'].get('correlation_id', '')[:8]
                    
                    icon = {
                        'cart': '🛒',
                        'order': '📦',
                        'payment': '💳',
                        'inventory': '📋',
                        'notification': '🔔',
                    }.get(event_type.split('.')[0], '📨')
                    
                    print(f"  {icon} {event_type:<30} "
                          f"evt={event_id}... "
                          f"corr={corr_id}... "
                          f"P{result['kafka_metadata']['partition']}:"
                          f"{result['kafka_metadata']['offset']}")
                
                messages_since_commit += 1
                
                if messages_since_commit >= commit_interval:
                    self.consumer.commit(asynchronous=False)
                    messages_since_commit = 0
        
        finally:
            if messages_since_commit > 0:
                try:
                    self.consumer.commit(asynchronous=False)
                except Exception:
                    pass
            self.consumer.close()
        
        elapsed = time.time() - start_time
        
        return processed_results, elapsed
    
    def display_stats(self, elapsed):
        """Display consumption statistics."""
        stats = dict(self.stats)
        total = stats.get('total_received', 0)
        filtered = stats.get('filtered_out', 0)

        print(f"  │  Invalid envelopes:   {stats.get('invalid_envelope', 0):>8}"
              f"                    │")
        print(f"  │  Deserialize errors:  {stats.get('deserialize_errors', 0):>8}"
              f"                    │")
        print(f"  │  Elapsed time:        {elapsed:>7.1f}s                   │")
        
        if elapsed > 0 and total > 0:
            print(f"  │  Rate:                {total/elapsed:>7.0f} msg/s"

        print(f"  ├────────────────────────────────────────────────────┤")
        print(f"  │  EVENTS BY TYPE:                                    │")
        
        for key, val in sorted(stats.items()):

                print(f"  │    {event_name:<30} {val:>6}        │")
        
        print(f"  └────────────────────────────────────────────────────┘")
    
    def display_correlation_chains(self, max_chains=5):

# PURPOSE: Analyze message sizes across topics
# TEACHES: Understanding size implications of message design
#          Monitoring message sizes in production

from confluent_kafka import Consumer, KafkaError
import json
import time
import signal
from collections import defaultdict
import statistics

BOOTSTRAP = 'localhost:9092,localhost:9093,localhost:9094'


class MessageSizeAnalyzer:
    """
    Consumes messages and analyzes their size distribution.
    
    Helps answer:
      → How big are our messages on average?
      → What's the overhead of the envelope pattern?
      → Which event types produce the largest messages?
      → Are we at risk of hitting message.max.bytes?
    """
    
    def __init__(self, bootstrap_servers, topics):
        self.running = True
        self.topics = topics
        
        self.consumer = Consumer({
            'bootstrap.servers': bootstrap_servers,
            'group.id': f'size-analyzer-{int(time.time())}',
            'auto.offset.reset': 'earliest',
            'enable.auto.commit': True,
        })
        
        self.consumer.subscribe(topics)
        signal.signal(signal.SIGINT, lambda s, f: setattr(self, 'running', False))
        
        # ── Size tracking ──
        self.sizes_by_topic = defaultdict(list)
        self.sizes_by_event_type = defaultdict(list)
        self.component_sizes = {
            'key': [],
            'value': [],
            'headers': [],
            'envelope_overhead': [],
            'business_data': [],
        }
    
    def analyze_message(self, msg):
        """Break down a message into its size components."""
        topic = msg.topic()
        
        # ── Key size ──
        key_bytes = msg.key() if msg.key() else b''
        key_size = len(key_bytes)
        
        # ── Value size ──
        value_bytes = msg.value() if msg.value() else b''
        value_size = len(value_bytes)
        
        # ── Header size ──
        header_size = 0
        if msg.headers():
            for k, v in msg.headers():
                header_size += len(k.encode('utf-8') if isinstance(k, str) else k)
                if v:
                    header_size += len(v)
        
        total_size = key_size + value_size + header_size
        
        # ── Track by topic ──
        self.sizes_by_topic[topic].append(total_size)
        
        # ── Track components ──
        self.component_sizes['key'].append(key_size)
        self.component_sizes['value'].append(value_size)
        self.component_sizes['headers'].append(header_size)
        
        # ── Try to break down envelope vs business data ──
        event_type = 'unknown'
        try:
            value = json.loads(value_bytes.decode('utf-8'))
            event_type = value.get('event_type', 'unknown')
            
            # ── Measure envelope overhead vs business data ──
            if 'data' in value:
                data_json = json.dumps(value['data']).encode('utf-8')
                data_size = len(data_json)
                envelope_overhead = value_size - data_size
                
                self.component_sizes['business_data'].append(data_size)
                self.component_sizes['envelope_overhead'].append(envelope_overhead)
        except (json.JSONDecodeError, UnicodeDecodeError):
            pass
        
        self.sizes_by_event_type[event_type].append(total_size)
        
        return {
            'topic': topic,
            'event_type': event_type,
            'key_size': key_size,
            'value_size': value_size,
            'header_size': header_size,
            'total_size': total_size,
        }
    
    def run(self, max_messages=500):
        """Analyze messages from subscribed topics."""
        print("  📏 Analyzing message sizes...\n")
        
        count = 0
        
        while self.running and count < max_messages:
            msg = self.consumer.poll(timeout=1.0)
            
            if msg is None:
                if count > 0:
                    break  # No more messages
                continue
            
            if msg.error():
                if msg.error().code() == KafkaError._PARTITION_EOF:
                    continue
                continue
            
            self.analyze_message(msg)
            count += 1
        
        self.consumer.close()
        return count
    
    def display_report(self):
        """Display comprehensive size analysis report."""
        print(f"\n  ╔══════════════════════════════════════════════════════╗")
        print(f"  ║           MESSAGE SIZE ANALYSIS REPORT                ║")
        print(f"  ╠══════════════════════════════════════════════════════╣")
        
        # ── Overall statistics ──
        all_sizes = []
        for sizes in self.sizes_by_topic.values():
            all_sizes.extend(sizes)
        
        if not all_sizes:
            print(f"  ║  No messages analyzed                                 ║")
            print(f"  ╚══════════════════════════════════════════════════════╝")
            return
        
        print(f"  ║                                                        ║")
        print(f"  ║  OVERALL:                                               ║")
        print(f"  ║    Messages analyzed: {len(all_sizes):>8}                       ║")
        print(f"  ║    Average size:      {statistics.mean(all_sizes):>8.0f} bytes"
              f"                ║")
        print(f"  ║    Median size:       {statistics.median(all_sizes):>8.0f} bytes"
              f"                ║")
        print(f"  ║    Min size:          {min(all_sizes):>8} bytes"
              f"                ║")
        print(f"  ║    Max size:          {max(all_sizes):>8} bytes"
              f"                ║")
        
        if len(all_sizes) > 1:
            print(f"  ║    Std deviation:     {statistics.stdev(all_sizes):>8.0f} bytes"
                  f"                ║")
            p95 = sorted(all_sizes)[int(len(all_sizes) * 0.95)]
            p99 = sorted(all_sizes)[int(len(all_sizes) * 0.99)]
            print(f"  ║    P95 size:          {p95:>8} bytes"
                  f"                ║")
            print(f"  ║    P99 size:          {p99:>8} bytes"
                  f"                ║")
        
        # ── Component breakdown ──
        print(f"  ║                                                        ║")
        print(f"  ║  COMPONENT BREAKDOWN (averages):                        ║")
        
        for component, sizes in self.component_sizes.items():
            if sizes:
                avg = statistics.mean(sizes)
                pct = (avg / statistics.mean(all_sizes) * 100) if all_sizes else 0
                print(f"  ║    {component:<22} {avg:>7.0f} bytes"
                      f"  ({pct:>5.1f}%)     ║")
        
        # ── By event type ──
        print(f"  ║                                                        ║")
        print(f"  ║  BY EVENT TYPE:                                         ║")
        
        for event_type, sizes in sorted(self.sizes_by_event_type.items()):
            avg = statistics.mean(sizes)
            count = len(sizes)
            print(f"  ║    {event_type:<28} avg={avg:>6.0f}B"
                  f"  n={count:<5}║")
        
        # ── By topic ──
        print(f"  ║                                                        ║")
        print(f"  ║  BY TOPIC:                                              ║")
        
        for topic, sizes in sorted(self.sizes_by_topic.items()):
            avg = statistics.mean(sizes)
            total_mb = sum(sizes) / (1024 * 1024)
            short_topic = topic.split('.')[-2] + '.' + topic.split('.')[-1]
            print(f"  ║    {short_topic:<24} avg={avg:>6.0f}B"
                  f"  total={total_mb:.2f}MB ║")
        
        # ── Warnings ──
        max_size = max(all_sizes)
        kafka_max = 1048576  # 1MB default
        
        print(f"  ║                                                        ║")
        print(f"  ║  SIZE WARNINGS:                                         ║")
        
        if max_size > kafka_max * 0.8:
            print(f"  ║    🚨 Max message {max_size}B is >{kafka_max*0.8:.0f}B"
                  f" (80% limit)    ║")
        elif max_size > kafka_max * 0.5:
            print(f"  ║    ⚠️  Max message {max_size}B is >{kafka_max*0.5:.0f}B"
                  f" (50% limit)    ║")
        else:
            print(f"  ║    ✅ All messages well within size limits"
                  f"              ║")
        
        # ── Envelope overhead analysis ──
        if self.component_sizes['envelope_overhead']:
            overhead_avg = statistics.mean(self.component_sizes['envelope_overhead'])
            data_avg = statistics.mean(self.component_sizes['business_data'])
            overhead_pct = overhead_avg / (overhead_avg + data_avg) * 100
            
            print(f"  ║                                                        ║")
            print(f"  ║  ENVELOPE OVERHEAD:                                     ║")
            print(f"  ║    Avg envelope overhead: {overhead_avg:>6.0f} bytes"
                  f"  ({overhead_pct:.1f}%)      ║")
            print(f"  ║    Avg business data:     {data_avg:>6.0f} bytes"
                  f"                ║")
            
            if overhead_pct > 50:
                print(f"  ║    ⚠️  Overhead > 50% — consider binary format"
                      f"        ║")
            else:
                print(f"  ║    ✅ Overhead is reasonable"
                      f"                        ║")
        
        print(f"  ║                                                        ║")
        print(f"  ╚══════════════════════════════════════════════════════╝")

# ============================================================
# FILE: 05_correlation_tracer.py

from confluent_kafka import Consumer, KafkaError

class CorrelationTracer:
    """
    Traces the lifecycle of a business transaction across services.
    
    A single shopping session generates events across:
      cart-service → order-service → payment-service
    
    All events share a correlation_id.
    This tracer reconstructs the full chain.
    """
    
    def __init__(self, bootstrap_servers, topics):
        self.running = True
        self.chains = defaultdict(list)
        
        self.consumer = Consumer({
            'bootstrap.servers': bootstrap_servers,
            'group.id': f'correlation-tracer-{int(time.time())}',
            'auto.offset.reset': 'earliest',
            'enable.auto.commit': True,
        })
        
        self.consumer.subscribe(topics)
        signal.signal(signal.SIGINT, lambda s, f: setattr(self, 'running', False))
    
    def collect_events(self, max_messages=500):
        """Collect events and group by correlation_id."""
        print("  🔍 Collecting events for correlation tracing...\n")
        
        count = 0
        empty_polls = 0
        
        while self.running and count < max_messages and empty_polls < 20:
            msg = self.consumer.poll(timeout=0.5)
            
            if msg is None:
                empty_polls += 1
                continue
            
            if msg.error():
                if msg.error().code() == KafkaError._PARTITION_EOF:
                    empty_polls += 1
                    continue
                continue
            
            empty_polls = 0
            count += 1
            
            # ── Extract correlation_id from headers (fast path) ──
            correlation_id = None
            event_type_header = None
            
            if msg.headers():
                for k, v in msg.headers():
                    if k == 'correlation_id' and v:
                        correlation_id = v.decode('utf-8')
                    elif k == 'event_type' and v:
                        event_type_header = v.decode('utf-8')
            
            if not correlation_id:
                # ── Fallback: try to get from value ──
                try:
                    value = json.loads(msg.value().decode('utf-8'))
                    correlation_id = value.get('metadata', {}).get('correlation_id')
                    if not event_type_header:
                        event_type_header = value.get('event_type', 'unknown')
                except (json.JSONDecodeError, UnicodeDecodeError):
                    continue
            
            if correlation_id:
                # ── Parse full value for chain details ──
                try:
                    value = json.loads(msg.value().decode('utf-8'))
                except (json.JSONDecodeError, UnicodeDecodeError):
                    value = {}
                
                ts_type, ts_value = msg.timestamp()
                
                self.chains[correlation_id].append({
                    'event_type': event_type_header or value.get('event_type', 'unknown'),
                    'event_id': value.get('event_id', 'unknown'),
                    'event_time': value.get('event_time', ''),
                    'source_service': value.get('metadata', {}).get('source_service', 'unknown'),
                    'topic': msg.topic(),
                    'partition': msg.partition(),
                    'offset': msg.offset(),
                    'kafka_timestamp_ms': ts_value,
                    'data_summary': self._summarize_data(value.get('data', {})),
                })
        
        self.consumer.close()
        return count
    
    def _summarize_data(self, data):
        """Create a brief summary of business data for display."""
        summary_parts = []
        
        if 'user_id' in data:
            summary_parts.append(f"user={data['user_id']}")
        if 'order_id' in data:
            summary_parts.append(f"order={data['order_id']}")
        if 'payment_id' in data:
            summary_parts.append(f"payment={data['payment_id']}")
        if 'item' in data:
            item = data['item']
            summary_parts.append(
                f"item={item.get('product_name', '?')} "
                f"x{item.get('quantity', '?')}"
            )
        if 'total_amount' in data:
            summary_parts.append(f"total=${data['total_amount']}")
        if 'amount' in data:
            summary_parts.append(f"amount=${data['amount']}")
        
        return ' | '.join(summary_parts) if summary_parts else '(no summary)'
    
    def display_chains(self, min_events=2, max_display=10):
        """Display correlation chains with full detail."""
        # ── Sort by number of events (most complete first) ──
        sorted_chains = sorted(
            self.chains.items(),
            key=lambda x: len(x[1]),
            reverse=True
        )
        
        # ── Filter chains with minimum events ──
        qualifying = [
            (cid, events) for cid, events in sorted_chains
            if len(events) >= min_events
        ]
        
        print(f"\n  ╔══════════════════════════════════════════════════════════╗")
        print(f"  ║           CORRELATION CHAIN ANALYSIS                      ║")
        print(f"  ╠══════════════════════════════════════════════════════════╣")
        print(f"  ║  Total unique correlations: {len(self.chains):<28}║")
        print(f"  ║  Chains with {min_events}+ events:     {len(qualifying):<28}║")
        print(f"  ╠══════════════════════════════════════════════════════════╣")
        
        for i, (corr_id, events) in enumerate(qualifying[:max_display]):
            # ── Sort events by timestamp ──
            events_sorted = sorted(events, key=lambda e: e.get('kafka_timestamp_ms', 0))
            
            # ── Determine chain completeness ──
            event_types = {e['event_type'] for e in events_sorted}
            has_cart = any('cart' in et for et in event_types)
            has_order = any('order' in et for et in event_types)
            has_payment = any('payment' in et for et in event_types)
            
            if has_cart and has_order and has_payment:
                status = "✅ COMPLETE"
            elif has_cart and has_order:
                status = "⏳ AWAITING PAYMENT"
            elif has_cart:
                status = "🛒 CART ONLY"
            else:
                status = "❓ PARTIAL"
            
            # ── Calculate latency ──
            timestamps = [e['kafka_timestamp_ms'] for e in events_sorted
                         if e.get('kafka_timestamp_ms')]
            latency_ms = (max(timestamps) - min(timestamps)) if len(timestamps) >= 2 else 0
            
            print(f"  ║                                                          ║")
            print(f"  ║  🔗 Chain: {corr_id[:36]}...    ║")
            print(f"  ║     Status: {status:<20} "
                  f"Latency: {latency_ms}ms{' '*(14-len(str(latency_ms)))}║")
            print(f"  ║     Events ({len(events_sorted)}):                       "
                  f"                    ║")
            
            for j, event in enumerate(events_sorted):
                is_last = j == len(events_sorted) - 1
                connector = "└──" if is_last else "├──"
                
                icon = {
                    'cart': '🛒',
                    'order': '📦',
                    'payment': '💳',
                }.get(event['event_type'].split('.')[0], '📨')
                
                et_short = event['event_type']
                source = event['source_service']
                
                print(f"  ║     {connector} {icon} {et_short:<28} [{source}]   ║")
                print(f"  ║     {'   ' if is_last else '│  '}"
                      f"  {event['data_summary'][:48]:<50}║")
            
            if i < len(qualifying[:max_display]) - 1:
                print(f"  ║  ────────────────────────────────────────────────────── ║")
        
        remaining = len(qualifying) - max_display
        if remaining > 0:
            print(f"  ║                                                          ║")
            print(f"  ║  ... and {remaining} more complete chains                 ║")
        
        print(f"  ║                                                          ║")
        print(f"  ╚══════════════════════════════════════════════════════════╝")
    
    def display_chain_statistics(self):
        """Show aggregate statistics about correlation chains."""
        chain_lengths = [len(events) for events in self.chains.values()]
        
        if not chain_lengths:
            print("  No chains to analyze")
            return
        
        print(f"\n  ┌────────────────────────────────────────────────┐")
        print(f"  │  CHAIN LENGTH DISTRIBUTION                       │")
        print(f"  ├────────────────────────────────────────────────┤")
        
        length_counts = defaultdict(int)
        for length in chain_lengths:
            length_counts[length] += 1
        
        for length, count in sorted(length_counts.items()):
            bar = "█" * min(count, 40)
            print(f"  │  {length} events: {count:>4} chains  {bar:<40}│")
        
        print(f"  ├────────────────────────────────────────────────┤")
        
        import statistics as stats_module
        print(f"  │  Average chain length: "
              f"{stats_module.mean(chain_lengths):.1f} events"
              f"{' '*17}│")
        print(f"  │  Longest chain:        "
              f"{max(chain_lengths)} events"
              f"{' '*20}│")
        
        # ── Completeness rate ──

            1 for events in self.chains.values()
            if any('payment.completed' in e['event_type'] for e in events)

        total = len(self.chains)
        rate = complete / total * 100 if total > 0 else 0
        
        print(f"  │  Completion rate:       "
              f"{rate:.1f}% ({complete}/{total})"
              f"{' '*(17-len(str(complete))-len(str(total)))}│")
        print(f"  └────────────────────────────────────────────────┘")
